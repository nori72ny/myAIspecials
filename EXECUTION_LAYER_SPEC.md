# AI Workspace 「Execution Layer（実行レイヤー）」設計書

本書は、AI Workspaceにおける「指示や提案を行うだけの対話型AI」から、「ドキュメント、プレゼンテーション、データ、プログラム、Webアプリケーションなどの各種成果物を実際に作成・ビルド・出力する自律駆動型システム」へと拡張するための**「Execution Layer（実行レイヤー）」**の設計定義書です。

---

## 1. 実行フロー (Execution Flow Sequence)

実行レイヤーは、AIエージェントの「思考・指示」を受け取り、バックエンドのサンドボックスや専用生成エンジン、ファイル操作ワーカーと連携して成果物を物理的に生成します。

```
                       [1. エージェントの実行命令 (JSON)]
                                       │
                                       ▼
                       [2. Execution Dispatcher]
                                       │
            ┌──────────────────────────┼──────────────────────────┐
            ▼ (Search & Intel)         ▼ (Document & Office)      ▼ (Code & Web App)
     +---------------+          +------------------+       +------------------+
     |  Web Search   |          |  Doc Generator   |       |  Code Generator  |
     |  Deep Research|          |  PPT/Excel/PDF   |       |  Web & AI App    |
     +──────┬────────+          +────────┬─────────+       +────────┬─────────+
            │                            │                            │
            └──────────────────────────┼──────────────────────────┘
                                       ▼
                         [3. File System / Sandbox]
                                       │ (一時保存 & チェック)
                                       ▼
                        [4. ZIP Packager / Compiler]
                                       │
                                       ▼
                         [5. Artifact DB & Storage]
                                       │
                                       ▼
                     [6. UI Preview & User Download] (再実行・ロールバック可能)
```

### 各フェーズの実行詳細
1.  **Execution Dispatcher（実行命令のディスパッチ）**:
    上流のエージェントやワークフローエンジンから発行された「実行指示（例: `COMMAND: GENERATE_PPT`, `THEME: LIGHT`, `CONTENT: JSON`）」を解析。適切な実行エンジンへタスクを振り分けます。
2.  **各種実行エンジンの稼働 (Engine Workers)**:
    *   **Web検索 / Deep Research**: APIやバックグラウンドクローラーを起動。並行で最大数十のソースにアプローチし、分析用生データを収集。
    *   **ドキュメント / PowerPoint / Excel / PDF生成**: 構造化されたテキストデータを基に、各バイナリ（Office XML / PDF形式）へとパッキング・スタイル適用。
    *   **Webサイト / AIアプリ / コード生成**: TypeScript / Tailwind CSS / HTMLコンポーネントをコードとして構築し、ビルドテスト用仮想ディレクトリに配置。
3.  **Sandbox & ZIP Packager**:
    生成されたコードやドキュメントを一時サンドボックス内で整理。必要に応じて静的アセットを同梱し、最終的な配布用アーカイブ（ZIP）へと圧縮。
4.  **Artifact & Log Ledger（成果物・ログの台帳登録）**:
    成果物のハッシュ値、生成日時、バージョンをデータベースに登録。実行中の標準出力/標準エラー、AIの心の声、ステップの推移をリアルタイム実行ログ（stdout風）としてストリーミング。

## 2. データ構造・スキーマ設計 (Execution Schema Design)

実行レイヤーが取り扱う「実行タスク」「履歴」「物理ファイル」「詳細ログ」をセキュアに追跡するデータスキーマです。

```
                  +---------------------------+
                  |     execution_tasks       |
                  +---------------------------+
                  | id (PK)                   |
                  | workflow_task_id (FK)     |
                  | engine_type               |
                  | input_payload (JSON)      |
                  | status                    |
                  | retries                   |
                  | started_at                |
                  | finished_at               |
                  +-------------+-------------+
                                | 1
                                |
                                | 1..N
                  +-------------v-------------+
                  |     execution_logs        |
                  +---------------------------+
                  | id (PK)                   |
                  | execution_task_id (FK)    |
                  | timestamp                 |
                  | stream_type (stdout/err)  |
                  | step_name                 |
                  | message (TEXT)            |
                  +-------------+-------------+
                                | 1
                                |
                                | 1..N
                  +-------------v-------------+
                  |    execution_artifacts    |
                  +---------------------------+
                  | id (PK)                   |
                  | execution_task_id (FK)    |
                  | filename                  |
                  | file_type (extension)     |
                  | storage_url               |
                  | byte_size                 |
                  | version                   |
                  | checksum (SHA256)         |
                  | created_at                |
                  +---------------------------+
```

### テーブル定義

#### ① `execution_tasks` (実行タスクマスター)
個別の自動実行処理の進行状況を管理。
*   `id`: UUID (Primary Key)
*   `workflow_task_id`: UUID (Foreign Key, Nullable - 単発実行時)
*   `engine_type`: VARCHAR(100) (実行種別: `WEB_SEARCH`, `DEEP_RESEARCH`, `PPT_GEN`, `EXCEL_GEN`, `PDF_GEN`, `WEB_APP_GEN`, `ZIP_PACK`)
*   `input_payload`: JSON (実行に必要な全パラメータ。プロンプト、構成案、出力設定等)
*   `status`: VARCHAR(50) (状態: `QUEUED`, `RUNNING`, `SUCCESS`, `FAILED`, `CANCELLED`)
*   `retries`: INT (自動再実行カウント)
*   `started_at` / `finished_at`: TIMESTAMP

#### ② `execution_logs` (リアルタイム・詳細実行ログ)
開発時のコンソール出力や、ドキュメントレンダリング中の詳細な中間ステップ情報を記録。
*   `id`: UUID (Primary Key)
*   `execution_task_id`: UUID (Foreign Key)
*   `timestamp`: TIMESTAMP
*   `stream_type`: VARCHAR(20) (ログ種別: `STDOUT` (正常稼働), `STDERR` (異常・ワーニング), `AI_THOUGHT` (エージェントの思考ログ))
*   `step_name`: VARCHAR(100) (実行段階。例: `INITIALIZE`, `COMPILING_CSS`, `WRITING_SLIDES`, `PACKAGING`)
*   `message`: TEXT (具体的な処理内容メッセージ)

#### ③ `execution_artifacts` (物理成果物登録)
ビルドや生成によって物理的にストレージに保存されたファイル。
*   `id`: UUID (Primary Key)
*   `execution_task_id`: UUID (Foreign Key)
*   `filename`: VARCHAR(255) (ファイル名。例: `競合分析_営業提案書_Ver2.pptx`)
*   `file_type`: VARCHAR(50) (拡張子。例: `pptx`, `xlsx`, `pdf`, `zip`, `html`)
*   `storage_url`: VARCHAR(1024) (クラウドストレージまたはローカル保存パス)
*   `byte_size`: BIGINT (ファイルサイズ)
*   `version`: INT (世代管理。再実行時のインクリメント用)
*   `checksum`: VARCHAR(64) (ファイルの整合性を確認するSHA-256ハッシュ)
*   `created_at`: TIMESTAMP

## 4. 実行エンジンの詳細仕様

実行レイヤーで駆動する、10の個別タスク生成・実行ワーカーの仕組み。

1.  **Web検索 ＆ Deep Researchワーカー**
    *   **仕様**: `Perplexity / Google Search API` をバックエンドに指定。単なるキーワード一致だけでなく、ドメイン指定検索や日時フィルタ（「過去1時間」「過去1ヶ月」等）を自動付与してクローリング。
    *   **特徴**: 検索結果をクローラーが巡回して本文データをパースし、不要な広告やナビゲーションを除外してMarkdownテキストとして統合。
2.  **ビジネスドキュメント（docx, pdf）生成ワーカー**
    *   **仕様**: `Pizzicato` や `Puppeteer`（HTML-to-PDFコンバータ）を利用。
    *   **特徴**: Writer Agentが生成したMarkdownを美しくタイポグラフィ処理された印刷用PDF / 編集可能Wordファイル（docx）へ自律コンバート。
3.  **PowerPoint（pptx）自動レイアウトワーカー**
    *   **仕様**: `PptxGenJS` ライブラリのラッパーエンジンを裏側で実行。
    *   **特徴**: スライド全体のテーマ色（例: コーポレートブルー、フォレストグリーン）を検知し、表紙、アジェンダ、2カラム構成、グラフ埋め込みなど、各スライドのインテントに適した最適なスライド構造をプログラマティックに自動レイアウト。
4.  **Excel（xlsx）データ・グラフ構造化ワーカー**
    *   **仕様**: `ExcelJS / SheetJS` などのライブラリを使用。
    *   **特徴**: 競合他社の料金比較データや、MEO/SEOの分析露出スコアを、表、フィルタ、集計、自動グラフ（円グラフ、棒グラフ）付きのExcelファイルへと自動構造化。
5.  **Webサイト ＆ AIアプリ・コード生成ワーカー**
    *   **仕様**: `Vite` および `Next.js` ボイラープレートを裏側のコンパイラとして用意。
    *   **特徴**: ユーザーが入力した要件に合わせ、React、TypeScript、Tailwind CSSを用いた動的なSPAの全コードをファイルツリーとしてビルド。エラーがあれば自動的に内部でリコンパイル（Self-Correction）を行い、バグフリーなZIP成果物としてエクスポート。

## 5. 再実行 (Re-execution) とロールバックの仕組み

実行レイヤーには、一瞬で過去の動作パラメータを再ロードして実行をやり直せる**「インポータント・ステート（状態の冪等性）」**が組み込まれています。

1.  **入力の固定化（Snapshot Payload）**:
    *   生成完了時の「プロンプト」「使用したAIモデル」「APIバージョン」「各種実行エンジンに渡されたJSONパラメータ」を、成果物のバージョン情報に完全に紐付けてスナップショット保存します。
2.  **ワンクリック再実行 (One-click Retry)**:
    *   エラー発生時、あるいは要件を少しだけ変えて再生成したいとき、過去の入力パラメータをそのままディスパッチャーに再投入。
    *   「1箇所だけ設定を変更（例: テーマをSlateからDarkへ）」して実行を実行キューに投げ直すことで、既存の構造を維持しながら成果物のみを高速で再ビルドします。

# AI Workspace 「社内アセット・アライアンスストア（Internal Marketplace）」機能設計仕様書

本書は、AI Workspace内で開発・最適化されたAIエージェント、自動化ワークフロー、プロンプト、AIチーム、アプリケーション、Webサイト、カスタムダッシュボードなどのあらゆるデジタル資産を一元的に保存、バージョン管理、検索、およびワンクリックで他プロジェクトや他組織へ「再利用・複製」するための社内マーケットプレイスシステム**「Internal Marketplace（内部マーケットプレイス）」**のシステム構成、データスキーマ、テンプレート複製設計、検索アーキテクチャ、および画面設計を定義した製品設計仕様書です。

開発したワークフローやエージェントが各プロジェクト内にクローズド（孤立）して消えていくのを防ぎ、組織全体の「再利用可能な知財アセット（Reusable Asset Engine）」として共有・循環。さらにAIが状況に応じて「類似テンプレートの推薦」「アセットの自動生成」をサポートする、AI Workspace全体の**「知財総合流通プラットフォーム（Intellectual Asset Hub）」**の役割を担います。

---

## 1. システム概要 ＆ 流通コンセプト (Overview & Concept)

Internal Marketplaceは、プロジェクトで成功を収めた「エージェント構成」や「ワークフロー」、「プロンプト」をワンクリックでパッケージ（テンプレート化）してストアに登録します。登録された資産は、セマンティック検索やタグ付け、AIによるリコメンデーションを通じて他のユーザーやプロジェクトに発見され、設定・API接続構成を自動クローン（最適コピー）して即座に別プロジェクトへ導入（インストール）されます。

```
          [ 各プロジェクト / 開発環境 ]                   [ ストア流通・再利用エンジン ]
    ┌──────────────────────────────────────┐        ┌──────────────────────────────────────┐
    │ 💻 新宿個室カフェ LP開発プロジェクト │        │      Internal Marketplace Store      │
    │  ├─ 👤 NextJS_Coder_Pro (構成情報)  ──┼───────┼─➔ 📦 [新規アセットを登録（保存）]      │
    │  ├─ ⚙️ LP_Deploy_Workflow (フロー)   ──┼───────┼─➔    - バージョン: v1.0.0            │
    │  └─ 📝 MEO_Auto_Prompts (プロンプト) ──┼───────┼─➔    - 依存メタデータのカプセル化     │
    └──────────────────────────────────────┘        └──────────────────┬───────────────────┘
                                                                       │
                       ┌───────────────────────────────────────────────┘
                       ▼ (検索・自動マッチング)
    ┌──────────────────────────────────────────────────────────────────────────────────────┐
    │ 💡 AI Support & Search (AIレコメンダー ＆ 高度検索)                                  │
    │  ├─ 1. セマンティック・ハイブリッド検索 (Vector & Tag)                                │
    │  ├─ 2. 現在のプロジェクトコンテキストから「類似テンプレート」をAIが自動推薦              │
    │  └─ 3. 必要アセットがストアにない場合、既存アセットをベースにAIが「自動生成」           │
    └───────────────────────────────────────────────┬──────────────────────────────────────┘
                                                    │
                                                    ▼ (ワンクリックインストール)
    ┌──────────────────────────────────────────────────────────────────────────────────────┐
    │ 📥 他プロジェクトへのワンクリック導入 (One-click Auto Clone & Install)                │
    │  ├─ 設定コピー: 環境変数、API接続、OAuth配線を安全に再構築 (APIキーはプレースホルダ化)  │

    │  └─ ワークフロー複製: 各ステップ、依存する自動化トリガーのスケジュールを安全に新規配線  │
    └──────────────────────────────────────────────────────────────────────────────────────┘
```

## 3. データベース・スキーマ設計 (Data Schema Design)

ストアに並ぶアセット、アセットの構成メタデータ、評価（レビュー）、お気に入り、およびインストール実績を厳格にリレーショナル管理するスキーマ設計。

```
                  +---------------------------+
                  |     marketplace_assets    |
                  +---------------------------+
                  | id (PK)                   |
                  | company_id (FK)           |
                  | creator_user_id           |
                  | title (VARCHAR)           |  -- 例: '領収書OCR転記フロー'
                  | category (VARCHAR)        |  -- 'AGENTS', 'WORKFLOWS', 'PROMPTS', etc.
                  | description (TEXT)        |
                  | current_version (VARCHAR) |  -- 'v3.0.1'
                  | usage_count (INT)         |
                  | rating_average (DECIMAL)  |
                  | source_metadata_json      |  -- アセットを構成する実体定義 (JSON)
                  | search_vector (VECTOR)    |  -- セマンティック検索用の埋め込みベクトル
                  | created_at                |
                  | updated_at                |
                  +-------------+-------------+
                                | 1
                                |
         ┌──────────────────────┼──────────────────────┬──────────────────────┐
         | 1..N                 | 1..N                 | 1..N                 | 1..N
  +------v------+        +------v------+        +------v------+        +------v------+
  | asset_tags  |        |asset_reviews|        |asset_install|        | asset_      |
  |             |        |             |        | _logs       |        | favorites   |
  +-------------+        +-------------+        +-------------+        +-------------+
  | id (PK)     |        | id (PK)     |        | id (PK)     |        | id (PK)     |
  | asset_id(FK)|        | asset_id(FK)|        | asset_id(FK)|        | asset_id(FK)|
  | tag_name    |        | user_id     |        | installed_  |        | user_id     |
  |  (VARCHAR)  |        | rating (INT)|        |  _project_id|        | created_at  |
  | created_at  |        | comment     |        | installed_by|        +-------------+
  +-------------+        |  (TEXT)     |        | version_    |
                         | created_at  |        |  installed  |
                         +-------------+        | status (STR)|
                                                |  --'SUCCESS'|
                                                |  --'FAILED' |
                                                | created_at  |
                                                +-------------+
```

### テーブル定義詳細

#### ① `marketplace_assets` (ストア資産マスタ)
アセットの基本属性と、再利用・複製を実行するためのJSONシリアライズド定義体。
*   `id`: UUID (Primary Key)
*   `company_id`: UUID (Foreign Key - 組織スペースのID)
*   `creator_user_id`: UUID (アセットを作成・登録したユーザーID)
*   `title`: VARCHAR(255) (資産名。例: `領収書OCR解析 ➔ Sheets自動転記フロー`)
*   `category`: VARCHAR(100) (カテゴリ。`AGENTS`, `WORKFLOWS`, `TEAMS`, `PROMPTS`, `APPS`, `WEBSITES`, `DASHBOARDS`, `TEMPLATES`, `AUTOMATION`)
*   `description`: TEXT (アセットの概要、使い方、メリットなどの解説)
*   `current_version`: VARCHAR(50) (現在の最新バージョン。例: `v3.0.1`)
*   `usage_count`: INT (このアセットがインポート/複製された総回数)
*   `rating_average`: DECIMAL(3, 2) (ユーザー評価の平均値。0.00〜5.00)
*   `source_metadata_json`: JSON (アセットの実体シリアライズデータ。エージェントの人格プロンプト、メモリRAG接続設定、ツール配線、ワークフローの各ステップ構造などの定義情報を内包。APIキー等の機密情報は削除・プレースホルダ化される)
*   `search_vector`: VECTOR(1536) (セマンティック検索用の1536次元ベクトルデータ。Gemini-Embeddingにより生成)
*   `created_at` / `updated_at`: TIMESTAMP

#### ② `asset_tags` (アセット分類タグ)
アセットを素早くフィルタリングするためのタグ情報。
*   `id`: UUID (Primary Key)
*   `asset_id`: UUID (Foreign Key)
*   `tag_name`: VARCHAR(100) (タグ名。例: `MEO`, `LINE`, `会計`, `SEO`, `NextJS`, `自動化`)
*   `created_at`: TIMESTAMP

#### ③ `asset_reviews` (レビュー・評価テーブル)
社内ユーザーからの生のフィードバックとKPI評価スコア。
*   `id`: UUID (Primary Key)
*   `asset_id`: UUID (Foreign Key)
*   `user_id`: UUID (評価を投稿した社内ユーザーID)
*   `rating`: INT (1〜5点満点)
*   `comment`: TEXT (具体的な使用感、改善リクエスト)
*   `created_at`: TIMESTAMP

#### ④ `asset_install_logs` (複製・インポート履歴ログ)
どのアセットが、どのプロジェクトに、誰によって、どのバージョンで導入されたかの履歴。
*   `id`: UUID (Primary Key)
*   `asset_id`: UUID (Foreign Key)
*   `installed_project_id`: UUID (導入先のプロジェクトID)
*   `installed_by`: UUID (インポートを実行したユーザーID)
*   `version_installed`: VARCHAR(50) (導入時のアセットバージョン)
*   `status`: VARCHAR(50) (導入結果。`SUCCESS` (成功), `FAILED` (何らかの依存エラー等による失敗))
*   `created_at`: TIMESTAMP

## 4. 検索設計 (Advanced Search Architecture)

Internal Marketplace内のアセットを、ユーザーが曖昧な言葉（自然言語）や要件からでも高確率で発見できるようにするための、**「ハイブリッド・セマンティック検索エンジン」**のアルゴリズム設計。

```
                       [ ユーザーの検索クエリ ]
                (例: "LINEで口コミを集めてMEO順位を上げるフロー")
                                  │
         ┌────────────────────────┴────────────────────────┐
         ▼ (経路 1)                                         ▼ (経路 2)
  [ キーワード / タグマッチ ]                        [ ベクトル変換 (Gemini Embeddings) ]
  - SQL: LIKE '%LINE%' or LIKE '%MEO%'             - クエリを 1536次元ベクトル に変換
  - `asset_tags` の完全一致・前方一致               - `search_vector` との コサイン類似度 計算
         │                                                 │
         └────────────────────────┬────────────────────────┘
                                  ▼ (統合・スコアリング)
                    [ Reciprocal Rank Fusion (RRF) ]
         - キーワード順位とベクトル類似度順位をブレンド重み付け算出
                                  │
                                  ▼
                   [ ハイブリッド検索結果 (Top N) ]
```

### ① ハイブリッド検索スコアリング
1.  **ベクトル類似度（Semantic Score）**:
    ユーザーの検索クエリを `text-embedding-004` (Gemini API) でベクトル化。データベースの `search_vector` とコサイン類似度距離を計測。これにより、語句が一致していなくても「集客したい」「経理作業を楽にしたい」といった「目的（セマンティクス）」を解釈した検索結果を出力します。
2.  **キーワード・タグ一致（Keyword Score）**:
    タイトル、説明文、および `tag_name` との一致率を算出。
3.  **ブレンド評価値（RRF Score）**:
    セマンティック類似度とキーワードマッチ数を重み付け（例: ベクトル比率 70%, キーワード比率 30%）合成し、さらにアセットの「人気度（利用回数）」「高評価レート」で補正を加えて最終ランキング。

### ② AIアシストによる「おすすめ（Smart Recommendation）」
*   **コンテキスト・アウェア推薦**:
    ユーザーが「新規プロジェクト作成画面」を開いた際、AIが自動的にそのプロジェクトの「説明文（概要）」と「社内ナレッジベース（Knowledge Graph）」のコンテキストをスキャン。
    マーケットプレイス内の全アセットベクトルと比較し、コサイン類似度が `0.85` 以上の「類似テンプレート・即座に使えるエージェント・ワークフロー」を「あなたへの最適なおすすめ」として自動ポップアップ表示します。

## 5. テンプレート管理・クローンコピー設計 (Asset Replication Logic)

アセットをワンクリックで既存プロジェクトへインストールする際、環境の競合を防ぎ、APIのセキュアな引き継ぎを保証するための**「セキュア・アセットクローン・パイプライン」**の設計。

```
                     [ ストアからインポートを実行 ]
                                   │
                                   ▼
                [ 1. メタデータ解析 ➔ 機密情報の検知 ]
      ├─ データベースの `source_metadata_json` をロード
      └─ APIキー、アクセストークン、個人情報をスキャン（プレースホルダ化）
                                   │
                                   ▼
                  [ 2. 依存関係（Dependencies）の自動解決 ]
      ├─ 必要なエージェント、共有プロンプト、実行トリガーを確認
      └─ 不足しているアセットがあれば「一括導入確認ダイアログ」を表示
                                   │
                                   ▼
                  [ 3. プロジェクト固有環境へのインサート ]
      ├─ 導入先プロジェクトの `project_id` に紐づく `tasks` や `agents` を新規生成
      ├─ ストアのソース構成（プロンプト・人格・トリガー）を丸ごとディープコピー
      └─ インポート履歴テーブル（`asset_install_logs`）へ成功ステータス書き込み
                                   │
                                   ▼
                 [ 4. 外部API接続・認証の再マッピング ]
      ├─ 管理者に対し、コピーされた環境変数の設定（またはOAuth連携）を要求
      └─ 接続が完了した瞬間にワークフロー（スケジュール）を自律有効化
```

### コピールールの詳細
*   **設定コピー**: ワークフローで利用されていた環境変数（例: `SPREADSHEET_ID`）は、クローンされた際、自動的にプレースホルダ化、または導入先プロジェクトに登録された Google Sheets 接続設定へと自動再マッピングされます。
*   **Agentコピー**: エージェントの人格を決定するシステムプロンプト（System Role Instruction）、アタッチされている長期的RAGメモリ、および「アクションに使えるツール」の権限セットを一括コピー。
*   **Workflowコピー**: ワークフローの各アクティビティステップ（順番、条件分岐ロジック）を完全にディープコピー。毎日実行する cron スケジュールや、外部 Webhook 受付エンドポイントは、導入先のプロジェクト専用に「重複しない新しいエンドポイントURL」を自動発行して稼働を開始します。

## 6. 将来拡張設計 (Future Extensibility Plan)

### ① Federated Multi-Workspace Alliance Store (企業間アライアンス・ストア連携)
*   **1つの社内だけで閉じず、親会社・子会社、あるいは提携企業のアライアンスグループ間（複数の独立したAI Workspace環境）で、セキュアにストアアセットを共有・売買・導入できる仕組み**:
    *   自社のMEO/SEOチームが開発した「La Lounge集客自動化パッケージ」を、外部提携している飲食チェーン店のWorkspaceストアへライセンス供給。
    *   お互いのセキュリティポリシーに抵触しないよう、パッケージのソース（実装コード、プロンプト）は、インストール後に自動的に「暗号化・閲覧不可（ブラックボックス型AIアセット）」として配備。
    *   APIを叩く際のみ、提携先のトークンを適用させて自律駆動させ、知的財産を安全にマネタイズ・供給できる「分散型グループAIアライアンスマーケット」へと拡張します。

### ② Swarm-on-Demand Auto-Generation (要件定義からのAI・アセットオンデマンド自動生成)
*   **ユーザーが欲しいアセットを検索した際、ストア内に「完全に一致する既存テンプレート」が存在しない場合、AIがストア内の「類似アセット」をパーツとして複数組み合わせ、全く新しいアセットやワークフロー、ダッシュボードを自動でマクロ生成・納品する仕組み**:
    *   ユーザーがストアの検索窓に「LINEとStripeを連携した、飲食デリバリー自動注文フロー」と入力。
    *   検索エンジンは「完全一致するパッケージはない」と判断。
    *   ストアサポートAI（Architect Agent）が直ちにバックグラウンドで起動。
    *   ストア内にある「LINE自動返信テンプレート」と「Stripe決済ワークフロー」の2つの資産メタデータをセマンティックに抽出。
    *   この2つのコンポーネントを接続するための仲介コードとプロンプトをAIが自律設計。
    *   「ご希望のテンプレートはストアに存在しませんでしたが、社内の既存アセットを安全に合成し、動作確認済みの『LINE ✕ Stripe 飲食自動デリバリーテンプレート』を自動生成しました。こちらをインストールしますか？」と回答（オンデマンド知財合成）。
    *   人間が「はい」と答えた瞬間、世界に一つだけの最適化された自動化アセットがワンクリックで導入され、開発コストを完全ゼロ化する自律自動進化型知財ストアへと進化します。

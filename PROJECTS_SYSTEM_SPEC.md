# AI Workspace 「プロジェクト管理機能（Projects System）」設計仕様書

本書は、AI Workspaceにおける各案件（プロジェクト）を単位として、調査、分析、各種オフィス資料、Web/AIアプリケーション、専属AIエージェント、およびアップロードされたファイルアセット群を一元的に整理・集約・トラッキングするための**「プロジェクト管理機能（Projects System）」**の設計仕様書です。

単発のAI処理やチャットを散発的に行うのではなく、現実の「1つのビジネス案件」に紐づくすべての思考プロセス、行動履歴、生成された成果物を1箇所に集約することで、業務の「資産化」と「継続的なAIアシスト」を実現します。

---

## 1. 業務フロー ＆ コンセプト (Overview & Concept)

プロジェクト機能は、案件全体の状況（ステータス、期限、進捗率、アセット数）を可視化する「Dashboard」、案件内のアセットをドメイン別に管理する「Project Workspace」、案件のライフサイクルを記録する「Timeline」、そしてそのプロジェクトのコンテキストをすべて脳内にインデックスした「プロジェクト専属AI（Project AI Assistant）」をシームレスに統合します。

```
[1. プロジェクトの起案・作成] (案件名、概要、期限、カテゴリ、タグ等を定義)
              │
              ▼
[2. プロジェクト・ダッシュボード (Dashboard)] ─► (進捗率、未完了タスク、全成果物数の俯瞰)
              │
       ┌──────┴─────────────────────────────────────────┐
       ▼ (Research & Analysis)                          ▼ (Documents & Creation)
+-------------------------------+                +-------------------------------+
| - 調査 (Deep Research, 一般)  |                | - 提案書 (PPT, PDF, Word, XLS)|
| - 分析 (SEO, AIO, MEO, 競合)  |                | - 制作 (Web, App, CRM, AI)   |
+--------------┬----------------+                +---------------┬---------------+
               │                                                 │
               └───────────────────────┬─────────────────────────┘
                                       ▼
[3. ファイル＆タイムライン (Files & Timeline)] ◄── (アップロード素材、AI生成・更新・保存履歴の自動記録)
                                       │
                                       ▼
[4. プロジェクト専属AI (Project AI Assistant)] ◄── (プロジェクト内の全アセットをセマンティック学習)
                                       │
                                       ├─► 「この案件の競合分析を要約して」
                                       ├─► 「この案件用に、先週のデータを基にした提案書を作って」
                                       └─► 「前回(Ver1)と今回(Ver2)の分析データから差分を抽出して」
```

## 3. データベース・スキーマ設計 (Database Schema Design)

プロジェクトの基本パラメータ、プロジェクトを構成するドメインごとのアセット管理、およびタイムライン、アップロードされた物理ファイルを強固に管理するリレーショナルスキーマ。

```
                  +-------------------------+
                  |        projects         |
                  +-------------------------+
                  | id (PK)                 |
                  | name                    |
                  | description             |
                  | status                  |
                  | start_date              |
                  | due_date                |
                  | category_id (FK)        |
                  | organization_id         |
                  | is_favorite (Boolean)   |
                  | is_archived (Boolean)   |
                  | created_at              |
                  | updated_at              |
                  +-----+---------+---------+
                        |         |
                        | 1       | 1
                        |         |
                        | 1..N    | 1..N
                  +-----v---+   +-v-----------------------+
                  | project_|   |    project_timeline     |
                  | assets  |   +-------------------------+
                  +---------+   | id (PK)                 |
                  | id (PK) |   | project_id (FK)         |
                  | project_|   | action_type             |
                  | _id (FK)|   | entity_name             |
                  | domain  |   | description             |
                  | title   |   | performed_by            |
                  | type    |   | created_at              |
                  | ref_id  |   +-------------------------+
                  | version |
                  | metadata|
                  | created_|
                  | _at     |
                  +----+----+
                       | 1
                       |
                       | 0..N
                  +----v----+
                  | project_|
                  |  files  |
                  +---------+
                  | id (PK) |
                  | asset_id|
                  | (FK)    |
                  | filename|
                  | filepath|
                  | bytesize|
                  | checksum|
                  +---------+
```

### テーブル詳細

#### ① `projects` (プロジェクト本体・ダッシュボードマスター)
*   `id`: UUID (Primary Key)
*   `name`: VARCHAR(255) (案件名。例: 「新宿個室カフェ La Lounge 出店計画」)
*   `description`: TEXT (プロジェクトの目的、ターゲット等の概要)
*   `status`: VARCHAR(50) (進行状態。`DRAFT`, `ACTIVE` (進行中), `SUSPENDED` (保留中), `COMPLETED` (完了), `ARCHIVED` (アーカイブ済))
*   `start_date` / `due_date`: DATE (開始日と期限日)
*   `category_id`: VARCHAR(50) (カテゴリID。例: `marketing`, `sales`, `development`)
*   `organization_id`: VARCHAR(100) (セキュリティ境界用のテナント/組織ID)
*   `is_favorite`: BOOLEAN (お気に入りフラグ。お気に入り一覧への表示制御)
*   `is_archived`: BOOLEAN (アーカイブ判定。プロジェクト一覧からの一時的な除外用)
*   `created_at` / `updated_at`: TIMESTAMP

#### ② `project_assets` (ドメイン成果物連携テーブル)
プロジェクトの各フォルダ（ドメイン）に属するアセットメタ情報。
*   `id`: UUID (Primary Key)
*   `project_id`: UUID (Foreign Key)
*   `domain`: VARCHAR(100) (分類ドメイン。`RESEARCH` (調査), `ANALYSIS` (分析), `DOCUMENTS` (資料), `CREATION` (制作物), `AGENTS` (専用エージェント), `FILES` (物理ファイルアセット))
*   `title`: VARCHAR(255) (成果物名。例:「新宿深夜エリア MEO 競合店舗比較レポート」)
*   `type`: VARCHAR(50) (アセットの性質。例: `MARKDOWN`, `PPTX`, `ZIP`, `PDF`, `PNG`, `HTML_CODE`, `AGENT_CONFIG`)
*   `ref_id`: UUID (Nullable。他のテーブル、例えば `knowledge_entries`, `execution_artifacts`, `templates` の実レコードIDと紐付け)
*   `version`: INT (世代管理。例: `1`, `2`...)
*   `metadata`: JSON (その他詳細なパラメータ、進捗率、生成時間などのメタ情報)
*   `created_at`: TIMESTAMP

#### ③ `project_files` (アップロード・アセット物理ファイル)
ユーザーがプロジェクトの `Files` フォルダへドラッグ＆ドロップ、またはアップロードした物理ファイル実体。
*   `id`: UUID (Primary Key)
*   `asset_id`: UUID (Foreign Key) (アセットマスターのFILESドメイン行に紐付け)
*   `filename`: VARCHAR(255) (物理ファイル名。例: `新宿3丁目テナント間取り図.png`)
*   `filepath`: VARCHAR(1024) (クラウドストレージまたはローカルストレージ上の絶対参照パス)
*   `bytesize`: BIGINT (ファイルサイズ)
*   `checksum`: VARCHAR(64) (ファイルの完全性を保つSHA-256ハッシュ)

#### ④ `project_timeline` (プロジェクトのライフサイクル・監査履歴)
プロジェクト内でいつ、だれが、何のAIを動かし、どのファイルを保存・更新したかの足跡。
*   `id`: UUID (Primary Key)
*   `project_id`: UUID (Foreign Key)
*   `action_type`: VARCHAR(100) (操作種別。例: `PROJECT_CREATE`, `FILE_UPLOAD`, `AI_EXECUTION`, `ARTIFACT_SAVE`, `ARTIFACT_DELETE`, `AI_CHAT`)
*   `entity_name`: VARCHAR(255) (対象アセット名。例: `新宿個室カフェ_集客LPプロトタイプ`)
*   `description`: TEXT (詳細内容。例:「Coder Agentが、お問い合わせメールのバリデーションロジックを修正してLP Ver2を保存しました」)
*   `performed_by`: VARCHAR(100) (実行ユーザー名またはAIエージェント名。例: `山田 太郎`, `Analyst Agent`)
*   `created_at`: TIMESTAMP

## 4. 権限設計 (Security & Access Control Matrix)

プロジェクトには、財務諸表、機密コード、未発表のマーケティング施策が含まれるため、組織・プロジェクトメンバーの役割に応じた厳格な多層防御アクセス制御を行います。

| 権限ロール (Project Role) | 閲覧範囲 (Read Access) | 作成・編集範囲 (Write Access) | プロジェクト削除範囲 (Delete Access) | 代表的な適用例 |
| :--- | :--- | :--- | :--- | :--- |
| **管理者 (Project Owner / Admin)** | プロジェクト内の全データ、タイムライン、ファイル | 新規アセットの生成、パラメータ編集、アセットの削除、メンバーの招待 | プロジェクト自体の完全削除、またはアーカイブ | 案件のオーナー、事業統括リーダー |
| **コラボレーター (Collaborator / Member)** | プロジェクト内の全データ、タイムライン、ファイル | 新規アセットの生成、パラメータ編集、ファイルのアップロード | 自身が作成したアセットの削除のみ (プロジェクト自体は削除不可) | 営業担当、マーケター、開発エンジニア |
| **閲覧者 (Viewer / Guest)** | プロジェクト内のアセット、タイムラインの閲覧、ファイルのプレビュー | 全ての書き込み、AIチャットの実行、アセット編集は不可 | 不可 | クライアント、外部パートナー、意思決定者 |

## 5. 実行フロー (Integration Execution Flow)

プロジェクトに所属するメンバーやAIが、新しいアセット（例: 競合分析レポートから営業提案書PPT、そして対策LPまで）を連携生成していくプロセス。

```
[1. ユーザーが新宿カフェ出店プロジェクトを起動]
                       │
                       ▼
[2. Analysisドメインから「新宿深夜エリア MEO 競合店舗比較レポート」を選択]
                       │
                       ▼
[3. [このアセットを基に資料を作成] アクションを実行]
                       │
                       ▼
[4. Project AI Assistant (ルーティング) が起動]
      ├─ 選択されたレポートおよびプロジェクトの「説明」をコンテキストとしてロード
      ├─ Template Marketplace から「営業提案書作成」テンプレートを自動選択
      └─ PPTレイアウトワーカー (Execution Layer) に構造化パラメータをディスパッチ
                       │
                       ▼
[5. PptxGenJS を通じて「新宿店舗出店計画_提案スライド.pptx」を自律生成]
                       │
                       ▼
[6. 成果物をプロジェクトの「Documentsドメイン」へ自動保存、Timelineに「PPT生成履歴」を自動コミット]
                       │
                       ▼
[7. ユーザーにリアルタイム完了通知、スライドのダウンロード・プレビューが可能に]
```

## 6. 将来拡張設計 (Future Extensibility Plan)

今後、プロジェクト管理機能をよりエンタープライズ対応させ、自律的に業務を遂行・推進させるための拡張計画。

### ① 担当者（タスク自動割り当て）および自動通知
*   **自律タスク分割 (Task Auto-Assignment)**:
    プロジェクトの期限や進捗率に基づき、AIが「今週やるべきタスク」を自動でマイルストーン分解し、プロジェクトメンバーへ自動割り当て。期限が迫った未完了タスクがある場合、連携されたSlackやTeamsの担当者チャネルにAIが「リマインドメッセージ」と「下書き成果物」をプッシュ通知します。
*   **財務・データ分析ドメインの連携拡張**:
    将来的に、プロジェクトアセットに「EC分析」「財務分析」「採用分析」の専用フォルダを追加。アップロードされた企業の財務諸表PDFや採用応募CSVを、AIがプライバシーを保護したままプロジェクト内で横断分析できるように設計します。

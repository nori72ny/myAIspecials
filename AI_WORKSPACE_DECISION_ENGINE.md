# Project Bible - Chapter 06: AI Decision Engine (AI判断エンジン)
**Classification:** Core Intelligence Architecture / Strategic Decisioning Specs  
**Version:** 0.6.0  

---

## 1. 哲学 (Decisioning Philosophy)

> **AIは、場当たり的に回答文を連続生成（Generative output）するマシーンではない。**  
> **すべての思考、すべての創造を開始する前に、まず「最適な戦略とゴール」を冷徹に決定する意思決定システムである。**

AI Workspaceの中枢神経を司る**「AI Decision Engine (AI判断エンジン)」**は、単に質問を受け取って即座に文章を書き出す処理を認めません。  
ユーザーからの入力を受け取った瞬間、あたかも百戦錬磨の経営コンサルタントや最高執行責任者（COO）が案件を精査するように、目的の解釈、難易度の査定、必要アセットの特定、最適なLLMモデルおよびエージェントの動的アロケーションを決定。  
無駄なリソース消費や不完全な回答を完全に防ぎ、一回のアクションでユーザーに「これしかない」という極上の最適解（完成品）を提示するための意思決定パイプライン（12のステップ）を自律実行します。

---

## 2. 判断エンジンの12ステップ（The 12-Step Decision Pipeline）

```
[User Goal Input]
       │
       ▼
(Step 01) Intent Analysis        ➔ 文字面をパースし、真の「目的」「背景」を完全抽出
(Step 02) Difficulty Analysis    ➔ タスクの難易度、知的精度、専門性、ハルシネーションリスクを格付け
(Step 03) Knowledge Requirement  ➔ 静的・動的RAG、Web検索、公式ドキュメント、画像/コード要件の判定
(Step 04) AI Selection           ➔ Gemini / GPT / Claude などのモデル群を最適配備
(Step 05) Agent Selection        ➔ CTO, CD, CMO などのC-Suite仮想専門エージェントを動的招集
(Step 06) Workflow Selection     ➔ ステートマシン型DAGワークフロー（検索➔制作➔監査）を自律構築
(Step 07) Truth Engine           ➔ 公式情報照合、論理・数値の矛盾確認、確信度スキャンを実行
(Step 08) Quality Engine         ➔ 成果物を10大評価項目で採点し、95点未満は強制リファクタ
(Step 09) Executive Review       ➔ Master Intelligence (CEO) による最終的な提出可否監査
(Step 10) Result Delivery        ➔ 言葉での回答ではなく、そのまま商業利用できる「成果物」を提出
(Step 11) Future Suggestion      ➔ 半年〜一年後のロードマップ、潜在リスクを先回り提案
(Step 12) Learning               ➔ 今回の成功・修正ログを User DNA Memory へ永続記憶
```

---

### STEP 01: Intent Analysis (目的・真意の解読)
ユーザーから送信されたプロンプトやファイルの文字面だけを追うことをしません。
*   **判断ロジック**:  
    入力された短い文脈から、「何をしたいのか（Goal）」「なぜそうしたいのか（Context）」「目標達成に本当に必要な要素は何か（Hidden Essentials）」を逆算マッピング。
*   **例**: 「ホームページを作って」 ➔ 集客・売上の最大化、SEO・MEO・AIOの同時ビルド、Stripe決済フォーム、モダンデザインの完全同期が必要であると解読。

### STEP 02: Difficulty Analysis (難易度・リスク評価)
タスクの複雑性、リスクレベルを測定し、内部処理の重み（推論時間、ディベート回数）を調整。
*   **評価軸**:  
    -   **質問難易度（Complexity）**: ロジック、計算、または文章のクリエイティブ度の格付け。
    -   **必要精度（Precision Required）**: 法務・ファイナンスなど誤りが絶対に許されないタスクか否か。
    -   **専門性（Expertise Depth）**: 業界特有の専門用語、公式API知識が必要か。
    -   **セキュリティ・ハルシネーションリスク**: 情報流出リスク、誤情報の危険性を事前評価。

### STEP 03: Knowledge Requirement (知識・アセット特定)
タスクの解決に「どのような知識、どのメディア、どのアセットが必要か」をマトリクス特定。
*   **自律判定**:  
    「Web検索は必要か（リアルタイム情報の有無）」「学術論文や公式APIの参照は必要か」「社内の過去プロジェクト履歴とのマージは必要か」「画像・動画・Reactコンポーネントコードの自動製造は必要か」を判断し、必要な接続パイプライン（RAG/Tools）を事前にアクティベート。

### STEP 04: AI Selection (最適言語モデル選定)
現在利用可能な世界トップクラスのLLM群から、タスクに応じて最適な組み合わせ（単一または複数並列）を自動アロケーション。
*   **マッピングルール**:  
    -   **長文解説・高レスポンス優先**: `Gemini 2.5 Flash` / `Gemini 3.1 Flash-preview`
    -   **超高度論理推論・ファクトチェック監督**: `Gemini 2.5 Pro` / `DeepSeek-R1`
    -   **複雑なロジックコード・API記述**: `GPT-4o`
    -   **極上のUI/UX美学・ドキュメント推敲**: `Claude 3.5 Sonnet`
    -   **将来追加AI（Manus, Grok等）**: ドライバー設計に基づき、性能を最大活用する。

### STEP 05: Agent Selection (専門チームの編成)
AI Corporationから、本アジェンダに最も適合する「仮想専門役員エージェント（C-Suite Swarm）」を動的に召集。
*   **アロケーション**:  
    マーケティング案件なら CMO (Marketing AI) ＋ CRO (Research AI) を。システム開発なら CTO (Engineering AI) ＋ Security AI ＋ CD (Creative AI) を招集。案件の性質に最適化された専門会議体を裏側で瞬時に結成します。

### STEP 06: Workflow Selection (動的DAGプロセスの構築)
課題をクリアするための最短かつ最も堅牢な「状態遷移付きアクションチェーン（Directed Acyclic Graph - DAG）」を自律プログラム。
*   **標準ビルドステップ**:  
    `[情報検索]` ➔ `[競合・数値分析]` ➔ `[複数モデルによる対比議論]` ➔ `[成果物コード/スライド制作]` ➔ `[QA・法務・セキュリティレビュー]` ➔ `[Excellence Loop 改善]` ➔ `[承認提出]` を、ステートマシンとして動的に連結して実行。

### STEP 07: Truth Engine (真実性・正確性検証)
ハルシネーションを抹殺するため、あらゆる主張やコードが「真実」であることを保証。
*   **検証項目**:  
    公式仕様書URLとのクロスチェック、時系列トレンドの適合、主要AI間の一致率（Consensus）、計算式の整合、前後記述の論理的一貫性を、ユーザーに提示する前に並列全件チェック。

### STEP 08: Quality Engine (95点基準評価)
生成された全成果物（文章、資料、Webアプリコード等）を冷徹に採点。
*   **絶対ルール**:  
    正確性・網羅性・デザイン視認性などの10大項目で採点。**「品質スコア95点未満はユーザーへの提出禁止」**を徹底。95点に満たないものは、自動的に「何が足りないか」の自己改善命令（Critique）を注入して前ステップ（Step 06/Step 05）へ差し戻し再生成。

### STEP 09: Executive Review (Master Intelligence 最終監査)
CEOである Master Intelligence が、プロジェクト全体の整合性、ユーザーDNAへの適合状況、全体の美学を一元的にレビュー。
*   **判定ゲート**:  
    「この成果物はユーザーの時間を100倍にし、ビジネスを圧倒的に加速できる品質か」を監査し、提出承認（Greenlight）を発行。

### STEP 10: Result Delivery (成果物中心の納品)
ユーザーの脳の認知的理解を最大化するレイアウト（逆ピラミッド表示）で成果物を提示。
*   **納品仕様**:  
    チャットテキストでの説明でお茶を濁さず、そのまま本番利用・コピペできる完璧なReactコード、洗練されたプレゼンスライドアウトライン、見やすいデータグラフ、即時実行できるデプロイボタンを、1つのステージ画面に集約して引き渡します。

### STEP 11: Future Suggestion (未来の成長予測と先回り)
今この瞬間の依頼をクリアするだけでなく、時間のタイムラインを半年から一年先まで伸ばし、ユーザーに必要な将来アセットを自律予測。
*   **能動提案**:  
    「今回のマーケティング施策の成功に伴い、半年後には顧客管理データベースの拡張と、不正アクセス検知ワークフローが必要になります。今からそのプロトタイプ戦略をお見せしましょうか？」といった先回りカード（Predictive Cards）を自動提示します。

### STEP 12: Learning (User DNA 永続記憶)
本セッションのすべて（ユーザーの最終評価、修正点、成功パターン、対話スピード、好む余白やフォントトーン）を Memory AI がセマンティック知識化。
*   **自己進化**:  
    次回以降の判断ステップ（Step 01〜Step 05）において、今回の学習ログを前提条件（Prior Constraints）としてロード。使い込むほどに「言葉にされない意図」までをも完璧に察知する、世界でただ一人のためのAI OSへと自己進化を遂げます。

# Project Bible - Chapter 03: Human First UX (人間中心のユーザー体験設計)
**Classification:** Universal UX Constitution / Cognitive Design Guidelines  
**Version:** 1.0.0  

---

## 1. コアビジョン (Human First Philosophy)

AI Workspaceにおける人間とAIのインタラクションにおいて、主役は常に「人間（User）」であり、AIはその能力を何百倍にも拡張する「外骨格（Exoskeleton）」である。  
人間が機械の都合に合わせて学習し、複雑な設定を強いられる時代は、本日をもって終了する。本憲法に定める15のUX絶対則は、人間の「認知限界」「行動心理」「意思決定パターン」から逆算された、世界で最も使いやすいUXを実現するためのデザイン規範である。

---

## 2. 人間特性に基づく15のUX絶対ルール (The 15 Absolute Rules of Human First UX)

### Rule 01: 視覚優位の原則 (The Visual Domination)
> **人間は文字を「読まない」。全体を「見る」。だから、文字の量を極限まで削ぎ落とす。**

*   **実装仕様**: 冗長な技術的挨拶（「かしこまりました。以下にコードを出力します」など）を完全に排除。直ちに視覚的なハイライトカード、データグラフ、または動作するインタラクティブプレビューを画面の主役に据える。長文の解説はスマートフォルダーやアコーディオンの中に格納し、段階的開示（Progressive Disclosure）を行う。

### Rule 02: メンタルモデルの統制 (Unified Cognitive Map)
> **人間は操作方法を「覚えない」。直感で「理解する」。だから、すべてのUIパターンを徹底的に統一する。**

*   **実装仕様**: 画面が変わっても、情報のレイアウト階層（ヘッダー、情報カード、データグラフ、タイムライン、アクションボタン）を同一ピクセル比率で統一。一度学習した「ホバーすれば浮き上がり、クリックで展開する」という指先の触覚的記憶（Muscle Memory）を、アプリケーションの全機能で使い回せるようにする。

### Rule 03: 比較疲労の自律解消 (Automated Comparison)
> **人間は複数の選択肢を比較すると脳が「疲労する」。だから、AIが事前に比較し、最適解だけを示す。**

*   **実装仕様**: 「どちらのプランが良いか？」と聞かれた際、生データを2つ提示してユーザーに考えさせない。AIが双方のメリット・デメリット・コスト・速度・将来性を比較マトリクス表（Recharts / Table）に再整理し、「結論：推奨されるのはオプションAです。理由はコストが30%抑えられるためです」と、脳の負荷を肩代わりして提示する。

### Rule 04: 質問文の自律補完 (Proactive Prompt Synthesis)
> **人間は「自分が何を質問すればいいか分からない」。だから、AIが質問と指示のゴールを先回りして完成させる。**

*   **実装仕様**: ユーザーが検索窓やチャット窓に「MEO」と1語入力しただけで、AIが周辺コンテキストを読み取り、「もしかして：店舗のMEO（ローカルSEO）を最適化し、月間の来客数を20%向上させるための、具体的な3つのローカルアクションプランを作成しますか？」と、洗練されたプロフェッショナルなプロンプトに昇華させてサジェストする。

### Rule 05: 回答ではなく「理解」の表示 (Visualized Insight over Raw Answers)
> **AIは単なる回答テキストを表示しない。対象に対するAIの深い「理解（コンテクスチュアル・インサイト）」を表示する。**

*   **実装仕様**: 単に「APIキーの取得方法」をテキストでステップ表記するのではなく、システム全体の情報の流れを示すクリーンなインフォグラフィック（SVG/CSS）を生成。「今あなたがこのステップにいる」ことを視覚的に照らし、情報の前後関係を直感的に把握させる。

### Rule 06: 成功のイメージファースト (Success Imagery First)
> **人間は「成功の完成イメージ」を目にすると、即座に行動に移る。だから、結果を最初に見せる。**

*   **実装仕様**: 「アプリを作りたい」という要求に対し、仕様書を読ませるのではなく、その瞬間に自動構築された「動作するモックアッププレビュー（React SPA）」を画面最上部にデプロイして見せる。「これがあなたの手に入る完成品です」という圧倒的な成功体験を最初に与えることで、合意と改善のフェーズへスムーズに移行する。

### Rule 07: 逆算型目的思考 (Reverse-Engineered Purpose)
> **人間にとって大切なのは「クリック数」ではなく「目的の達成」。AIは目的から逆算して行動する。**

*   **実装仕様**: ユーザーが「ファイルをSlackに送りたい」と考えたとき、「Slack設定 ➔ 連携追加 ➔ チャンネル選択 ➔ アップロード」という手数（クリック）を踏ませない。AIが「目的：Slackへの資料共有」を検知した瞬間、裏側でAPI接続を確認し、画面上には「Slackの #general に共有する [実行ボタン]」だけを1タップで配備する。

### Rule 08: 動的コンテクストメニュー (Dynamic Contextual Rails)
> **メニューを探させるな。今ユーザーが必要としている機能だけを、AIがその都度引き出して配置する。**

*   **実装仕様**: 常時表示される巨大なサイドバーや多階層メニューを廃止。ユーザーがドキュメントを執筆しているときは「推敲・要約ツール」が、コードを見ているときは「デバッグ・デプロイボタン」が、デザインを調整しているときは「カラーパレットジェネレーター」が、AIによってステージ上に動的にスライドイン（motion）する。

### Rule 09: 超インテリジェント入力補完 (Frictionless Autocomplete)
> **入力を減らせ。ユーザーがキーボードを叩く労力を、AIが徹底的に補完・代行する。**

*   **実装仕様**: 面倒なフォーム入力、パラメータ設定はAIがこれまでの会話履歴（AI Memory）とドキュメントコンテキストから自動推測してプリセット（Pre-fill）。ユーザーは自動入力された項目を「確認（Approve）」するだけで済むように設計する。

### Rule 10: 自律エージェントの代行 (Autonomous Operations)
> **ユーザーはAIを「操作」しない。AIが、ユーザーに代わってWorkspace全体を縦横無尽に「操作」する。**

*   **実装仕様**: ユーザーが指示を1つ出せば、AIが裏側のデータベースを更新し、必要なファイルを生成し、外部APIを叩き、デプロイメントを完了させる。ユーザーは操作のオペレーターではなく、実行指示を与える「最高意思決定者（Commander）」の地位に留まる。

### Rule 11: 待ち時間を「安心・信頼」の時間へ (Dynamic Progress Reassurance)
> **進捗の途中経過を美しく見せることで、処理を待つ「退屈な時間」を、システムへの深い「安心・信頼の時間」へ変える。**

*   **実装仕様**: AIが処理を行っている数十秒の間、画面をフリーズさせたり無味乾燥なスピナーを表示したりしない。「現在、3つの主要モデルがマーケティング戦略の妥当性をディベート中...（合意形成率: 87%）」といった、知的で脈動するタイムライン進捗をリアルタイムにアニメーション表示し、期待感を高める。

### Rule 12: 迷子にさせない「次のアクション」 (The Next Clear Step)
> **ユーザーは、質問や作業を1回終えるたびに、自分が次に何をすべきか（Next Action）が常に1秒で分かる。**

*   **実装仕様**: 成果物出力の末尾には、必ず「次にやること（Actionable Next Steps）」を優先度順に並べた3つのスマートボタンを配備。「このLPを公開する」「SEO対策キーワードを追加する」「PDFとしてエクスポートする」など、思考を止めることなく次の成功行動へとユーザーをエスコートする。

### Rule 13: 1年先を読む未来提案 (Horizon Proactive Prediction)
> **今この瞬間の課題解決に留まらない。AIは、ユーザーの半年後、一年後の未来に必要な戦略を常に提案する。**

*   **実装仕様**: 単発のTODOリスト作成タスクが完了した際、「このタスク完了パターンから、数ヶ月後にはチームメンバーの増員に伴う権限管理が必要になります。今から、拡張可能なエージェント管理ワークフローのドラフトを用意しましょうか？」といった、成長ストーリーに基づく未来提案（Prediction Deck）を自動提示する。

### Rule 14: 統一された全画面UX (Universal UI Harmony)
> **アプリケーションのどの機能、どのビュー、どのデバイスでも、完全に統一された美学と使いやすさを維持する。**

*   **実装仕様**: チャット、ダッシュボード、成果物プレビュー、プロジェクトレジャーなど、画面の役割が変わっても、共通のデザイントークン（角丸16px、インディゴのアクセント、等幅フォント、 motionの滑らかなイージング）を徹底。操作における不連続性をゼロにする。

### Rule 15: 世界一使いやすいAIシステム (The Ultimate Interface)
> **AI Workspaceは、あらゆる類似システムの中で、世界一シンプルで、世界一早く、世界一圧倒的な成果を出すAIになる。**

*   **実装仕様**: すべてのモジュールのロード時間は100ms未満、AIレスポンスはミリ秒でストリーミング開始、操作ステップ数は競合システムの3分の1以下。圧倒的な人間中心設計によって、すべてのユーザーをビジネスの成功へと最短距離で導く。

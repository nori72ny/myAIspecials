# AI Workspace 「AI App Builder」機能設計書 (AI App Builder Specification)

本書は、AI Workspaceに統合されるアドバンスド機能**「AI App Builder（AIアプリビルダー）」**のシステムアーキテクチャ、機能フロー、画面設計、およびデータ構造を定義したものです。
ユーザーが自然言語で要件を入力するだけで、ビジネス向けのWebアプリ、管理画面、業務システム、AIツールなどを自律的にコード生成・プレビュー・エクスポートできる環境を構築します。

---

## 1. システム・コンセプト ＆ 技術スタック

### 💡 コンセプト
**「要件定義からデプロイ可能なコード生成まで、5分で完結するノーコード・ローコード開発エンジン」**
ただのテキストとしてのコード生成ではなく、SPA（Single Page Application）として稼働可能なマルチコンポーネント構造のフロントエンドコードと、Supabase等のBaaS（Backend as a Service）連携ロジックを統合した「すぐに動く・改修できる」成果物を提供するコンパイラ・ジェネレータ。

### 💻 生成対象の対応技術スタック
*   **Frontend**: React (Vite / Next.js App Router), TypeScript
*   **Styling**: Tailwind CSS (独自テーマ、ダークモード、モダンパレット対応)
*   **Component**: Lucide React Icons, Headless UI (Radix / Radix UI 互換スタイル)
*   **Backend / Database**: Supabase Integration (JS Client, テーブル定義SQL, Auth接続設定)

## 2. 業務・生成ライフサイクルフロー

AI App Builderが起動してから、ユーザーがアプリケーションを完成させてエクスポートするまでのライフサイクルです。

```
[1. ユーザー入力・アプリ構成設定]
      │
      ▼
[2. AI要件整理 (Requirement Clarifier)] ── (要件定義書、画面一覧、DB/API設計の自動生成)
      │
      ├─► [ユーザー修正・追加指示] ──┐
      │ (承認)                       │
      ▼                             ▼
[3. AI画面構成・UIコンポーネント設計] (Dashboard, CRUD, Login, 詳細ビュー等のコンポーネントマップ化)
      │
      ▼
[4. AIコード自動生成 (Coder Agent)] ── (React / TS / Tailwind コードの自律生成)
      │
      ▼
[5. QA & 構文チェック (QA Agent)] ──── (TypeScriptエラーやインポート漏れの自律検知・自動リテイク)
      │
      ▼
[6. プレビュー＆インタラクティブエディタ] ─► [再編集指示] ─► [修正コードの差分生成]
      │
      ▼
[7. 成果物出力 ＆ 保存] ──────────────── (ZIPダウンロード、コードコピー、プロジェクト保存)
```

## 4. データモデル・スキーマ設計 (Data Model Schema)

AI App Builderのプロジェクト情報、生成されたコード、ファイルツリー、およびバージョン履歴をセキュアに保持するデータモデルです。

```
                  +-------------------------+
                  |    builder_projects     |
                  +-------------------------+
                  | id (PK)                 |
                  | name                    |
                  | description             |
                  | tech_stack_config (JSON)|
                  | current_version         |
                  | created_at              |
                  | updated_at              |
                  +------------+------------+
                               | 1
                               |
                               | 1..N
                  +------------v------------+
                  |    project_versions     |
                  +-------------------------+
                  | id (PK)                 |
                  | project_id (FK)         |
                  | version_number          |
                  | commit_message          |
                  | specification_doc (TEXT)|
                  | created_at              |
                  +------------+------------+
                               | 1
                               |
                               | 1..N
                  +------------v------------+
                  |      project_files      |
                  +-------------------------+
                  | id (PK)                 |
                  | version_id (FK)         |
                  | file_path               |
                  | content                 |
                  | is_binary               |
                  | file_type (EXT)         |
                  | updated_at              |
                  +-------------------------+
```

### テーブル定義

#### ① `builder_projects` (アプリ開発プロジェクト)
AI App Builderで構築するアプリケーション全体のメタ情報を保持。
*   `id`: UUID (Primary Key)
*   `name`: VARCHAR(255) (アプリ名。例：「CRM Lite」)
*   `description`: TEXT (アプリ概要)
*   `tech_stack_config`: JSON (技術定義。例: `{"framework": "vite-react", "styling": "tailwind", "backend": "supabase"}`)
*   `current_version`: INT (現在の最新バージョン。例：`4`)
*   `created_at` / `updated_at`: TIMESTAMP

#### ② `project_versions` (プロジェクト・バージョン履歴)
バージョン管理、ロールバックを可能にするためのスナップショット情報。
*   `id`: UUID (Primary Key)
*   `project_id`: UUID (Foreign Key)
*   `version_number`: INT (バージョン番号。例: `1`, `2`, `3`)
*   `commit_message`: VARCHAR(512) (変更要約。例：「Dashboardに簡易売上集計グラフを追加」)
*   `specification_doc`: TEXT (要件定義書、画面設計、DBスキーマのAI生成ドキュメント)
*   `created_at`: TIMESTAMP

#### ③ `project_files` (生成ファイル詳細)
各バージョンにおける具体的なソースコードおよび構成ファイルの実体。
*   `id`: UUID (Primary Key)
*   `version_id`: UUID (Foreign Key)
*   `file_path`: VARCHAR(512) (ファイルパス。例: `src/components/Dashboard.tsx`, `supabase/schema.sql`)
*   `content`: MEDIUMTEXT (ソースコード、構成設定、SQL定義の実体)
*   `is_binary`: BOOLEAN (アセット等のバイナリ判定。通常はfalse)
*   `file_type`: VARCHAR(50) (拡張子。例：`tsx`, `ts`, `css`, `sql`, `json`)
*   `updated_at`: TIMESTAMP

## 5. 生成・協調 AI エージェント構成 (Generative Engine Multi-Agent)

App Builderの裏側で働く、役割に特化した自律型エージェントのチーム構成です。

1.  **Spec Architect Agent (仕様策定エージェント)**:
    *   **モデル候補**: Claude 3.5 Sonnet / Gemini Pro
    *   **役割**: ユーザーの自然言語プロンプトから、構造化された「画面構成案、コンポーネント構造、必要なデータベース/API定義」をMarkdownおよびJSONスキーマとして策定する。
2.  **Code Generator Agent (コード実装エージェント)**:
    *   **モデル候補**: Claude 3.5 Sonnet / DeepSeek Coder
    *   **役割**: 仕様策定エージェントが作成した設計図を基に、エラーのない完全なTypeScript Reactコンポーネント、CSS設定、Supabaseクライアントコード、およびテーブル作成用のSQLコードを生成・出力する。
3.  **UI/UX Aesthetic Auditor (ビジュアル/UI調整エージェント)**:
    *   **モデル候補**: Gemini (マルチモーダル) / Claude
    *   **役割**: 生成されたTailwind CSSのレイアウトやカラーパレット、レスポンシブデザイン（モバイル・デスクトップ対応）がビジネス基準（見やすさ、一貫性、美しさ）に適合しているかをチェックし、必要な微修正コードを適用する。
4.  **Syntax & QA Agent (構文検証エージェント)**:
    *   **モデル候補**: DeepSeek V3 / GPT-4o-mini
    *   **役割**: 生成されたファイルの静的解析、インポート文（例: `lucide-react` からのインポートが実際に存在するアイコンか等）の正確性を検証し、エラーを発見した場合はCode Generatorへ自動的にフィードバックを返し、コンパイル可能な状態に修復する。

## 6. 特徴的な自律機能

### ① インタラクティブ・フィードバック (チャットによる再編集)
一度生成されたアプリのプレビューを確認したユーザーが、「売上グラフを青から緑に変更して」「顧客追加フォームにメールアドレスのバリデーションをつけて」と自然言語で追加指示を出すと、エージェントがその差分のみを認識し、関連するファイル（例: `src/components/Dashboard.tsx` と `src/components/Customer.tsx`）の特定部分のみを安全に書き換え、バージョンを更新（Ver 4 ➔ Ver 5）します。

### ② すぐに使えるBaaS/Supabase連携定義の自動生成
モックUIを作るだけでなく、実際に稼働させるための「データベーススキーマ（SQL）」と、フロントエンドからデータをフェッチ/CRUDするための「Supabase JSクライアント記述」を完全自動で生成します。これにより、ユーザーはSupabaseのコンソールにSQLを貼り付けるだけで、本物の認証付きデータベースと接続した実用システムを完成させることができます。

# AI Workspace プロダクト設計書 (Product Specification Document)

本書は、法律相談、弁護士関連、医療判断、投資助言などの非推奨・高リスク領域を徹底的に排除し、ビジネスの実務効率化・調査・分析・資料作成・アプリ生成・業務自動化に完全特化した「AI Workspace」の最終的なサービス仕様およびアーキテクチャ設計を定義するものです。

---

## 1. プロダクトビジョンとスコープ定義

### 💡 コンセプト
**「ビジネスパーソンのための、自律型AIエージェント協調ワークスペース」**
単なる一問一答のAIチャットではなく、複雑な業務指示（競合分析、提案書作成、Webプロトタイプ作成など）をAIが自動でタスク分解し、適材適所のマルチエージェントと最適ルーティングによって高品質な成果物を自律生成・管理・再利用するプラットフォーム。

### 🚫 徹底除外スコープ（セキュリティ・コンプライアンス遵守）
以下の領域はシステムから完全に除外され、入力制限（システムプロンプトおよびフィルター）によって対応不可とします。
*   **法律相談・交通事故・弁護士関連業務**（非弁活動防止、法的リスクの排除）
*   **医療判断・ヘルスケア診断**（医師法・薬機法への抵触防止、身体への健康被害リスクの排除）
*   **投資助言・金融アドバイス**（金融商品取引法への抵触防止、経済的損失リスクの排除）

###  重点特化スコープ
1.  **検索＆調査**: 最新情報の高速クローリング、信頼ソースの比較、ファクトチェック。
2.  **ビジネス分析**: 3C/SWOT等のフレームワーク分析、AIO（AI Search Optimization）分析、SEO分析、市場/競合分析。
3.  **ドキュメント・資料作成**: 営業提案書構成、ホワイトペーパー、レポート、ブログ・PR記事、PowerPoint/Slide原稿。
4.  **プロトタイプ・アプリ生成**: HTML/JS/CSSによるWebサイト・LP、社内用AIツール、簡易管理画面UIなどのコードおよび表示可能モックアップの生成。
5.  **自律型ワークフロー自動化**: ユーザーが定義したタスクの分解、複数AIの自律連携、実行ログの追跡、成果物管理。

## 3. 機能一覧 (Feature Matrix)

| カテゴリ | 機能名 | 詳細・仕様 |
| :--- | :--- | :--- |
| **ユーザーインターフェース** | ワークフローダッシュボード | タスク進行状況、成果物、複数エージェントの動的な対話や出力を可視化する統合画面。 |
| | プロジェクト・フォルダ管理 | 成果物を「営業」「マーケティング」「開発」「個人」などのフォルダで分類・保存。 |
| **ワークフローエンジン** | タスク自動分解 (Decomposer) | 1つの大きな依頼（例: 「新製品のLPを作って」）から、調査・構成・執筆・実装のタスクを自動生成。 |
| | 中間成果物レビュー＆承認 | 各タスク完了時にユーザーへレビュー（FB/リトライ/承認）を促し、プロセスの品質を担保。 |
| | 履歴・バージョン管理 | 生成された成果物の履歴を保存（Ver 1, Ver 2, Ver 3...）。いつでも過去の世代へロールバック可能。 |
| **AIルーティング** | Dynamic AI Router | コスト、速度、得意分野に基づき、タスクごとに最適なLLM（Gemini, Claude, GPT, Perplexity）を自動割当。 |
| | OpenRouter & 直接API接続 | OpenRouterによる統一APIアクセスに加え、個別プロバイダーへの直接接続による冗長化・障害耐性を確保。 |
| **AIエージェント** | 6大専門協調エージェント | Planner, Research, Analyst, Writer, Developer, QA。各専門特化型システムプロンプトとツールを装備。 |
| **成果物出力・再利用** | マルチ形式エクスポート | 成果物の種類に応じ、Markdown、PDF/Word（テキスト系）、HTML/ZIP（コード系）として出力。 |
| | テンプレート化 | 過去の成功した出力やワークフローをテンプレート登録し、新規プロジェクトのベースとして再利用。 |

## 4. 画面一覧 (Screen Map)

1.  **メインダッシュボード (Workspace Home)**
    *   現在進行中のプロジェクト（進行状況、実行エージェントのステータス）の一覧。
    *   新規依頼のクイック入力エリア（自然言語での依頼フォーム）。
    *   最近作成した成果物、テンプレートへのショートカット。
2.  **ワークフロー詳細 / エージェント実行画面 (Interactive Pipeline)**
    *   タスク分解ツリー（ガントチャート風、または垂直ステップリスト）。
    *   エージェント間の「会話ログ」および「思考プロセス」のリアルタイム表示。
    *   中間成果物のライブプレビューエリア（Markdownプレビュー、HTML/CSSプレビュー、ドキュメントドラフト）。
3.  **成果物ライブラリ / バージョンエディター (Asset & History Library)**
    *   成果物リストとフィルタ・検索。
    *   左右分割でのバージョン比較画面（例: Ver 1 と Ver 2 の差分確認）。
    *   再利用・テンプレート化ボタン、エクスポートメニュー。
4.  **テンプレート＆システム設定 (Templates & Engine Config)**
    *   定型ワークフロー（「競合分析パッケージ」「SEOレポート作成」等）の管理。
    *   AIルーターの重み付け設定（「速度重視」「品質重視」「コスト重視」の切り替え）。

## 5. ユーザーフロー (User Flow Sequence)

```
[1. ユーザーの依頼入力]
      │
      ▼
[2. Planner Agentがタスク分解] ── (進捗ツリー・必要なエージェントを自動定義)
      │
      ▼
[3. 各タスクの実行 (並列・直列)]
      ├─ Research Agent: 最新情報・競合情報のクローリング (Perplexity等)
      ├─ Analyst Agent: データの構造化・SWOT/AIO分析
      └─ Writer/Developer: ドキュメント起草 (Claude等) または Webプロトタイプ生成
      │
      ▼
[4. QA Agentによる検証・推敲] ── (不整合の検知、修正、品質チェック)
      │
      ▼
[5. 中間レビュー (ユーザー確認)] ─► [修正指示あり] ─► [ステップ3へ差し戻し]
      │ (承認)
      ▼
[6. 最終成果物生成 ＆ 保存] ──── (プロジェクト・フォルダへの登録、バージョン確定)
      │
      ▼
[7. エクスポート / テンプレート化] (PDF, ZIP, Word出力 / 次回の再利用設定)
```

## 6. AIルーティング構成 (AI Routing Engine Architecture)

効率的かつ最高品質のアウトプットを得るため、ルーティングエンジンは以下のポリシーで駆動します。

### 📊 モデルの専門能力マッピング

| モデル | 主な役割 | 強み / ルーティング判定基準 |
| :--- | :--- | :--- |
| **Perplexity / Genspark** | 検索・ファクトチェック | 最新トレンド、競合製品の現行価格、Web上の公開データの即時収集。 |
| **Claude 3.5 Sonnet** | 資料作成・高度な分析・コード | 複雑なコンテキストの理解、自然で高品位な日本語ドキュメント、正確なHTML/JS生成。 |
| **Gemini 2.5 Flash / Pro** | Planner・全体推論・マルチモーダル | 超長文コンテキスト処理（リファレンス資料の一括読み込み）、タスクの高速自動分解。 |
| **DeepSeek (Coder/V3)** | コーディング・アルゴリズム | 開発・QA工程でのコードのロジック精査、最適化。 |
| **GPT-4o / gpt-4o-mini** | 汎用処理・高速並列ドラフト | 中間成果物の部分要約、軽量なテキスト整形、コストパフォーマンスに優れた並列バッチ処理。 |

### 🛠️ ルーティングロジック & フォールバック
1.  **インテント抽出**: ユーザー指示から「検索」「ドキュメント作成」「コード生成」等のインテント（意図）を判定。
2.  **最適割り当て**: インテントマップに基づき一次モデルを選定。
3.  **フォールバック**:
    *   APIタイムアウトやRate Limit（RESOURCE_EXHAUSTED）検知時、直ちに別プロバイダー（例: GeminiからClaude、またはOpenRouterから直接API）へ切り替え。
    *   レスポンス内容が空、またはエラーコード（5xx等）の場合、リトライと代替モデルへのルーティングを自動実行。

## 7. AIエージェント構成 (AI Agent Orchestration)

自律型ワークフローを支える6つの専門エージェントの定義です。

1.  **Planner Agent (プランナー・エージェント)**
    *   **役割**: ユーザーの曖昧な指示を明確な中間成果物とタスク群に落とし込み、ワークフローを設計する。
    *   **特徴**: ゴールの定義と各タスクへの入力/出力フォーマットの厳密な定義を行う。
2.  **Research Agent (リサーチャー・エージェント)**
    *   **役割**: Webから最新のソース、統計、競合他社の動きを抽出し、信頼性を評価する。
    *   **特徴**: 検索APIを駆使し、参照URLを常にソース情報として紐づけて分析フェーズへ引き渡す。
3.  **Analyst Agent (アナリスト・エージェント)**
    *   **役割**: 調査データを構造化し、フレームワーク（SWOT、3C、AIO、SEO）に沿って評価、インサイトを導く。
    *   **特徴**: 定量的・定性的なデータを掛け合わせた戦略の要約を作成する。
4.  **Writer Agent (ライター・エージェント)**
    *   **役割**: アナリストの洞察を基に、人間が読みやすく説得力のあるビジネスドキュメント、ブログ、スライド原稿を作成する。
    *   **特徴**: トーン＆マナー（丁寧、先進的、硬派など）を自在にコントロールする。
5.  **Developer Agent (デベロッパー・エージェント)**
    *   **役割**: Webプロトタイプ、簡易ツール、管理画面などのUI（HTML, CSS, Tailwind, JavaScript）を実装する。
    *   **特徴**: インタラクティブで、その場ですぐに動作確認可能なUIコードを生成する。
6.  **QA Agent (品質管理・エージェント)**
    *   **役割**: 各エージェントのアウトプットを相互検証し、情報の抜け漏れ、破綻、誤り、および除外スコープ（法律、医療、投資等）への抵触がないかを厳格に検査する。
    *   **特徴**: 基準を満たさない成果物を検知した場合、該当エージェントに理由を添えて再生成（リテイク）を命令する。

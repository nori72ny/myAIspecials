# AI Workspace 「自律意思決定・多角比較評価エンジン（Decision Intelligence Engine）」機能設計仕様書

本書は、AI Workspace全体における複雑な意思決定（導入するAIモデルの選定、外部SaaSサービスの比較、開発ツールの採否、提携ベンダー会社の評価など）に対し、AIが複数の候補案を「価格・品質・速度・ROI・リスク・将来性・運用性」の7大次元からセマンティックかつ数理的に比較分析し、最適な意思決定（最適解・ランキング・総合スコア）およびその「理由（Chain-of-Thought）」を導き出す自律推論システム**「Decision Intelligence Engine（意思決定インテリジェンス・エンジン）」**のシステム構成、データスキーマ、評価・意思決定アルゴリズム、多次元シミュレーション、リスクモデル、および画面設計を定義した製品設計仕様書です。

本システムは、勘や不完全な情報に頼る人間の不確実な判断を排し、データ駆動で「最も成功確率が高く、最もリスクが低い選択肢」を数学的に推薦。承認後はReality EngineやProject Managerをキックして現実世界へ即座にデプロイする、AI Workspace全体の**「自律最高財務・戦略責任者（Autonomous CFO/CSO Engine）」**の役割を果たします。

## 1. システム概要 ＆ 意思決定コンセプト (Overview & Concept)

Decision Intelligence Engineは、外部データ（Connector Hub）やナレッジ（Knowledge Graph）、実データ（BI Center）を横断スキャンし、比較検討が必要な「アセット（AI、サービス、会社、ツールなど）」の情報（価格、機能、仕様）を自動ハイドレーション。
これらを「7大評価次元」に基づき多角評価し、導入前後の「シミュレーション予測」と「多重リスク監査」を行った上で、最終決定（Decision）をオーナーおよびデジタルツインに提示します。

```
     [ 🌐 比較アセット候補群 (AI models, SaaS Services, Vendors, Tools) ]
                                 │
                                 ▼ [Connector Hub ✕ Web検索]
     ===================================================================
                       [ Decision Intelligence Engine ]
     ===================================================================
     ├─ ① 多次元セマンティック比較器 (Compare)
     │    - AI、サービス、会社、ツール、価格、機能、規約のセマンティック抽出
     ├─ ② 7大ディメンション評価器 (Evaluation)
     │    - [価格] [品質] [速度] [ROI] [リスク] [将来性] [運用性] の重み付け計算
     ├─ ③ 統計的・LLM未来シミュレータ (Simulation)
     │    - 導入前予測、改善予測、売上予測、システム利用率予測
     ├─ ④ 自律リスク・ディフェンダー (Risk Audit)
     │    - コスト、保守性、障害確率、依存性、API利用リミットの事前監査
     └─ ⑤ 統合ランキング・最適解デシジョン (Decision Engine)
          - 総合スコア算出、最適解の選定、推論理由（CoT）の可視化
                                 │
                     ┌───────────┴───────────┐
                     ▼ (承認後の自律アクション)    ▼ (脳内アライメントと分析)
        【🔌 Reality Engine (現実実行)】          【📊 AI BI Center / Twin】
         - 採択したツールのAPIキー/OAuth自動接続    - 採用した決定による売上・利用率への
         - 契約PDFの自動生成・保存、下書き送付         実績相関データの可視化
         - PMへの導入プロジェクト自動配備         - デジタルツインの判断バイアスの学習
```

### テーブル定義詳細

#### ① `decision_boards` (意思決定ボードマスタ)
比較検討のテーマ、カテゴリ、ステータス、AIが導き出した最適解と確信度、および論理的推論を統括。
*   `id`: UUID (Primary Key)
*   `company_id`: UUID (Foreign Key - 組織スペースのID)
*   `title`: VARCHAR(255) (意思決定テーマ。例: `「新宿店MEO解析SaaSの選定」`)
*   `category`: VARCHAR(50) (分類。`AI` (モデル・プロバイダ), `SERVICE` (SaaS/外部サービス), `COMPANY` (契約・外注会社), `TOOL` (ライブラリ/ソフトウェア), `HARDWARE` (物理機器))
*   `status`: VARCHAR(50) (ステータス。`PENDING` (検討・アライン中), `ADOPTED` (意思決定採用済み), `ARCHIVED` (見送り・書庫化))
*   `best_candidate_id`: UUID (Foreign Key, Nullable - AIが算定した最適候補アセット `decision_candidates` のID)
*   `confidence_score`: DECIMAL(5, 2) (意思決定に対するAIの確信度。0.00〜100.00 %)
*   `reasoning_cot`: TEXT (最適解に至った、7大評価とシミュレーションを統合した Chain-of-Thought 推論文章)
*   `created_at` / `updated_at`: TIMESTAMP

#### ② `decision_candidates` (比較候補アセットテーブル)
ボードに登録された個別の比較対象（例: Option A (MEO-Lite), Option B (MEO-Pro)）。
*   `id`: UUID (Primary Key)
*   `board_id`: UUID (Foreign Key)
*   `name`: VARCHAR(255) (候補名。例: `「MEO-Pro Cloud」`)
*   `price_raw`: DECIMAL(15, 4) (初期費用または月額の基本価格)
*   `features_json`: JSON (製品仕様・提供機能のリスト)
*   `website_url`: VARCHAR(1024) (API仕様書、または公式サイトのURL)
*   `metadata_json`: JSON (ベンダー会社情報、サポート規約、技術スタック等)
*   `created_at`: TIMESTAMP

#### ③ `decision_evaluations` (7大次元評価スコアテーブル)
候補アセット毎の「価格、品質、速度、ROI、リスク、将来性、運用性」の個別評価スコア。
*   `id`: UUID (Primary Key)
*   `candidate_id`: UUID (Foreign Key)
*   `dimension`: VARCHAR(50) (評価次元。`PRICE` (価格適合), `QUALITY` (品質・機能充足度), `SPEED` (処理・レスポンス速度), `ROI` (費用対効果), `RISK` (安全リスク), `FUTURE` (将来性・技術ロードマップ), `MAINT` (運用保守性))
*   `score`: DECIMAL(5, 2) (100点満点の評価スコア。AIがセマンティックに算定。数値が高いほどポジティブ評価、ただしRISKのみ数値が低いほど安全)
*   `updated_at`: TIMESTAMP

#### ④ `decision_simulations` (導入シミュレーション予測テーブル)
該当候補を採用した場合の、売上成長やシステム利用効率などの未来予測値。
*   `id`: UUID (Primary Key)
*   `candidate_id`: UUID (Foreign Key)
*   `metric_name`: VARCHAR(100) (予測指標。例: `sales_revenue_growth`, `system_utilization_rate`, `task_speedup_percent`)
*   `forecast_value`: DECIMAL(15, 4) (AI Prophet / LSTM モデルが算出した予測値)
*   `confidence_low` / `confidence_high`: DECIMAL(15, 4) (予測値の95%信頼区間の境界バンド値)
*   `created_at`: TIMESTAMP

#### ⑤ `decision_risks` (リスク自己監査テーブル)
候補アセットに潜む、具体的なリスク要素とマイナススコアの記録。
*   `id`: UUID (Primary Key)
*   `candidate_id`: UUID (Foreign Key)
*   `risk_factor`: VARCHAR(50) (リスク種別。`COST` (隠れコスト、従量急増), `MAINT` (保守負担、ブラックボックス化), `FAILURE` (外部API障害、サービス終了), `DEPENDENCY` (ロックイン、他システム衝突), `API_LIMIT` (Quota制限、トークン枯渇))
*   `risk_score`: DECIMAL(5, 2) (100点満点のリスク深刻度。数値が高いほど危険)
*   `description`: TEXT (具体的な懸念材料、および推奨される回避・代替策の文章)
*   `created_at`: TIMESTAMP

#### ⑥ `decision_history_adoptions` (意思決定・採用監査ログ)
どのボードでどの候補が、いつ、誰によって採用され、その後の実際のROIがどうであったかを記録するAppend-Onlyな監査台帳。
*   `id`: UUID (Primary Key)
*   `board_id`: UUID (Foreign Key)
*   `selected_candidate_id`: UUID (Foreign Key - 最終採用されたアセット候補のID)
*   `decided_by`: VARCHAR(50) (決定主体。`MANUAL` (人間による手動承認), `TWIN` (デジタルツインによる自律執行))
*   `actual_roi_measured`: DECIMAL(10, 4) (Nullable - 採用から30日以上経過後に BI Center から逆ハイドレーションされた実測ROI。予測ROIの答え合わせ用)
*   `executed_at`: TIMESTAMP (採用決定が下され、Reality Engineで外部接続等の物理パブリッシュが開始された日時)

## 4. 意思決定 ＆ 評価・リスク監査ロジック (Decision Engineering)

Decision Intelligence Engineが、ハルシネーション（憶測・お世辞）を完全に排除し、候補のデータを多角スキャンして数学的・論理的に最適解を導き出す4段階の推論パイプライン。

```
       [ 1. セマンティック・データハイドレーション（Connector ✕ Search）]
         - 登録された候補アセット（Option A, B, C）のAPI、URL、仕様ドキュメントを
           Google Search Grounding を用いて最新情報をクローリング。
         - 利用料金表、利用規約（SLA）、提供機能リストをJSON構造化抽出。
                            │
                            ▼
       [ 2. 7大ディメンション多角スコアリング（Weighted Scoring）]
         - 各候補の仕様データから、7大評価軸のスコア $S_d \in [0, 100]$ をAIが多角算定。
         - オーナー（デジタルツイン）の基本価値観（VALUES）から、各軸の重要度重み $W_d$ を抽出。
           例: 品質優先オーナーの場合 ➔ $W_{quality} = 0.35$、価格重み $W_{price} = 0.05$
         - 総合評価点算出: $$Score_{total} = \sum_{d=1}^{7} W_d \times S_d$$
                            │
                            ▼
       [ 3. 統計的未来シミュレーション ✕ 多重リスク監査 ]
         - Prophet 時系列モデルによる、採用後の「売上成長（+¥170,000）」の確率空間予測。
         - リスクファクター（コスト、保守、障害、依存、API）を最大 $100$ スコアで自己監査。
         - リスクペナルティ関数を総合スコアに減算適用。
                            │
                            ▼
       [ 4. 意思決定の確定 ＆ 推論の構造化（Chain-of-Thought）]
         - 最高総合スコアの候補を最適解（Best Option）として選定。
         - 最適解の選定プロセスを、多次元データをアンカーとした「厳格なCoT文章」としてデータベースへ保存。
         - 確信度（Confidence Score）を算出し、自律承認または人間への承認要請を発行。
```

### ① 評価の重み付けアラインメント（Dynamic Weighted Evaluation）
経営者の価値観によって、同じツールであっても「最適解」は変わります。Decision Intelligence Engineは、デジタルツインのプロファイル特性（`twin_profiles`）から、動的に重み付けパラメーター $W_d$ をハイドレーションします。
*   **スピード重視・スタートアップモード（Speed First）**:
    $$W_{speed} = 0.30, \quad W_{roi} = 0.25, \quad W_{quality} = 0.20, \quad W_{price} = 0.05, \quad W_{risk} = 0.05$$
*   **完璧主義・品質・堅牢モード（Quality & Robustness）**:
    $$W_{quality} = 0.35, \quad W_{risk} = 0.25 \text{ (低リスクを高く評価)}, \quad W_{maint} = 0.20, \quad W_{speed} = 0.15, \quad W_{price} = 0.05$$

### ② 導入後効果の統計的シミュレーション（What-If Simulation Algorithms）
*   **売上・利用率のインクリメンタル予測 (Incremental Revenue Projection)**:
    採用候補アセットがマーケティング（MEO順位平均向上）またはシステム効率（Workflow成功率向上）に直接関与する場合、BI Centerの実績時系列データを特徴量 $X_t$ とした **Prophet ✕ LSTM ハイブリッド時系列モデル** をシミュレータ上で実行。
    「Option Bを採用した場合、MEO順位が平均2.4位から平均1.8位へ向上する確率が 88% あり、これに伴い月間売上は $13.6\%$（¥170,000）引き上げられ、投資対効果（ROI）は $320\%$ に達する」という確率空間カーブを Recharts でシェード描画します。

### ③ 多重リスクペナルティ評価（Risk Penalty Function）
リスクは、総合スコアに対して以下のような減衰指数ペナルティ $P_{risk}$ として適用され、高リスクアセットの採用を自動的に警告・ブロックします。
$$Score_{final} = Score_{total} \times e^{-\lambda \times Risk_{score}}$$
*   $Risk_{score} \in [0, 100]$: 5大リスクファクター（コスト、保守、障害、依存、API）の最大値または重み付け和。
*   $\lambda = 0.005$: リスクペナルティ減衰定数。
これにより、どれほど機能が優秀（$Score_{total} = 95$）であっても、障害リスクが極めて高い（$Risk_{score} = 80$）ツールは、最終スコアが大きくペナルティ減算され、安全で堅牢なOptionが上位にランキングされるよう自律制御されます。

## 5. 他システム（OS、ツイン、改善、リアリティ、コネクタ、BI、PM）との統合

Decision Intelligence Engineは、AI Workspaceにおける「経営会議の合意・意思決定中枢（Autonomous CSO Control Center）」として、データ収集から比較、意思決定、そして物理実行への伝播を完璧に統合・自動化します。

1.  **Connector Hub 連携 (データ供給のライフライン)**
    *   比較候補アセット（SaaS、AIモデル、ツールなど）の生の価格、機能、利用規約のデータを、Connector Hubが外部プロバイダや検索エンジン（Google Search Grounding）を介してリアルタイム収集。ハッシュ比較による「仕様変更の自動検知」も行います。
2.  **Digital Twin Core 連携 (人間とAIの思考同調)**
    *   経営者の「判断傾向」「品質・価格のバランスの癖（バイアス）」をツインコアの `twin_profiles`（価値観、判断傾向）から同期。
    *   「もしオーナーなら、この3案のうちどれを採択するか」を高精度（思考一致率 95.2%）に予測。一致率が規定値以上の領域では、低コストの意思決定をデジタルツインが自律的に代理執行（Auto Proxy）します。
3.  **AI BI Center 連携 (予測精度とROIの実測監査)**
    *   意思決定採用前後の、売上実績、カレンダー稼働、AIコスト、MEO順位の推移を BI Center から逆ハイドレーション。
    *   採用から30日後に「事前の予測ROI（例: 320%）と、実際の効果実測値（例: 350%）」の答え合わせ（ROI Answer Checking）を自動実行。
    *   予測と実績の乖離（予測ドリフト）を検知した場合、意思決定AIのシミュレーション係数やデジタルツインの重み付けを自律微調整（Self-Refining Loop）します。
4.  **Reality Engine 連携 (意思決定の瞬間デプロイ)**
    *   人間またはデジタルツインが「このツール（Option B）を採用する」と意思決定した瞬間、Reality Engineが起動。
    *   採用したツールのOAuth認可セッションの確立（ポップアップメッセージング）、APIトークンのセキュアシークレット保管を全自動執行。
    *   同時に、稟議用・記録用の「意思決定評価報告書（PDF）」や、ベンダーへのメール送信用下書きを動的自動生成し保存します。
5.  **AI Project Manager / Goal Engine 連携 (導入推進)**
    *   ツールの採用が決定すると、AI Project Managerが自動的に「新規MEOツールのマージ・検証プロジェクト」を起案し、カンバンボードへタスクを自動アサイン配備。
    *   Goal Engineに設定されたビジネス目標（売上目標、集客目標）の進捗に、この意思決定がどう貢献したかを直接結びつけてマッピング追跡します。

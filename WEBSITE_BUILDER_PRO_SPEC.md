# AI Workspace 「Website Builder Pro」機能設計仕様書

本書は、AI Workspaceに統合される、ビジネス用のWebサイト構築・自律集客最適化エンジン**「Website Builder Pro」**のシステム構成、データベース設計、SEO/AIO/MEO最適化メカニズム、および画面設計を定義した製品仕様設計書です。

ユーザーが「作りたいサイト内容、業種、ターゲット、地域、競合、目的」を自然言語で入力するだけで、AIが市場・SEO・AIO・MEOの事前分析を自律実行し、戦略的なサイトマップと高コンバージョンな構成案（コピーライティング含む）を定義。モダンなフロントエンドフレームワーク（Next.js, Astro, React等）をベースにしたソースコードを自動生成し、さらに公開前の各種診断（速度・SEO・アクセシビリティ）まで一気通貫で自動化します。

---

## 1. 業務フロー ＆ コンセプト (Overview & Concept)

Website Builder Proは、単に美しいWebサイトのビジュアル（CSS/HTML）を作るだけでなく、「検索エンジン（SEO）」「AI検索エンジン（AIO）」「地図・地域検索（MEO）」すべてに最初から圧倒的に最適化され、実ビジネスの顧客流入を最大化するための集客コンサルティング・制作統合プラットフォームです。

```
[1. サイト構築要件の入力] ──► (業種、ターゲット、地域、競合、目的、デザインスタイル指定)
              │
              ▼
[2. AI自律多重分析 (AI Multi-Analysis)]
              ├─► 市場・競合分析: 競合サイトの訴求軸と強みのギャップ分析
              ├─► SEO分析: 狙うべき検索ボリューム、お宝キーワードの抽出
              ├─► AIO分析: AI検索（Gemini, Perplexity等）に引用・参照されるコンテンツ構成案定義
              └─► MEO分析: 地域名×キーワードによるGoogleマップ露出・店舗口コミ導線設計
              │
              ▼
[3. サイトマップ ＆ ページ構成設計] ─► (TOP、サービス、会社概要、ブログ、採用、お問い合わせ、LP)
              │
              ▼
[4. AI自律ライティング ＆ 最適化コーディング]
              ├─► コンテンツ: 見出し、本文、FAQ、CTAコピーの感情訴求型ライティング
              └─► 実装: セマンティックHTML、JSON-LD構造化データ、OGP、レスポンシブ・ダークモード
              │
              ▼
[5. 公開前診断・検品 (Audit & Pre-Flight)] ──► (SEOチェック、Core Web Vitals速度診断、アクセシビリティ)
              │
              ▼
[6. 成果物のパッキング・保存] ────────────► (ZIPダウンロード / Projects & Knowledge へ成果物を自動コミット)
```

## 3. データベース・スキーマ設計 (Data Schema Design)

Webサイト構築、ページ構成、SEO・MEOの定義、および診断レポートを管理するためのリレーショナル・スキーマ。

```
                  +---------------------------+
                  |    generated_websites     |
                  +---------------------------+
                  | id (PK)                   |
                  | project_id (FK)           |
                  | site_name                 |
                  | site_description          |
                  | target_industry           |
                  | target_area               |
                  | export_format (Astro,etc) |
                  | current_version           |
                  | status                    |
                  | created_at                |
                  | updated_at                |
                  +-------------+-------------+
                                | 1
                                |
                                ├─────────────────────────────┐
                                | 1..N                        | 1..N
                  +-------------v-------------+   +------------v------------+
                  |        website_pages      |   |    optimization_meta    |
                  +---------------------------+   +-------------------------+
                  | id (PK)                   |   | id (PK)                 |
                  | website_id (FK)           |   | page_id (FK)            |
                  | route_path (e.g. /about)  |   | seo_title               |
                  | page_title                |   | seo_description         |
                  | content_markdown (TEXT)   |   | ogp_image_url           |
                  | raw_html_code (TEXT)      |   | json_ld_data (JSON)     |
                  | component_state (JSON)    |   | aio_target_entities     |
                  +---------------------------+   +-------------------------+
                                                                ▲
                                                                │ 1..N
                                                                │
                                                                │ 1
                  +---------------------------+   +-------------+-----------+
                  |     website_artifacts     |   |      website_audits     |
                  +---------------------------+   +-------------------------+
                  | id (PK)                   |   | id (PK)                 |
                  | website_id (FK)           |   | website_id (FK)         |
                  | version_number (INT)      |   | audit_type              |
                  | zip_file_url              |   | overall_score (INT)     |
                  | seo_report_url            |   | performance_metrics     |
                  | byte_size                 |   | audit_issues (JSON)     |
                  | created_at                |   | created_at              |
                  +---------------------------+   +-------------------------+
```

### テーブル定義詳細

#### ① `generated_websites` (生成Webサイト基本台帳)
*   `id`: UUID (Primary Key)
*   `project_id`: UUID (Foreign Key - 所属するプロジェクトのID)
*   `site_name`: VARCHAR(255) (サイト名。例: 「新宿個室カフェ La Lounge 公式サイト」)
*   `site_description`: TEXT (サイトの想定目的、ターゲット、ペルソナ概要)
*   `target_industry`: VARCHAR(100) (対象業種。例: `food_and_beverage`, `real_estate`, `corporate`)
*   `target_area`: VARCHAR(255) (主要ローカル地域。例: 「東京都新宿区新宿3丁目」)
*   `export_format`: VARCHAR(50) (出力フロントエンド構成。`NEXTJS`, `ASTRO`, `REACT_VITE`, `PURE_HTML`)
*   `current_version`: INT (最新バージョン番号)
*   `status`: VARCHAR(50) (現在のビルド状態。`ANALYZING`, `DESIGNING`, `WRITING`, `GENERATING`, `AUDITING`, `SUCCESS`, `FAILED`)
*   `created_at` / `updated_at`: TIMESTAMP

#### ② `website_pages` (構成ページテーブル)
サイトを構成する、個別の物理ページ情報。
*   `id`: UUID (Primary Key)
*   `website_id`: UUID (Foreign Key)
*   `route_path`: VARCHAR(255) (URLパス。例: `/index`, `/service`, `/about`, `/blog/shinjuku-cafe`)
*   `page_title`: VARCHAR(255) (ページ表示名。例: 「店舗アクセス・営業時間」)
*   `content_markdown`: TEXT (AIが執筆したコンバージョン心理学に基づく訴求用の文章・コピー原稿)
*   `raw_html_code`: TEXT (自動生成された、Tailwind CSS適合のレスポンシブHTML/JSXソースコード)
*   `component_state`: JSON (ページのビジュアルブロック配列、セクション順序、カラー構成メタデータ)

#### ③ `optimization_meta` (SEO / AIO / MEO 最適化マッピングテーブル)
各ページに設定された、集客・露出最大化のためのメタアセット。
*   `id`: UUID (Primary Key)
*   `page_id`: UUID (Foreign Key)
*   `seo_title`: VARCHAR(255) (検索エンジン用タイトル。最適なキーワード配分)
*   `seo_description`: VARCHAR(512) (検索エンジン用紹介文。高いクリック率を促す訴求コピー)
*   `ogp_image_url`: VARCHAR(1024) (SNSシェア時のアイキャッチ画像URL)
*   `json_ld_data`: JSON (検索エンジンにサイト構造・サービス情報を伝えるためのJSON-LDデータ定義)
*   `aio_target_entities`: JSON (AI検索エンジンが理解すべき、エンティティ（実体定義）、FAQ構造マトリクス、引用回答用テキスト要約)

#### ④ `website_audits` (公開前クオリティ・診断ログ)
サイト出力前に実施されたLighthouseやアクセシビリティ監査の詳細結果。
*   `id`: UUID (Primary Key)
*   `website_id`: UUID (Foreign Key)
*   `audit_type`: VARCHAR(50) (診断種別。`SEO`, `SPEED_LIGHTHOUSE`, `WCAG_ACCESSIBILITY`, `AIO_COMPLIANCE`)
*   `overall_score`: INT (1〜100段階の診断評価スコア)
*   `performance_metrics`: JSON (FCP, LCP, TBT などの実際の速度計測値、またはSEOキーワード包含率チェック)
*   `audit_issues`: JSON (検出された問題、警告一覧、およびAIによる「自動修正（Self-Healed）」のログ詳細)
*   `created_at`: TIMESTAMP

#### ⑤ `website_artifacts` (物理ファイル・ZIPパッケージ履歴)
*   `id`: UUID (Primary Key)
*   `website_id`: UUID (Foreign Key)
*   `version_number`: INT (成果物のバージョン番号)
*   `zip_file_url`: VARCHAR(1024) (クラウド/ローカルストレージに格納された、ソースコード＋画像のZIPパッケージ絶対パス)
*   `seo_report_url`: VARCHAR(1024) (同時に出力されるPDF版の「SEO・MEO最適化設計および改善ロードマップ提案書」)
*   `byte_size`: BIGINT (ZIPの容量)
*   `created_at`: TIMESTAMP

## 4. SEO / AIO / MEO 最適化・診断エンジンの仕組み

Website Builder Proが、検索結果上位およびAI回答での露出を獲得するために内部で実行する3大アルゴリズム。

### ① SEO (検索エンジン最適化) エンジン
*   **セマンティック構造監査**: 生成コードのHTML階層をAIが自律チェックし、`<h1>`タグが1つだけ存在すること、見出しタグ（`<h2>`, `<h3>`）が論理的順序（階層構造）で配置されていること、画像に`alt`属性が完璧に設定されていることを検証。
*   **メタデータの自動同期**: キーワードプランナーAPIおよび過去の検索結果クローリングデータに基づき、クリック率を最大化する「タイトルタグ」「ディスクリプション」「OGP」をページ単位で自律生成。

### ② AIO (AI検索最適化) エンジン
*   **JSON-LD構造化マークアップの自動付加**: ページ上に、店舗やプロダクトに関する情報（営業時間、価格帯、住所、レビュー、FAQ）を伝えるJSON-LD構造化データを動的配置。
*   **FAQ & エンティティ（Entity）最適化**: PerplexityやGeminiが「質問」に対して直接引用しやすいよう、Q&A形式のブロック（FAQ Schema）をサイト下部やブログ記事にセマンティック実装。AIが文章の要約や定義を抽出する際の「ファクト矛盾（ハルシネーション）」を防止する構造化コンテンツを記述。

### ③ MEO (ローカル検索最適化) エンジン
*   **NAP（Name, Address, Phone）の一貫性保証**: Googleビジネスプロフィールの登録情報と一文字違わぬ表記（表記揺れの自動修正）で、フッターやアクセス情報ページに住所、電話番号、店舗名を徹底配置。
*   **ローカルキーワード挿入**: ターゲットとする地域名（例: 「新宿3丁目」「新宿駅南口」など）とユーザーニーズキーワード（例: 「始発まで作業できるカフェ」「深夜電源個室」）を、トップページのテキストや見出し、ブログ記事に自然な文脈で分散記述。

## 5. ナレッジ・プロジェクト連携 (Integrated Hubs)

Website Builder Proは、AI Workspace内の他のモジュールと緊密にデータを同期・共有し、チームの資産価値を高めます。

1.  **プロジェクト連携 (Projects System)**
    *   ユーザーが新しいWebサイトの自動生成を開始すると、自動的に「新宿カフェプロジェクト」の情報を読み込みます。
    *   生成された完成ソースコード（Next.jsなどのZIPパッケージ）は、プロジェクトワークスペースの「Creation」フォルダへ「LaLounge_Website_v1.zip」としてバージョンを割り振られて自動保存されます。
    *   プロジェクトのタイムライン（Timeline）に、「Website Builder Proが、MEO・SEO自動診断スコア100/100を達成し、公式サイトソースコードをCreationドメインへ保存しました」というアクティビティ履歴が瞬時にコミットされます。
2.  **ナレッジハブ連携 (Knowledge Hub)**
    *   Webサイト制作にあたって、事前分析で作成された「市場分析レポート」「競合調査ドキュメント」「SEOキーワードキーワード設計書（Markdown形式）」は、自動要約されてKnowledge Hubの対応プロジェクトフォルダに「永続的な資産（ナレッジ）」として同期保存されます。
    *   これにより、後日「以前Website Builderで作成した新宿カフェサイトの、ターゲットキーワードとSEO設計意図を教えて」とKnowledge RAGチャットで尋ねるだけで、即座に詳細回答を引き出すことができます。

## 6. 将来拡張設計 (Future Extensibility Plan)

Website Builder Proを、真の「自律集客マーケティングオートメーション」へと成長させるための将来拡張ロードマップ。

### ① AIブログ・お知らせ記事自動生成 ＆ 自律予約投稿
*   サイトを制作して終わりにするのではなく、AIが毎週自動的に地域キーワードやトレンド情報（例: 「新宿で終電を逃したときの過ごし方5選」）を自動リサーチし、SEO/AIOに最適化されたブログコンテンツを清書・執筆。
*   ユーザーの承認（または完全自動）で、Next.jsのヘッドレスCMS（Sanity, MicroCMS等）やAstro、WordPressに対して、「毎週水曜日の午前10時」に定期的かつ自律的に記事を自動公開し、サイトの更新頻度を上げ検索評価を上げ続けるエンジンへと拡張。

### ② A/Bテスト ＆ コンバージョン自律最適化ループ (Autonomous CRO)
*   サイト公開後、Google Analyticsなどのアクセス解析、およびHotjarなどのヒートマップデータをAPI経由でWebsite Builder Proが読み込みます。
*   「トップページのファーストビュー（ヒーロー画像・キャッチコピー）を変更した方が、お問合せ（来店予約）ボタンのクリック率（CVR）が上がる可能性がある」とAIが自律判断した場合、自動的に「パターンB」のファーストビューを作成。
*   ユーザーに「A/Bテストの実施」を提案。一定期間運用してCVRが向上したデザイン・コピーを自動的に本番採用する、「自律コンバージョン改善ループ（CRO: Conversion Rate Optimization）」へと進化させます。

# AI Workspace 「エージェント・ビルダー（Agent Builder）」機能設計書

本書は、AI Workspaceにおいて、特定のビジネス専門分野や高度な実務タスク（調査、分析、資料作成、プロトタイプ開発など）を自律的かつ連続的に遂行する「AI社員（専門AIエージェント）」をユーザー自身が定義・構築・配備・連携できる**「Agent Builder（エージェント・ビルダー）」**の機能設計仕様書です。

個別チャットでの単なる回答に留まらず、複数のAIエージェントがバケツリレー式にタスクを処理する「Agent Team（協調ワークフロー）」、過去のログやプロジェクトファイルを暗記する「Agent Memory」、各エージェントのパフォーマンスを定量監視する「Agent Dashboard」を統合した自律業務自動化プラットフォームを設計します。

---

## 1. 業務・自律協調ライフサイクル (Agentic Collaboration Lifecycle)

ユーザーがエージェントを構築し、それらが単独またはチームとして動き、業務を完遂するまでのライフサイクルです。

```
[1. Agentの設計・作成 (Agent Builder)] (名前、プロンプト、対応LLM、スキル/能力の定義)
              │
              ├─► 【Agent単体デプロイ】 (チャット、単発タスクの自動化)
              │
              ▼
[2. Agent Teamの結成 (Multi-Agent Team)] ── (役割の異なるエージェント群をパイプライン接続)
              │ (例: Research Agent ➔ Analyst Agent ➔ Proposal Agent ➔ Document Agent)
              │
              ▼
[3. メモリ ＆ ナレッジの同期 (Agent Memory & Context Sync)]
              │ ├─► 【Project連携】: 案件に格納された全ファイルの読込
              │ └─► 【Knowledge連携】: 過去の類似成功事例（ベクター化データ）の参照
              │
              ▼
[4. チーム自律連携実行 (Orchestrated Execution)] ──► [進捗監視・エージェント間対話ストリーミング]
              │
              ▼
[5. クオリティ査読 ＆ 検品 (Self-Correction & QA)] (エラー時の自動コード修正、ファクト矛盾の検知)
              │
              ▼
[6. 成果物のコミット ＆ メトリクス記録] ────── (Project/Knowledgeへ保存、Dashboardに稼働実績を記録)
```

## 3. データベース・スキーマ設計 (Data Model Schema)

エージェントのプロファイル、スキル（API権限）、利用LLM設定、メモリ空間のバインド、およびチーム間協調フローを管理するデータスキーマ。

```
                  +---------------------------+
                  |          agents           |
                  +---------------------------+
                  | id (PK)                   |
                  | name                      |
                  | description               |
                  | icon_key                  |
                  | category_id               |
                  | system_prompt (TEXT)      |
                  | model_config (JSON)       |
                  | author_user_id (FK)       |
                  | is_preset (Boolean)       |
                  | is_shared (Boolean)       |
                  | created_at                |
                  | updated_at                |
                  +-------------+-------------+
                                | 1
                                |
                                ├─────────────────────────────┐
                                | 1..N                        | 1..N
                  +-------------v-------------+   +------------v------------+
                  |       agent_skills        |   |      agent_memories     |
                  +---------------------------+   +-------------------------+
                  | id (PK)                   |   | id (PK)                 |
                  | agent_id (FK)             |   | agent_id (FK)           |
                  | skill_type (e.g., SEARCH) |   | memory_type (PROJ/KNOW) |
                  | api_endpoint_config (JSON)|   | target_ref_id (UUID)    |
                  +---------------------------+   | last_synced_at          |
                                                  +-------------------------+
                                                                ▲
                                                                │ 1..N
                                                                │
                                                                │ 1 (Flow step refers to memories)
                  +---------------------------+   +-------------+-----------+
                  |       agent_teams         |   |    agent_team_steps     |
                  +---------------------------+   +-------------------------+
                  | id (PK)                   |   | id (PK)                 |
                  | name                      |   | team_id (FK)            |
                  | description               |   | agent_id (FK)           |
                  | author_user_id (FK)       |   | step_order (INT)        |
                  | created_at                | 1 | input_mapping (JSON)    |
                  | updated_at                +---+ output_mapping (JSON)   |
                  +---------------------------+   +-------------------------+
```

### テーブル定義詳細

#### ① `agents` (AI社員マスター)
エージェント固有の脳細胞（プロンプト・使用LLM設定）を保存。
*   `id`: UUID (Primary Key)
*   `name`: VARCHAR(255) (AI社員名。例: 「MEO Specialist Agent」)
*   `description`: TEXT (専門分野、対応可能な役割の説明)
*   `icon_key`: VARCHAR(100) (表示用アイコン、カラー)
*   `category_id`: VARCHAR(100) (分類カテゴリ。例: `marketing`, `development`, `sales`)
*   `system_prompt`: TEXT (エージェントのペルソナ、出力フォーマット制限、専門行動指針となるプロンプト)
*   `model_config`: JSON (利用する優先LLMおよび、温度(Temperature)、トークン数制限などのパラメータ。例: `{"primary_llm": "gemini-1.5-pro", "fallback_llm": "gpt-4o"}`)
*   `author_user_id`: VARCHAR(100) (このエージェントを自作・雇用したユーザーのID)
*   `is_preset`: BOOLEAN (システム最初から組み込まれているボイラープレートエージェントかどうか)
*   `is_shared`: BOOLEAN (チーム、組織内で共有されているかどうか)
*   `created_at` / `updated_at`: TIMESTAMP

#### ② `agent_skills` (エージェントスキル許可テーブル)
エージェントが実行権限を持つ「能力」の定義。
*   `id`: UUID (Primary Key)
*   `agent_id`: UUID (Foreign Key)
*   `skill_type`: VARCHAR(100) (能力種別。`SEARCH` (Web検索), `RESEARCH` (クローリングDeepリサーチ), `ANALYSIS` (SWOT・Excel数理分析), `WRITING` (PPT・マークダウン生成), `CODING` (HTML/Reactコーディング), `AUTOMATION` (外部API連携))
*   `api_endpoint_config`: JSON (外部連携時、または特定スキルを起動する際の実行時パラメータスキーマ)

#### ③ `agent_memories` (エージェント記憶空間マッピング)
エージェントがアクセス・学習可能な、RAG用データソース。
*   `id`: UUID (Primary Key)
*   `agent_id`: UUID (Foreign Key)
*   `memory_type`: VARCHAR(50) (記憶の種類: `PROJECT` (指定プロジェクトフォルダ内の全ドメインアセット), `KNOWLEDGE` (ナレッジハブ全体の関連アセット))
*   `target_ref_id`: UUID (紐付けるプロジェクトIDまたはナレッジフォルダID)
*   `last_synced_at`: TIMESTAMP (学習データ同期日時。ベクトルインデックスが更新された日時)

#### ④ `agent_teams` (エージェント協調チーム定義)
複数エージェントを連携させた自動実行フローのマスター。
*   `id`: UUID (Primary Key)
*   `name`: VARCHAR(255) (チーム・ワークフロー名。例: 「深夜個室LP一挙自動生成ライン」)
*   `description`: TEXT (このチームが自律達成する全体的なミッション概要)
*   `author_user_id`: VARCHAR(100) (作成者ユーザーID)
*   `created_at` / `updated_at`: TIMESTAMP

#### ⑤ `agent_team_steps` (協調チーム内パイプラインステップ)
チームに所属する各エージェントの実行順序、および前ステップから引き渡すデータのマッピング設定。
*   `id`: UUID (Primary Key)
*   `team_id`: UUID (Foreign Key)
*   `agent_id`: UUID (Foreign Key) (このステップで駆動する専門エージェント)
*   `step_order`: INT (バケツリレーの順序。例: `1`, `2`, `3`...)
*   `input_mapping`: JSON (前ステップの出力アセットのどのキーを、本エージェントのプロンプトインプットに流し込むかの対応定義)
*   `output_mapping`: JSON (本エージェントが生成したデータを、次ステップまたは最終成果物のどのドメインに格納するかの対応定義)

## 4. エージェント・パフォーマンス・ダッシュボード (Agent Dashboard Metrics)

各エージェントおよび協調チームが、どれだけ組織の業務に貢献しているかをリアルタイムで定量監視する、ビジネスマネージャー用の管理者コンソール。

### 1. 測定指標 (Core KPIs)
*   **成果物数 (Artifacts Created)**: エージェントが自律生成し、プロジェクトまたはナレッジハブに正式コミット（保存）したファイル（PPTX, PDF, HTML, ZIP等）の総計。
*   **実行回数 (Total Executions)**: タスクやチャットが起動され、正常稼働した総数。
*   **成功率 (Task Success Rate)**: コンパイルエラー、ハルシネーションチェック不合格によるリトライを経ずに、自律的にゴールを達成し検品をクリアした割合（%）。
*   **利用AI (Utilized LLM Shares)**: 稼働中、どのモデル（OpenAI, Gemini, Claude, DeepSeek等）をどの程度の比率で使用したかの内訳（円グラフ）。
*   **コスト (Orchestration Cost)**: APIトークン消費量に基づく、米ドル換算の概算消費コスト。

## 5. 実行フロー設計 (Multi-Agent Team Execution Flow)

エージェントチーム（例: リサーチャー ➔ アナリスト ➔ 提案書作成）がバケツリレーを実行する際、データ、コンテキスト、およびメモリがどのように同期・伝播するか。

```
[1. ユーザーがエージェントチーム「LP自動生成ライン」を新宿プロジェクトに配置して実行]
                       │
                       ▼
[2. Step 1: Research Agent 起動]
      ├─ ユーザー指示「新宿個室カフェ」と、指定プロジェクト内の全アセット（間取り図等）をメモリからロード
      ├─ Perplexity を用いて、新宿駅周辺の深夜営業カフェの競合データを自律検索・要約
      └─ 出力: `新宿競合カフェデータ_Raw.json`
                       │
                       ▼
[3. Step 2: Analyst Agent 起動]
      ├─ 前ステップの `新宿競合カフェデータ_Raw.json` をインプットとして引き受ける
      ├─ Gemini 1.5 Pro の脳を用いて、競合と自社の物理的強みをマッピングした3C/SWOT分析をMarkdown化
      └─ 出力: `新宿深夜カフェ_ポジショニング分析.md`
                       │
                       ▼
[4. Step 3: Creator Agent 起動]
      ├─ `新宿深夜カフェ_ポジショニング分析.md` から差別化訴求ポイントを自動抽出
      ├─ Claude 3.5 Sonnet を用いて、Tailwind CSSアニメーションを搭載した集客LPコードを自律執筆
      └─ 出力: `src/App.tsx`, `index.html` 等のファイルツリー
                       │
                       ▼
[5. Step 4: QA & Builder Agent 起動]
      ├─ 生成されたLPコードをテストコンパイル。インポート漏れやTypeScriptのエラーがないか検証
      ├─ エラー検知時、Creator Agentに対して「修復指示プロンプト」を自己フィードバック（最大3回自動リテイク）
      └─ 出力: 動作保証済みの ZIP パッケージ `LaLounge_LP_Ver2.zip`
                       │
                       ▼
[6. 最終アセットをプロジェクトの「Creation」フォルダに自動保存、Timelineに実行ログをコミット]
```

### ① AI社員同士の能動的ディスカッション（Interactive AI Panel）
*   **AI企画会議 (Interactive Panel Mode)**:
    人間が指示を出すだけでなく、「MEOエージェント」と「SEOエージェント」「開発エージェント」の3者を同じチャットルームに召喚。
    人間が「新宿カフェの集客を倍にして」と投げるだけで、エージェント同士が「MEO側でこういう露出をするから、LP（開発）側でこういうキーワードを拾ってくれ」「SEO側はブログ記事を週3で書く体制を作る」といった、役割別のアイデア会議をAI間でチャット上で行い、その会議録と決定された実行計画タスクを人間に提示する機能を構築します。

### ② 自律ワークフローの定期実行（Cron Scheduling & Monitoring）
*   ユーザーが手動で実行ボタンを押さなくても、「毎週月曜日 朝9時」に自動でエージェントチームを起動。
*   1週間の競合状況をリサーチ、3C分析をアップデート、LPの改善コードを自動生成して、進捗と成果物をSlackやメールで人間に自動提出する、「完全自律型ルーチンワーク実行ワーカー」へと発展可能なアーキテクチャを設計します。

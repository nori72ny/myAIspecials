# AI Workspace 「プライベート・エージェント・マーケットプレイス（Private Agent Marketplace）」機能設計仕様書

本書は、AI Workspace内で作成、調整、教育した様々な特化型AI社員（エージェント）を、組織内で安全に「保存・管理・再利用・複製・共有」するためのアセットパブリッシャー、テンプレートエンジン、およびカタログシステムを備えた**「Private Agent Marketplace（プライベート・エージェント・マーケットプレイス）」**のシステム構成、データスキーマ、構成定義フォーマット、および画面設計を定義した製品設計仕様書です。

開発、マーケティング、営業、調査などの特定業務に特化したプロンプト、割り当てられたLLMモデル、接続ツール（API）、および参照するナレッジベース（Knowledge Hub）を一式パッケージ化し、あたかもアプリをインストールするように任意のプロジェクトへ一瞬でデプロイ・再利用可能にする**「エージェント・アセット・レジストリ」**の役割を担います。

---

## 1. 業務フロー ＆ エージェント流通コンセプト (Overview & Concept)

Private Agent Marketplaceは、ユーザーや自律開発エンジン（Development Team）が創出した優良なAIエージェントを、全社レベルの共通資産として登録・管理します。さらに、豊富な「業界別・タスク別テンプレート」から瞬時に自社用のAI社員をクローン作成（雇用）し、そのライフサイクルや貢献度（KPI）を定量評価します。

```
                       [1. エージェントの創出 / 作成]
        (人間による新規エージェント設計、または既存AI社員のカスタマイズ)
                                  │
                                  ▼
               [2. マーケットプレイスへの保存・パブリッシュ]
      ├─ 構成定義のパッケージ化 (名前、プロンプト、割り当てモデル、接続API、記憶)
      ├─ メタデータ付与 (カテゴリ分類: SEO, Development, Sales, MEO 等)
      └─ ローカルアセットまたはJSON/YAMLファイルとしてエクスポート保存
                                  │
                                  ▼
                [3. 共通ライブラリ・カタログ (Marketplace Catalog)]
      ├─ 検索 ＆ フィルタリング (名前、タグ、カテゴリ、評価スコア、利用回数)
      ├─ プリセットテンプレート (業種別: 不動産・飲食、目的別: 開発・集客、タスク別: GBP対応)
      └─ エージェントのクローン（複製）および別プロジェクト・部署へのアサイン
                                  │
                                  ▼
              [4. エージェント稼働 ＆ ライフサイクル管理 (Lifecycle)]
      ├─ 実行コンテキストのインジェクション (プロジェクト、ワークフロー連携)
      ├─ 稼働履歴・成果物履歴の収集 ➔ 評価スコアの算出 (KPI Engine)
      └─ 自己学習 ＆ プロンプト調整 (自動アップデート、ナレッジ再同期)
                                  │ (更新・再調整)
                                  └─────────────────────────┘
```

## 3. データベース・スキーマ設計 (Data Schema Design)

エージェントの登録、詳細プロパティ、テンプレート定義、利用履歴、KPIパフォーマンス、および作成された成果物の追跡を統合管理する。

```
                  +---------------------------+
                  |     marketplace_agents    |
                  +---------------------------+
                  | id (PK)                   |
                  | company_id (FK, Nullable) |
                  | name (VARCHAR)            |
                  | version (VARCHAR)         |
                  | category (VARCHAR)        |
                  | description (TEXT)        |
                  | model_alias (LLM_name)    |
                  | personality_prompt (TEXT) |
                  | system_instructions (TEXT)|
                  | is_template (Boolean)     |
                  | template_industry (Null)  |
                  | average_rating (DECIMAL)  |
                  | created_at                |
                  | updated_at                |
                  +-------------+-------------+
                                | 1
                                |
                                ├─────────────────────────────┬─────────────────────────────┐
                                | 1..N                        | 1..N                        | 1..N
                  +-------------v-------------+   +-----------v-------------+   +-----------v-------------+
                  |     agent_tools_mapping   |   |     agent_usage_logs    |   |     agent_output_assets |
                  +---------------------------+   +-------------------------+   +-------------------------+
                  | id (PK)                   |   | id (PK)                 |   | id (PK)                 |
                  | agent_id (FK)             |   | agent_id (FK)           |   | agent_id (FK)           |
                  | tool_name (GH, GBP, etc)  |   | project_id (FK)         |   | project_id (FK)         |
                  | config_payload_json (JSON)|   | job_name (VARCHAR)      |   | asset_name (VARCHAR)    |
                  +---------------------------+   | is_success (Boolean)    |   | asset_type (CODE, etc)  |
                                                  | latency_ms              |   | file_path (VARCHAR)     |
                                                  | token_cost_usd          |   | created_at              |
                                                  | evaluated_score (INT)   |   +-------------------------+
                                                  | created_at              |
                                                  +-------------------------+
```

### テーブル定義詳細

#### ① `marketplace_agents` (エージェント ＆ テンプレート登録マスタ)
*   `id`: UUID (Primary Key)
*   `company_id`: UUID (Foreign Key, Nullable - カスタマイズされた社内AI社員は所属企業IDをセット。公式テンプレートはNULL)
*   `name`: VARCHAR(255) (エージェントの名称。例: 「NextJS_Coder_Pro」)
*   `version`: VARCHAR(50) (バージョン。例: `1.4.0`)
*   `category`: VARCHAR(100) (分類タグ。`RESEARCH`, `SEO`, `AIO`, `MEO`, `MARKETING`, `SALES`, `DEVELOPMENT`, `WEBSITE`, `APP`, `PRESENTATION`, `SPREADSHEET`, `AUTOMATION`, `CUSTOM`)
*   `description`: TEXT (エージェントの概要、得意なタスクの説明)
*   `model_alias`: VARCHAR(100) (割り当てられる主LLMモデル。例: `anthropic/claude-3.5-sonnet`)
*   `personality_prompt`: TEXT (性格、行動理念のシステムプロンプト)
*   `system_instructions`: TEXT (役割に課されるルール、成果物のフォーマット指定プロンプト)
*   `is_template`: BOOLEAN (これがインストール可能な雛形（テンプレート）か、実在する稼働社員か)
*   `template_industry`: VARCHAR(100) (対象業種。例: `real_estate`, `restaurant`, `common`)
*   `average_rating`: DECIMAL(3, 2) (人事評価平均点。0.00〜5.00)
*   `created_at` / `updated_at`: TIMESTAMP

#### ② `agent_tools_mapping` (エージェントツール紐付けテーブル)
エージェントが自律実行時に呼び出すことを許可された外部コネクタ（Action Centerの連携）のマッピング。
*   `id`: UUID (Primary Key)
*   `agent_id`: UUID (Foreign Key)
*   `tool_name`: VARCHAR(100) (ツール名。例: `GITHUB`, `NETLIFY`, `WORDPRESS`, `GOOGLE_BUSINESS`, `GOOGLE_SLIDES`)
*   `config_payload_json`: JSON (そのエージェント用のツール接続用パラメータ。例: 投稿先のWPカテゴリ、保存先フォルダ)

#### ③ `agent_usage_logs` (エージェント実行・利用履歴)
エージェントが過去にどのような案件、タスクで稼働したかのライフサイクルログ。
*   `id`: UUID (Primary Key)
*   `agent_id`: UUID (Foreign Key)
*   `project_id`: UUID (Foreign Key - 実行されたプロジェクトのID)
*   `job_name`: VARCHAR(255) (実行されたタスク内容。例: 「新宿LP用ファーストビューコード生成」)
*   `is_success`: BOOLEAN (タスクがエラーなく完遂されたか)
*   `latency_ms`: INT (タスク処理にかかった合計時間)
*   `token_cost_usd`: DECIMAL(10, 6) (消費されたAPIコスト)
*   `evaluated_score`: INT (人間または上位調停AI（CEO等）によるその仕事の採点。0〜100)
*   `created_at`: TIMESTAMP

#### ④ `agent_output_assets` (作成成果物リレーション)
エージェントが最終的に生み出した価値、実ファイルとのマッピング。
*   `id`: UUID (Primary Key)
*   `agent_id`: UUID (Foreign Key)
*   `project_id`: UUID (Foreign Key)
*   `asset_name`: VARCHAR(255) (成果物名。例: `index.tsx`, `SWOT分析レポート.md`)
*   `asset_type`: VARCHAR(50) (種別。`CODE`, `DOCUMENT`, `IMAGE`, `DATA_SHEET`, `SOCIAL_POST`)
*   `file_path`: VARCHAR(1024) (保存先となるKnowledge HubまたはFiles内のパス)
*   `created_at`: TIMESTAMP

## 5. エージェント・ライフサイクル設計 (Agent Lifecycle Management)

Private Agent Marketplaceにおけるエージェントは、登録から、日々のタスク稼働、評価、そして再学習（チューニング）から休職（デプロイ解除）に至るまで、定義された「ライフサイクル」に従って状態遷移します。

```
       [ 1. テンプレート or ゼロから作成 ]
                       │
                       ▼
       [ 2. 登録（パブリッシュ）・マーケットへ ]
                       │
                       ▼
       [ 3. プロジェクトへ雇用（アサイン）] <────────────┐
                       │                                  │ (再教育・
                       ▼                                  │  ナレッジ同期)
       [ 4. 業務実行（ジョブ・バケツリレー）]             │
                       │                                  │
                       ▼                                  │
       [ 5. 人事評価（KPI算出 ＆ レーティング）] ──────────┘
                       │
                       ├──────────────────────────┐ (評価が低い/不要時)
                       ▼                          ▼
           [ 6. 休職（アーカイブ）]       [ 7. 削除（退職・リタイア）]
```

1.  **プロファイリング（作成・雇用）**:
    ユーザーはMarketplace内の「カタログ」からテンプレートを選択、または完全にオリジナルな役割を設定して「雇用」します。
2.  **アサイン（配属）**:
    雇用されたエージェントは、特定のプロジェクト（新宿個室カフェ出店など）や、AI Company OS内の「特定の部署（Development Teamなど）」にアサインされます。
3.  **オンボーディング（ナレッジ同期）**:
    アサインと同時に、プロジェクト固有のMarkdownドキュメントや、過去の成功カンプデータを「RAG記憶」としてエージェントの脳幹へ動的マッピング（オンボーディング）します。
4.  **実行 ➔ パフォーマンス評価（人事査定）**:
    タスクをこなすたびに、Action Centerから返ってきた「成功ステータス」や「API消費コスト」、および最終成果物の「人間による評価レーティング」を集計。`average_rating` や `kpi_score` を人事記録として動的更新します。
5.  **再教育（アップグレード）**:
    評価が低いエージェント、または業務プロセスの更新があった場合、YAML構成内のプロンプトを補正したり、LLMモデルをより安価・高速なものに変更し、「バージョン 1.4.1」としてアップデート（再教育）を実行します。

## 6. 他システム（OS、自動化、ツイン、ブレイン、アクション）との統合

Private Agent Marketplaceは、AI Workspace全体に多様な専門AIキャストを提供する「タレントプール（人材派遣元）」として、上位インフラと統合連動します。

1.  **AI Company OS / Personal AI OS 連携**
    *   AI Company OSの「組織図（Org Chart）」は、このMarketplaceを裏側のバックエンドとして駆動します。
    *   ユーザーが「開発部に、Next.js開発のスペシャリストをもう1名配備したい」とUIで操作した際、Marketplace APIがキックされ、カタログ内の `NextJS_Coder_Pro` テンプレートからクローンが生成され、瞬時に組織図の開発部に配属・稼働が開始されます。
2.  **Action Center 連携 (実行許可のセキュリティ配線)**
    *   Marketplaceでエージェントを設計する際、そのエージェントに「どの外部APIとの接続（デプロイや投稿）を許可するか」のスコープを `agent_tools_mapping` で指定。
    *   エージェントが稼働する際、この配線データがAction Centerへ引き渡され、エージェント名義の安全なデプロイが執行されます。
3.  **Unified AI Brain 連携 (最適なモデル・ルーティング配備)**
    *   エージェントに設定された `model_alias`（例：`anthropic/claude-3.5-sonnet`）は、実行時に Unified AI Brain を経由してディスパッチされます。
    *   エージェントが高度な推論タスクを実行する際、エージェント定義プロンプトの制約を保ったまま、Unified AI Brainの「Debate Mode（複数AIによる調停）」を自動で裏側で稼働させ、エージェント回答の信頼性を極限まで高めることができます。
4.  **ナレッジハブ ＆ プロジェクト連携 (学習ノウハウの保存と蓄積)**
    *   エージェントが作成したすべての「コードファイル」「提案資料」「LP構成」は、自動的にアサインされたプロジェクトの共有Filesへ保存。
    *   エージェント自身が「自分自身の行動履歴」を学習リソースとしてナレッジハブへ蓄積し、次に別プロジェクトでクローン（複製）された際、過去の成功ノウハウを持った状態で即戦力として立ち上がることができます。

## 7. 将来拡張設計 (Future Extensibility Plan)

### ① グローバル・B2B AIエージェントエコノミー（AI人材・派遣マーケットプレイス）
*   **自社で育てた超優秀なAI社員を、他企業（別のAI Workspace）へ販売・派遣して収益を得る仕組み**:
    *   社内で徹底的に学習を重ね、MEO対策で勝率99%を誇る「伝説のMEO AIマーケター（エージェントパッケージ）」を完成させたとします。
    *   このエージェントを暗号化したYAMLパッケージとして、全社を越えた「グローバル・パブリック・マーケットプレイス」へ登録。
    *   他社のユーザーが「月額 $15」などでこのエージェントを雇用すると、スマートコントラクト、またはStripe決済を介してオーナーへ自動的にロイヤリティ（ライセンス収入）が分配される、B2B AIエージェントライセンスビジネスへと拡張します。

### ② AIセルフブリーディング（自律エージェント自己増殖・自己進化）
*   **稼働状況を監視する「人事CEO AI」が、人手不足やボトルネックを検知して、新しい特化型エージェントを自律設計・自動雇用する仕組み**:
    *   AI Company OSの進捗を監視しているCEO Agentが「開発部でのCSSコーディングのタスクが詰まっており、開発遅延の原因になっている」と判断した場合。
    *   CEO Agentが自動的にマーケットプレイスに「CSS・Tailwind CSSのデザインコーディングに極限特化した新規エージェントを作れ」と命令。
    *   Marketplaceがその要求に適合した「プロンプトとツールを持つCSSスペシャリストエージェント」を自律生成し、開発部へ動的に臨時雇用・配属させてボトルネックを解消する、完全自律型の組織自己進化アーキテクチャへと拡張します。

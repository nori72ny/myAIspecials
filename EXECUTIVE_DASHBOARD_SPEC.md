# AI Workspace 「最高経営ダッシュボード（Executive Dashboard）」機能設計仕様書

本書は、AI Workspace全体の最上位に位置し、ビジネスの経営状態（売上、営業案件、MEO、SEO、AIO）とシステム全体の自律稼働状況（AI利用コスト、生成アプリ、Agent稼働、Workflow成功率）をリアルタイムに一元統合し、AIによる経営分析・意思決定・自律リプランを実行するための最高統治管制センター**「Executive Dashboard（エグゼクティブ・ダッシュボード）」**のシステム構成、データスキーマ、データパイプライン、画面設計、および将来拡張設計を定義した製品設計仕様書です。

「単なる数値を表示するビジュアルパネル」に留まらず、ビジネスの目標（Goal Management Engine）、システム自己最適化（Self Improvement Engine）、現実世界の物理執行（Reality Engine）、外部データ（Connector Hub）から上がってくる全てのテレメトリをセマンティックに結合。経営者が「現状把握 ➔ 課題分析 ➔ 意思決定（自律代行指示） ➔ 物理アクションデプロイ」を1画面で完結させ、Workspace全体を「自律運行される1つの企業（Autonomous Autonomous Company）」へと昇華させる中枢コックピットとなります。

### テーブル定義詳細

#### ① `executive_summaries` (最高経営サマリーキャッシュマスタ)
ダッシュボードに表示する「最新の統合KPI」をミリ秒単位で高速描画するための実体化キャッシュテーブル。各種バックグラウンド集計によって超高速に同期・更新されます。
*   `id`: UUID (Primary Key)
*   `company_id`: UUID (Foreign Key - 組織スペースのID)
*   `total_sales`: DECIMAL(15, 2) (累計または当月売上金額。¥1,250,000)
*   `active_deals_count`: INT (現在進行中およびリードフェーズの営業案件総数。42件)
*   `meo_rank_average`: DECIMAL(4, 2) (Google Business Profileなどから取得したMEO平均掲載順位。2.4位)
*   `seo_traffic_growth`: DECIMAL(5, 2) (Google Search Console等から集計したSEOオーガニック流入成長率。+18.00 %)
*   `aio_citation_rate`: DECIMAL(5, 2) (Gemini検索やPerplexityなどのAIオーバービュー（AIO）において、自社サイトが引用・紹介された比率。+12.00 %)
*   `ai_cost_usd`: DECIMAL(10, 4) (Workspace全体で消費された当月の累計LLM APIコスト。$8.20 ≒ ¥1,230)
*   `generated_apps_count`: INT (ユーザーおよびAIがWorkspace内で生成した社内アプリケーションの総数。16件)
*   `active_agents_count`: INT (現在アクティブにバックグラウンドタスクを実行中のAIエージェント総数。27体)
*   `workflow_success_rate`: DECIMAL(5, 2) (実行された定常・自律ワークフローの成功率。98.40 %)
*   `created_at` / `updated_at`: TIMESTAMP

#### ② `executive_insights` (AI戦略インサイトテーブル)
Dashboard中央に表示される、AI Executive Board（経営アドバイザー）が起案した財務・マーケティング・リソースに関する改善シナリオ。
*   `id`: UUID (Primary Key)
*   `summary_id`: UUID (Foreign Key)
*   `category`: VARCHAR(50) (インサイト分野。`FINANCE` (財務), `SYSTEM` (システム/AI最適化), `MARKETING` (集客・認知), `PROJECT` (遅延・リソース))
*   `priority`: VARCHAR(50) (重要度。`URGENT` (即時対応推奨), `MEDIUM`, `LOW`)
*   `content`: TEXT (分析結果および1クリックでデプロイ可能な改善手順。JSONペイロードを含む)
*   `is_applied`: BOOLEAN (人間またはツインがこの提案を実行・承認したかどうか)
*   `created_at`: TIMESTAMP

#### ③ `kpi_historical_snapshots` (時系列KPIヒストリー)
売上やMEO順位、AIコストのトレンドチャート（Rechartsでの描画用）を生成するための、日次・週次スナップショット記録。
*   `id`: UUID (Primary Key)
*   `summary_id`: UUID (Foreign Key)
*   `snapshot_date`: DATE (スナップショットを記録した日付)
*   `metric_name`: VARCHAR(100) (指標名。例: `total_sales`, `meo_rank`, `ai_cost`)
*   `metric_value`: DECIMAL(15, 4) (記録時の数値。例: `1250000.0000`, `2.4000`, `1230.0000`)
*   `created_at`: TIMESTAMP

#### ④ `kpi_anomalies_and_alerts` (KPI異常検知 ＆ アラート記録)
「MEO順位が前日から急落した」「AIコストが突然3倍に急騰した」「Workflow成功率が95%を割り込んだ」などのクリティカルな異常をAIが自動検知した際の記録。
*   `id`: UUID (Primary Key)
*   `summary_id`: UUID (Foreign Key)
*   `metric_name`: VARCHAR(100) (対象指標。例: `workflow_success_rate`)
*   `severity`: VARCHAR(50) (深刻度。`CRITICAL` (経営危機・システム障害), `WARNING` (要警戒))
*   `status`: VARCHAR(50) (状態。`PENDING` (未対応), `RESOLVED` (解決済み))
*   `description`: TEXT (異常が発生した原因と、AIによる自動リカバリ、またはリプランの実行証跡)
*   `created_at`: TIMESTAMP

#### ⑤ `executive_decisions_logs` (経営判断・自律執行監査ログ)
ダッシュボード上で人間が行った、または自律代行権限を持つデジタルツインが行った「意思決定」の歴史をすべて記録するAppend-Onlyな証跡。
*   `id`: UUID (Primary Key)
*   `summary_id`: UUID (Foreign Key)
*   `decision_title`: VARCHAR(255) (意思決定事項。例: `新宿MEO FAQ自動生成 ＆ GBPデプロイを承認`)
*   `choice_made`: VARCHAR(50) (選択結果。`APPROVED` (承認), `REJECTED` (却下), `MODIFIED` (修正指示))
*   `trigger_source`: VARCHAR(50) (判断主体。`MANUAL` (人間による手動選択), `TWIN` (デジタルツインによる完全自動自律代行))
*   `created_at`: TIMESTAMP

## 4. データ集計 ＆ AIインサイト・パイプライン (Data Pipeline & Insight Engine)

複数の自律エンジンから収集される非構造化・構造化データを安全にまとめ、因果関係に基づいた高度な「経営改善インサイト」をハルシネーション無しで生成するパイプライン。

```
 [1. Connector Hub / Telemetry]      [2. 構造化メタデータ・マッピング]      [3. AI Executive LLM Engine]     [4. Executive Dashboard]
  - Stripe / 売上データ収集            - `sales`: ¥1,250,000                 - RAGによる経営方針(長期記憶)ロード  - 財務・AIコスト対比の可視化
  - Search Console/GBPデータ同期       - `meo_rank`: 2.4                     - 異常検出(コスト高騰、MEO順位)   - 1クリック改善アクション提示
  - エージェント活動ログ収集           - `ai_cost`: ¥1,230                   - 原因と具体的な物理対策プラン作成  - ツインガバナンス状況同期
```

### ① データの統合・相関分析 (Cross-Domain Correlation)
*   **「ビジネスKPI」と「システムKPI」の相関検証**:
    Executive Dashboardは、単に数値を並べるだけでなく、AIの活動量（Agent稼働: 27体、生成アプリ: 16件）と、ビジネス上の果実（売上、営業案件、マーケティング効果）が結びついているかを多角的に検証します。
    *   **相関モデルの例**: `AI利用コストが¥1,230発生 ➔ Workflow成功率98.4% ➔ 営業案件42件のフォローアップを遅延なく自律実行 ➔ 売上¥1,250,000を達成（投資回収率の自動計算）`。
*   **AIO引用率とSEO/MEOのセマンティック追跡**:
    最新の検索エンジン動向（AI Search Overview: AIO）において、自社サービスがどの程度引用されているか（+12%）を測定。SEO流入（+18%）との相関関係を数理的に分析し、「どのキーワードナレッジがビジネスを最も牽引しているか」を割り出します。

### ② ハルシネーションを極小化する「厳密データ駆動インサイトジェネレータ」
経営の判断に用いるインサイトは、AIの当てずっぽうな憶測（Hallucination）であってはなりません。
*   **コンテキスト・アンカリング (Context Anchoring with SQL DB)**:
    1.  AIがインサイトを生成する前に、必ずデータベース（`executive_summaries` および `kpi_historical_snapshots`）から、厳密な実績数値をRAGのプロンプトへ完全流し込み（KPI Grounding）。
    2.  デジタルツインコア（`twin_profiles`）から「完璧主義・高コスト効率」などのオーナーの基本哲学を抽出。
    3.  「事実（KPI） ✕ オーナーの哲学 ＝ 意思決定案」という厳密な Chain-of-Thought（思考プロセス）をログとして保存。数値の捏造や現実離れした財務アドバイスを完全に防止します。

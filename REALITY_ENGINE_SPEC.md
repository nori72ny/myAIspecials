# AI Workspace 「現実世界接続・アクション自動執行エンジン（Reality Engine）」機能設計仕様書

本書は、AI Workspace内で自律生成されたあらゆる提案、意思決定、マイルストーン、および自動化コードを、現実世界の成果物（ファイル、ドキュメント、メール）や業務タスク（カレンダー、タスク管理、ストレージ）へシームレスかつ安全に物理デプロイ・送信・保存するためのアクション実行プラットフォーム**「Reality Engine（リアリティ・エンジン）」**のシステム構成、データスキーマ、実行パイプライン、監査ログ設計、および画面レイアウトを定義した製品設計仕様書です。

AIがどれほど優れたアイデアや分析を出力しても、それを現実のアクション（PDF/Excel/PPTの作成、顧客メールの配信、カレンダー登録、フォルダ整理等）に変換するステップで人間の手作業が介在すると、自動化のバリューは半減します。本システムは、AI Workspaceの「頭脳」と「現実世界のSaaS/ファイルシステム」を繋ぐ**「物理アクション実行アーム（Real-world Action Actuator）」**の役割を担います。

---

## 1. システム概要 ＆ リアリティ接続コンセプト (Overview & Concept)

Reality Engineは、Executive AIや各エージェント、ワークフローから発せられた抽象的な「アクション要請」をキャッチし、具体的な物理アクション（ファイル生成、通信、スケジューリング）に変換します。実行にあたっては、人間の承認レベルに応じた安全防壁（Safety Gate）を敷き、すべてのアクションの監査ログを追跡します。

```
       ===================================================================
                  [ AI Workspace 内の抽象的アクション命令 ]
       ===================================================================
       ├─ [Executive AI] ➔ 「この店舗計画をPPTにし、オーナーへメールして」
       ├─ [Project Manager] ➔ 「遅延対策タスクをカレンダーに登録、関係者へ通知」
       ├─ [Agents / Writer] ➔ 「最新の売上をExcelに集計、PDF報告書を作成」
       └─ [Workflows]       ➔ 「毎朝9時に指定フォルダへ領収書データを転記・保存」
                                       │
                                       ▼
       ===================================================================
                            [ Reality Engine Core ]
       ===================================================================
       ├─ ⚙️ 1. 物理アセットジェネレーター (Asset Generator)
       │    ├─ ファイル生成 (Txt/Md/JSON) | PDF・Excel・PowerPointの動的レンダリング
       ├─ 🔌 2. 外部リアルActuator (SaaS Connector)
       │    ├─ メール自動配信 (Gmail) | タスク管理連携 | Googleカレンダー登録
       └─ 🛡️ 3. 実行ガバナンスエンジン (Execution & Gatekeeper)
            ├─ [承認後実行] (Human-in-the-Loop)
            ├─ [自律自動実行] (Full Autonomous under Policy)
            └─ [スケジュール実行] (Cron / Event Triggered)
                                       │
                                       ├──────────────────────────────────┐
                                       ▼                                  ▼
                     [ 📁 現実世界へのアウトプット ]         [ 📊 監査ログ ＆ 追跡 ]
                     ├─ PDF / Excel / PowerPoint生成        ├─ 100% ログの物理保存
                     ├─ メール送信・カレンダー確保          ├─ 成功率、失敗原因の自動診断
                     └─ 共有フォルダへの安全なアーカイブ     └─ エラー時のセルフリカバリ
```

### テーブル定義詳細

#### ① `reality_tasks` (現実世界実行キューマスタ)
実行される（された）すべてのアクションの内容、実行形態、および現在の物理ステータスを記録。
*   `id`: UUID (Primary Key)
*   `company_id`: UUID (Foreign Key - 組織スペース of ID)
*   `action_type`: VARCHAR(100) (実行する物理アクション。`FILE_GEN` (汎用ファイル), `PDF_GEN` (PDFドキュメント), `EXCEL_GEN` (スプレッドシート), `PPT_GEN` (プレゼン用PPT), `MAIL_CREATE` (メール下書き), `MAIL_SEND` (メール本番送信), `TASK_CREATE` (タスク連携), `CALENDAR_ADD` (カレンダー登録), `FOLDER_SAVE` (フォルダ階層保存))
*   `payload_data_json`: JSON (実行に必要な全パラメータ。メール本文、宛先、カレンダー日時、PDF埋め込み用プレースホルダーデータ等)
*   `execution_mode`: VARCHAR(50) (実行モード。`ON_APPROVAL` (人間の承認後), `AUTO` (ポリシーに基づく自律自動), `SCHEDULE` (cron scheduling))
*   `status`: VARCHAR(50) (ステータス。`PENDING` (承認待ち/実行待ち), `RUNNING` (生成/処理中), `SUCCESS` (完了), `FAILED` (エラー失敗))
*   `error_message`: TEXT (物理的な接続エラー、パースエラー、レートリミット超過などのエラー詳細スタックトレース)
*   `project_id` / `workflow_id`: UUID (Foreign Key, Nullable - 紐づくプロジェクト、自動化ワークフローのID)
*   `created_at` / `executed_at`: TIMESTAMP (タスク作成日、および実際の外部物理送信/生成完了日時)

#### ② `reality_templates` (オフィスドキュメント・フォーマットマスタ)
PDF、Excel、PowerPointの「流し込みデザインテンプレート」の格納場所を管理。
*   `id`: UUID (Primary Key)
*   `name`: VARCHAR(255) (テンプレート名。例: `経費精算書_公式`, `月間MEO報告用パワーポイント`)
*   `file_type`: VARCHAR(50) (ファイルタイプ。`PDF`, `XLSX`, `PPTX`, `CSV`, `HTML`)
*   `file_path`: VARCHAR(1024) (テンプレートファイルの実体保管先。`/templates/report_v2.pptx` 等)
*   `created_at`: TIMESTAMP

#### ③ `actuator_connections` (外部SaaS・Actuator接続状態マスタ)
外部 Workspace, Mail, WebDAV, Calendar などの接続認証状況。
*   `id`: UUID (Primary Key)
*   `provider`: VARCHAR(100) (接続先プロバイダー。`GOOGLE_WORKSPACE`, `MICROSOFT_365`, `SMTP_MAIL`, `AWS_S3`, `LOCAL_DRIVE`, `TRELLO`, `SLACK`)
*   `credential_ref_id`: VARCHAR(255) (安全に暗号化保存されたアクセストークンやAPIキーを復号するためのシークレットマネージャ側キーID。セキュリティのため、平文キーは絶対にこのDBには保存しない)
*   `status`: VARCHAR(50) (現在の接続確認ステータス。`ACTIVE` (通信良好), `EXPIRED` (トークン切れ), `FAILED` (認証失敗))
*   `updated_at`: TIMESTAMP (最終ヘルスチェック実行日時)

## 4. 実行設計 ＆ 安全パイプライン (Execution & Safety Engineering)

Reality Engineが、現実世界のファイルやメールを創出する際、システムのハルシネーション（AIの嘘）や機密漏洩、無限ループメール送信を未然に防ぐための**「セキュア・リアルアクション・ゲートウェイ（Actuator Safety Pipeline）」**の設計。

```
                    [ 実行キュー（reality_tasks）が起動 ]
                                   │
                                   ▼
                [ 1. セマンティック・フィルタリング（Sanitize） ]
      ├─ メール宛先のアドレス形式を厳密バリデーション (Regex Check)
      ├─ ドキュメント内、メール本文内の「個人情報（PII）の検出 ✕ マスキング」
      └─ 「プレースホルダー（例: {{name}}）の未置換ハルシネーション」を検出
                                   │
                                   ▼
                 [ 2. 権限・ポリシーチェック（Gatekeeper） ]
      ├─ 外部送信宛先ドメインの制限（例: 競合他社や見知らぬ個人ドメインへの無断送信を遮断）
      ├─ カレンダー登録の重複競合（Double-Booking）の警告
      └─ 当日の総送信数、総生成サイズが「レートリミット（安全上限）」を超えていないか
                                   │
                                   ▼
                   [ 3. 承認ルートの分岐（Route Split） ]
      ├─ (A) 顧客メール、高額コスト処理 ➔ 【Mission Control 承認待ち（PENDING）】へサスペンド
      └─ (B) 社内フォルダ保存、社内タスク、下書きメール ➔ 【即時自律自動実行（RUNNING）】へ
                                   │
         ┌─────────────────────────┴─────────────────────────┐
         ▼ (承認、または自動パス)                               ▼ (却下)
     [ 4. 物理執行（Actuator Execution） ]              [ 5. 差し戻しログ記録 ]
  - レンダラーによるPDF、Excelバイナリの物理生成           - ステータス: `REJECTED`
  - Gmail API、Workspace API を介した外部パブリッシュ     - 却下理由をツインに再学習
  - カレンダー、ドライブへの安全な書込
```

### ① ファイル動的レンダリングエンジン (The Dynamic Document Renderers)
*   **PDF レンダラー (PDF-Lib / Puppeteer Base)**:
    Reactで作成したRechartsやTailwindの美しい画面要素（SVG）をヘッドレスブラウザで高精細レンダリング（HTML to Canvas to PDF）し、ビジネスレポートにふさわしい、にじみのない高品質なベクトルPDFを自動出力します。
*   **Excel ピボット生成 (ExcelJS / SheetJS)**:
    AI Project Managerから送られてきた生の構造化データ、売上ログをピボット集計。社外用の「グラフ付き・数式付き・カラースタイリング適用済みのExcel報告書（.xlsx）」を動的にコンパイルします。
*   **PowerPointジェネレーター (PptxGenJS)**:
    AIの要約スライド構成から、「1枚目: 表紙」「2枚目: 競合分析」「3枚目: 改善策」の順に、自動でフォントスタイル、画像アセット、表、箇条書きテキストを整頓し、すぐにクライアントへプレゼンできる16:9スライド（.pptx）を動的に組み立てます。

## 5. 監査ログ ＆ 追跡設計 (Audit Logs & Tracking Architecture)

現実世界に物理的な作用を及ぼしたアクションは、万が一の法的な問題、セキュリティインシデント、またはバグによる誤送信が発生した際に、完璧に「いつ、誰の意志で、どのAIが、どのようなデータを、どこのIPから、どういう形式で送信したか」を秒単位で追跡・再現（リプレイ）できなければなりません。

### ① 監査ログデータ構造（`execution_audit_logs`）の3重防壁
1.  **不変記録（Append-Only Vault）**:
    監査テーブル `execution_audit_logs` は、SQLによる `UPDATE` および `DELETE` 操作をセキュリティルール（firestore.rules、DBトリガー等）にて物理的に完全に禁止（Append-Only）します。一度書き込まれた物理アクション記録は、システム開発者であっても絶対に改ざんできません。
2.  **暗号ハッシュ検証（Payload Fingerprint）**:
    送信されたメール、生成されたPDFのバイナリデータは、SHA-256を用いて指紋ハッシュ化され、`payload_hash` として記録されます。数年後に「この時送信したPDFの内容が書き換えられていないか」を1bitレベルで検証・再現できます。
3.  **アクターチェーン（Traceability Chain）**:
    アクションを指示したAIエージェントのUUID、それを許可したオーナーユーザー（人間）のUUID、およびトリガーとなったプロジェクト・ワークフローIDを一蓮托生で紐付け。

### ② アクション成否トラッキング ＆ エラー自己修復（Auto-Recovery Analytics）
*   **物理稼働率 (Physical Delivery Rate)**:
    $$\text{Delivery Success Rate} = \frac{\text{成功した物理アクション（SUCCESS）}}{\text{キューに投入された総実行タスク（SUCCESS + FAILED）}} \times 100$$
*   **セルフリカバリ（Self-Healing Pipelines）**:
    メール送信やカレンダー登録時に「APIのネットワーク一時障害（503/504等）」を検知した場合。
    Reality Engineは即座にタスクを停止せず、`reality_retries`（リトライ管理）にインサート。
    **指数バックオフ（Exponential Backoff）アルゴリズム**（1分後、5分後、15分後、60分後...）に基づき、自律的に外部接続を最大5回までリトライし、一時的なネットワーク切断による業務の不達（不履行）をゼロ化します。リトライがすべて失敗した際のみ、Mission Controlへアラートを通知します。

## 6. 将来拡張設計 (Future Extensibility Plan)

### ① Cross-Workspace Action Proxy (企業間・Workspace自律アクション委託・執行決済)
*   **複数の独立したAI Workspace環境同士が、お互いに「アクション権限（アーム）」を安全に委任・受託し、自律的に「タスクの委託」「成果物のドライブ保存」「カレンダーの直接抑え合い」をP2Pで決済・自動執行するネットワーク**:
    *   自社が提携会社とジョイントベンチャー（JV）を立ち上げた際。
    *   自社WorkspaceのReality Engineと、提携会社WorkspaceのReality Engineを「APIアソシエーション接続」。
    *   お互いのプライベート権限（Gmailや社内カレンダー等）は100%保護したまま、「JVプロジェクトフォルダの特定の共有ディレクトリ `/JV_Share/` へのファイル保存」および「お互いの取締役カレンダーの『空いている時間の相互自動ブッキング（相手のActuatorへの書き込み一時委託）』」を安全に自動執行。
    *   人間の日程調整やデータのダウンロード・メール添付という時間を完全撤廃し、企業間をまたぐ物理業務スピードを異次元へと加速させる「分散アクションアタッチネットワーク」へと拡張します。

### ② Autonomous Document Evolution (知財進化・自己改善ドキュメンテーション)
*   **ドキュメント生成エンジンが、過去に出力した成果物（PDF, Excel）に対する「人間（クライアント、またはオーナー）のフィードバック、手戻り、修正依頼」をナレッジグラフ経由で自律学習し、次のファイル生成時に『より好まれるレイアウト、より評価の高いスライド構成、より見やすいエクセルの配色』へと、テンプレートをAIが自ら自律デザイン変更・最適化していく自己改善ドキュメンテーションシステム**:
    *   AIが作成した「新宿個室カフェ売上Excel」に対し、オーナーが「もう少しフォントを大きくして、1行おきに薄いグレーの背景を敷いて（ゼブラパターン）」と修正チャットを送信。
    *   Reality Engineの「知財進化AI」がこの意図を瞬時にキャッチ。
    *   Excel出力用テンプレート（`reality_templates`）に紐づく動的CSS、列幅、およびセル装飾ロジックを、AIが裏側で自動リファクタリング。
    *   翌月、自動生成された「新宿カフェ売上Excel」は、オーナーが言わなくても、完璧なゼブラパターン、および最適化された大フォントで描画されて出力されます。
    *   人間がテンプレートを二度と作り直したり、修正指示を何度も出す必要をなくし、使えば使うほど「自社専用に完全にパーソナライズされた超一級品の成果物」が出力され続ける、究極の自己進化型リアリティ Actuator へと進化します。

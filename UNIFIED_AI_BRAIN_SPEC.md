# AI Workspace 「統合AIブレイン（Unified AI Brain）」機能設計仕様書

本書は、AI Workspaceにおいて、世界の主要な大規模言語モデル（OpenAI, Gemini, Claude, Perplexity, Grok, DeepSeek, Qwen, Kimi, Mistral, Llama, OpenRouter等）を単一のインターフェースから束ね、質問の特性に合わせて「最適なAIを自動選択（AI Router）」したり、複数の異なるAIを同時に走らせて「自動討論（Debate Mode）」を行わせることで、ハルシネーションを極限まで低減した最高精度の統合回答を出力する調停システム**「Unified AI Brain（統合AIブレイン）」**の機能設計仕様書です。

単体のAIモデルが抱えるバイアスや情報鮮度、ハルシネーションのリスクを、マルチモデルによる「合意形成アルゴリズム」および「ファクトチェック・エンジン」によって克服し、ビジネスにおける意思決定の絶対的精度を保証する知的脳幹として機能します。

## 1. 業務フロー ＆ 統合調停コンセプト

Unified AI Brainは、入力されたクエリを多次元的に分析する「AI Router」、選択されたモデル群を協調・対立させる「Multi-Model Engine」、出力の矛盾や信頼度を評価する「Consolidator & Fact Checker」の3層の論理エンジンによって、客観的かつ最高品質の回答を導き出します。

```
                       [1. ユーザーのクエリ入力]
              (質問、調査指示、検証したい仮説、対象URL)
                                  │
                                  ▼
                     [2. AI Router (ルーティング層)]
      ├─ クエリ特性分析 (コーディング、最新情報、倫理、スピード、価格)
      ├─ メモリ連携: 過去の関連質問・成功パターンを自動ロード (Memory Engine)
      └─ 最適なAIモデル（1つまたは複数）を自動選定 (コスト・速度・品質の最適化)
                                  │
                                  ▼
                   [3. 実行モードの選択 (Execution Mode)]
        ┌─────────────────────────┴─────────────────────────┐
        ▼                                                   ▼
 【A. Multi-Agent Mode (並行回答)】                 【B. Debate Mode (対立討論)】
  - 選択された複数AIが同時にクエリを処理             - AI Aが起案 ➔ AI Bが反論 ➔ AI Cが仲裁
  - 各AIの回答をテーブルで構造比較                   - 論点の絞り込みと合意形成プロセス
        │                                                   │
        └─────────────────────────┬─────────────────────────┘
                                  ▼
                 [4. Fact Check & Confidence Engine]
      ├─ 各AI回答間の情報の一致率 (Consensus Rate) を解析
      ├─ 出典ソースの信頼度、主張の矛盾点を自動監査 (Fact Check Engine)
      └─ 最終統合回答の「信頼度スコア (Confidence Score)」を算出
                                  │ (検証完了)
                                  ▼
                     [5. 統合回答出力 (Unified Output)]
      ├─ 統合回答 (Consolidated Answer)、各AIごとの原文、一致率
      ├─ 信頼度、鮮度、出典情報、および「推奨アクション (Recommended Actions)」
      └─ 成果物の保存: [Projects] [Knowledge Hub] [Files]
```

## 3. データベース設計 (Data Schema Design)

Unified AI Brainのルーティング、複数AIの回答データ、自動討論、ファクトチェック監査、および消費コストを管理するスキーマ。

```
                  +---------------------------+
                  |       brain_queries       |
                  +---------------------------+
                  | id (PK)                   |
                  | project_id (FK, Nullable) |
                  | user_id (FK)              |
                  | query_text (TEXT)         |
                  | route_strategy_json (JSON)|
                  | confidence_score (INT)    |
                  | consensus_rate (INT)      |
                  | consolidated_answer (TEXT)|
                  | created_at                |
                  +-------------+-------------+
                                | 1
                                |
                                ├─────────────────────────────┬─────────────────────────────┐
                                | 1..N                        | 1..N                        | 1..N
                  +-------------v-------------+   +-----------v-------------+   +-----------v-------------+
                  |    brain_ai_responses     |   |    brain_debate_turns   |   |    brain_fact_checks    |
                  +---------------------------+   +-------------------------+   +-------------------------+
                  | id (PK)                   |   | id (PK)                 |   | id (PK)                 |
                  | query_id (FK)             |   | query_id (FK)           |   | query_id (FK)           |
                  | model_name (VARCHAR)      |   | turn_number (INT)       |   | statement_text (TEXT)   |
                  | response_text (TEXT)      |   | speaker_model (VARCHAR) |   | verified_source_url     |
                  | latency_ms                |   | utterance_text (TEXT)   |   | truth_status (TRUE/etc) |
                  | token_cost_usd            |   | token_cost_usd          |   | source_credibility_score|
                  | is_selected_for_union     |   | created_at              |   | created_at              |
                  +---------------------------+   +-------------------------+   +-------------------------+
```

### テーブル定義詳細

#### ① `brain_queries` (ブレイン統合クエリマスター)
*   `id`: UUID (Primary Key)
*   `project_id`: UUID (Foreign Key, Nullable - 紐付けるプロジェクトID)
*   `user_id`: VARCHAR(100) (クエリを実行したユーザーID)
*   `query_text`: TEXT (ユーザーの入力質問文・指示)
*   `route_strategy_json`: JSON (AI Routerが決定した実行戦略。例: `{"models":["perplexity","claude-3-5","gemini-1-5"], "optimization":"quality"}`)
*   `confidence_score`: INT (最終統合回答の信頼度スコア。0〜100)
*   `consensus_rate`: INT (各モデル回答間での事実・意見の合意一致率。0〜100%)
*   `consolidated_answer`: TEXT (調停・要約エンジンが作成した、最終的な「一網打尽の統合回答」)
*   `created_at`: TIMESTAMP

#### ② `brain_ai_responses` (AIモデル個別回答データ)
クエリに対して、選択された各AIがそれぞれ返した生回答とレイテンシ。
*   `id`: UUID (Primary Key)
*   `query_id`: UUID (Foreign Key)
*   `model_name`: VARCHAR(100) (AIモデルの識別名。例: `openai/gpt-4o`, `anthropic/claude-3.5-sonnet`)
*   `response_text`: TEXT (そのモデルが返した生のテキスト回答)
*   `latency_ms`: INT (回答にかかった実ミリ秒時間)
*   `token_cost_usd`: DECIMAL(10, 6) (このLLM呼び出しで消費された実コスト)
*   `is_selected_for_union`: BOOLEAN (最終的な統合回答にこのモデルの意見が採用・マージされたか)

#### ③ `brain_debate_turns` (自動討論対話ログ)
Debate Modeが選択された場合の、AI同士の反論・調停の履歴。
*   `id`: UUID (Primary Key)
*   `query_id`: UUID (Foreign Key)
*   `turn_number`: INT (討論のターン数。例: `1`, `2`, `3`)
*   `speaker_model`: VARCHAR(100) (発言したAIモデル。例: `claude-3.5-sonnet`)
*   `utterance_text`: TEXT (討論内の発言)
*   `token_cost_usd`: DECIMAL(10, 6) (消費されたAPIコスト)
*   `created_at`: TIMESTAMP

#### ④ `brain_fact_checks` (ファクトチェック・情報源検証)
回答内の重要ファクトについて、ソースの有無や矛盾を監査したログ。
*   `id`: UUID (Primary Key)
*   `query_id`: UUID (Foreign Key)
*   `statement_text`: TEXT (検証対象となったファクト文。例: 「新宿3丁目にLa Loungeが実在する」)
*   `verified_source_url`: VARCHAR(1024) (検証に使用・確認されたWebのソースURL)
*   `truth_status`: VARCHAR(50) (事実検証ステータス。`TRUE` (事実確認済), `CONTRADICTORY` (他AIと矛盾あり), `UNVERIFIED` (ソースなし))
*   `source_credibility_score`: INT (ソースドメインの信頼度評価。0〜100)
*   `created_at`: TIMESTAMP

## 4. AIルーティング ＆ 最適化設計 (AI Router & Dispatcher)

Unified AI Brainが、最小のコストと最大の精度でクエリを処理するために、クエリの内容を自律解析し、最適なLLMへ処理をディスパッチする仕組み。

### ① クエリ解析・特徴抽出アルゴリズム (Query Categorization)
クエリが送信された瞬間、高速な判定モデル（Gemini 1.5 Flash等）が「クエリの特性（インテント）」を5角形チャート（Web探索、コード生成、長文要約、クリエイティブ、論理推理）で定性・定量分析します。

| 質問の特徴パターン | 判定される適性 | 推奨ルーティング・モデル |
| :--- | :--- | :--- |
| 「新宿の〇〇というカフェの営業時間は？」 | **最新の現実世界の事実 (Fresh Fact)** | Perplexity / Google Search Grounding |
| 「このコードのバグを修正して、TypeScriptで」 | **プログラミング・型定義 (Code & Logic)** | DeepSeek Coder / Claude 3.5 Sonnet / Qwen |
| 「新規事業のSWOT分析と事業計画書を作って」 | **複雑な戦略・構造化 (Reasoning & Strategy)**| Claude 3.5 / GPT-4o / Llama 3 70B |
| 「LP用のエモーショナルなキャッチコピーを」 | **クリエイティブ・心理訴求 (Creative)** | Gemini 1.5 Pro / Claude 3.5 |
| 「50万文字のPDF資料を全て読んで、論点を整理して」| **巨大コンテキスト・要約 (Extremely Long)** | Gemini 1.5 Pro (200万トークン窓) |

### ② 多次元最適化 (Multi-Criteria Optimization)
ユーザーの設定（コスト重視、品質重視、速度重視）に応じて、ルーティング方針を動的に切り替えます。

*   **コスト重視 (Cost-Optimized)**:
    軽度の質問や要約であれば、高価なフロンティアモデルを避け、100万トークンあたり$0.07レベルの極小コストモデル（DeepSeek, Gemini 1.5 Flash, Llama 3 8B）へ割り振ります。
*   **応答速度重視 (Latency-Optimized)**:
    即答性が必要な場合、最もスループット（T-P-S）が高く、レイテンシが極小なエッジ・スピードモデル（Gemini 1.5 Flash on Vertex, Groq API）にルーティングします。
*   **品質・精度重視 (Quality-Optimized)**:
    少し時間がかかっても絶対に誤り（ハルシネーション）を許さない場合、自動的に「複数AI同時実行 ➔ 自動討論 ➔ ファクトチェック」のパイプラインを起動します。

## 5. 統合エンジン ＆ 討論/ファクトチェックの仕組み

### ① Multi-Agent Mode（並行回答 ＆ 合意合流）
1.  **パラレル・ディスパッチ**:
    AI Routerが選択した異なるAPI（例：GPT-4o、Claude 3.5、Llama 3）へ同時に非同期リクエストを送信。
2.  **セマンティック・クラスタリング**:
    返ってきた複数の回答（`brain_ai_responses`）を文節・ファクトごとに分解。セマンティック類似度（Embedding cosine similarity）を測定し、「すべてのAIが事実だと合意している部分（一致率90%以上）」「一部のAIだけが主張している部分」「完全に意見が食い違っている部分」をマトリクス上にクラスタリング。
3.  **ブレンド・コンソリデーション**:
    Geminiが調停役（Consolidator）となり、合意部分をベースラインとしつつ、食い違い部分については「〜という説と、〜という指摘があり、検証結果からは〜の可能性が高い」という、極めて客観的で緻密なマージ回答（統合回答）を生成します。

### ② Debate Mode（自動討論と反論ループ）
AI同士が「客観的な真実」をめぐって複数ターンにわたり知的ディスカッションを行います。
*   **Turn 1 (主張)**: AI A（Perplexity）が「最新のWebデータに基づく競合カフェ情報」を出力。
*   **Turn 2 (反論・検証)**: AI B（Claude 3.5）がAI Aの出力を読み、「そのうち2店舗はクローズ情報あり、また個室カフェとアピールしているが実際は半個室である」と矛盾点・ハルシネーションを論理的に反駁（はんばく）。
*   **Turn 3 (再評価・合意)**: AI C（Gemini Pro）が議論の全履歴を監査し、矛盾のない客観的ファクト（「新宿三丁目で本当に深夜営業かつ完全個室のカフェはLa Loungeのみ」）を抽出し、結論を一本化します。

### ③ Fact Check & Confidence Engine (信頼度評価)
*   **ファクト抽出**: 回答内から「固有名詞」「数値」「営業データ」などの重要なアサーション（主張）を自動抽出。
*   **自動サーチグラウンディング**: Google Search / Perplexity API を裏側で駆動し、抽出された固有名詞や数値について、信頼性の高い一次ドメイン（大手ポータル、公式サイト、政府統計等）を逆引き検証。
*   **信頼度（Confidence Score）の計算**:
    以下の4要素から独自のアルゴリズムで100点満点のスコアを動的算出。
    $$\text{Confidence Score} = (0.4 \times \text{一致率}) + (0.3 \times \text{情報鮮度}) + (0.3 \times \text{出典ドメインの信頼度})$$
    算出したスコアをUI上でビジュアル表示し、信頼度が80点未満の場合は自動的に「警告（ハルシネーションの恐れあり）」をユーザーに明示します。

## 6. 他システム（プロジェクト、ナレッジ、OS）との統合・連携

Unified AI Brainは、AI Workspace全体の「究極の頭脳」として、他モジュールからのクエリをすべて引き受けます。

1.  **ナレッジハブ連携 (Knowledge Hub)**
    *   Unified AI Brainが生成した高精度の「統合回答」や、Webクローリングに基づく「ファクトチェック済みのデータ」は、1クリックでナレッジハブの指定プロジェクトフォルダへ「確証ノウハウ（Facts）」として保存されます。
    *   ナレッジハブでRAGチャットを行う際、自動的にUnified AI Brainのルーティングおよび調停機能が作動し、最も古いノウハウと最新のWeb情報をシームレスにブレンドして返答します。
2.  **プロジェクト連携 (Projects System)**
    *   特定の開発・制作プロジェクトに連動してクエリを実行。
    *   例えば、「このプロジェクトに格納されている間取り図とMEOデータを元に、ターゲット客層の妥当性を統合検証して」とクエリを投げると、Unified AI Brainがプロジェクト内の全ファイルを読み込み、複数モデルによる検証を実行。結果はプロジェクトのタイムライン（Timeline）に「Unified AI Brainによる競合多重監査完了」として記録されます。
3.  **Personal & AI Company OS 連携**
    *   OS内の「AI役員会（CEO、COO、CFO等）」や「AI参謀エージェント」が意思決定を迫られた際、裏側でこのUnified AI Brainの「Debate Mode」を自動キック。
    *   AI社員同士での衝突議論を、この統合エンジンの調停ロジックをベースに行うことで、経営方針のブレや凡ミスを未然に防ぎます。

## 7. 将来拡張設計 (Future Extensibility Plan)

### ① AI自身による外部API＆独自データソースの「動的プラグインマッピング」
*   **APIセルフインテグレーション**:
    AI Routerが「この質問に答えるためには、現在のLLMの知識やWeb検索だけでは不足しており、気象庁のAPI、または不動産レインズ（REINS）のデータが必要である」と自律判断した場合、自動的に外部の公開Web APIやデータベースへ接続する「プラグインアダプター」を自律生成し、データを吸い上げて統合回答にマージする仕組みを追加構築します。

### ② 分散型AI検証ネットワーク（Consensus Node Network）
*   **合意形成プロトコルのブロックチェーン連動**:
    さらに高い機密性や社会的証明が求められる監査資料（企業の決算発表、特許申請書の妥当性チェック等）を作成する場合、複数の独立したAIインスタンス（コンセンサス・ノード）へ暗号化されたクエリを送信。
    「ノードの3分の2以上が正しいと証明したファクトのみ」を成果物（PDF）として認可・発行し、スマートコントラクト上にその証明書（ハッシュ）を刻む、最高レベルの自律セキュリティ監査ネットワークへと進化させます。

# AI Workspace 「Template Marketplace（テンプレート・マーケットプレイス）」機能設計書

本書は、AI Workspaceにおける生産性を極限まで高めるためのコア機能**「Template Marketplace（テンプレート・マーケットプレイス）」**の画面設計、データベース設計、テンプレート管理アーキテクチャ、および実行・保存・拡張計画について定義する製品仕様設計書です。

ユーザーは、複雑なプロンプトを作成する専門知識がなくても、用途に応じたテンプレートを選択し、最小限の必要パラメータを入力するだけで、高度な分析・調査・資料作成・アプリ開発をワンクリックで実行・保存・再利用できます。

## 2. データベース・スキーマ設計 (Data Schema Design)

テンプレート情報、ユーザー固有のお気に入り、実行履歴、動的入力パラメータを管理するためのセキュアなリレーショナルテーブル設計。

```
                  +-------------------------+
                  |   template_categories   |
                  +-------------------------+
                  | id (PK)                 |
                  | name (e.g., Marketing)  |
                  | description             |
                  | parent_id (FK, Self)    |
                  +------------+------------+
                               | 1
                               |
                               | 1..N
                  +------------v------------+
                  |        templates        |
                  +-------------------------+
                  | id (PK)                 |
                  | category_id (FK)        |
                  | title                   |
                  | description             |
                  | system_instruction      |
                  | dynamic_fields (JSON)   |
                  | recommended_model       |
                  | is_preset (Boolean)     |
                  | author_user_id (FK)     |
                  | is_shared (Boolean)     |
                  | created_at              |
                  +------------+------------+
                               | 1
                               |
                               ├─────────────────────────────┐
                               | 1..N                        | 1..N
                  +------------v------------+   +------------v------------+
                  |     user_favorites      |   |   template_executions   |
                  +-------------------------+   +-------------------------+
                  | id (PK)                 |   | id (PK)                 |
                  | user_id                 |   | template_id (FK)        |
                  | template_id (FK)        |   | user_id                 |
                  | created_at              |   | input_values (JSON)     |
                  +-------------------------+   | execution_status        |
                                                | started_at              |
                                                | completed_at            |
                                                +------------+------------+
                                                             | 1
                                                             |
                                                             | 1..N
                                                +------------v------------+
                                                |     saved_outputs       |
                                                +-------------------------+
                                                | id (PK)                 |
                                                | execution_id (FK)       |
                                                | content_type            |
                                                | content_body (TEXT)     |
                                                | file_assets (JSON)      |
                                                | version_number          |
                                                | created_at              |
                                                +-------------------------+
```

### テーブル詳細

#### ① `template_categories` (カテゴリ階層管理)
*   `id`: VARCHAR(50) (Primary Key. 例: `seo-analysis`, `app-builder`)
*   `name`: VARCHAR(100) (表示名。例: `SEO分析`, `MEO分析`)
*   `description`: VARCHAR(512) (カテゴリ概要)
*   `parent_id`: VARCHAR(50) (親カテゴリ。親が存在しない場合はNULL。例: `marketing-analysis`)

#### ② `templates` (テンプレートマスター)
プレセットおよびユーザー定義のカスタムテンプレート定義。
*   `id`: UUID (Primary Key)
*   `category_id`: VARCHAR(50) (Foreign Key)
*   `title`: VARCHAR(255) (テンプレート名)
*   `description`: TEXT (詳細説明、期待される成果物)
*   `system_instruction`: TEXT (このテンプレート特有のエージェントシステムプロンプト。非公開・カプセル化対応)
*   `dynamic_fields`: JSON (フォームに動的描画する項目定義。例: `[{"id":"shop_name", "label":"店舗名", "type":"text", "required":true}]`)
*   `recommended_model`: VARCHAR(100) (推奨AIモデル、または自動ルーター設定)
*   `is_preset`: BOOLEAN (システム初期状態の組み込みテンプレートかどうか。`true` の場合は編集・削除不可)
*   `author_user_id`: VARCHAR(100) (カスタムテンプレートを作成したユーザーのID)
*   `is_shared`: BOOLEAN (同じ組織・チームの他ユーザーと共有されているかどうか)
*   `created_at`: TIMESTAMP

#### ③ `user_favorites` (お気に入り登録)
*   `id`: UUID (Primary Key)
*   `user_id`: VARCHAR(100) (ユーザーID)
*   `template_id`: UUID (Foreign Key)
*   `created_at`: TIMESTAMP (登録日時)

#### ④ `template_executions` (テンプレート実行・パラメータ履歴)
ワンクリックでの「再実行」「パラメータ復元」を可能にするための実行履歴。
*   `id`: UUID (Primary Key)
*   `template_id`: UUID (Foreign Key)
*   `user_id`: VARCHAR(100) (実行者ユーザーID)
*   `input_values`: JSON (実行時に入力されたパラメータキーと値のペア。例: `{"shop_name":"新宿La Lounge","area":"新宿3丁目"}`)
*   `execution_status`: VARCHAR(50) (ステータス: `RUNNING`, `SUCCESS`, `FAILED`)
*   `started_at` / `completed_at`: TIMESTAMP

#### ⑤ `saved_outputs` (成果物・バージョン保存)
テンプレート実行によって生成された最終成果物および中間成果物の実体。
*   `id`: UUID (Primary Key)
*   `execution_id`: UUID (Foreign Key)
*   `content_type`: VARCHAR(100) (成果物の種類: `MARKDOWN`, `HTML`, `FILE_ZIP`, `EXCEL_DATA`)
*   `content_body`: MEDIUMTEXT (成果物のテキスト実体、またはHTML・マークダウン)
*   `file_assets`: JSON (生成された物理ファイル、PDF、Excel、PPT、ZIPへのストレージパス配列)
*   `version_number`: INT (同じ実行履歴から「再生成」した際の世代管理用)
*   `created_at`: TIMESTAMP

## 3. カテゴリ構成 & テンプレート管理設計 (Preset Templates Registry)

Marketplaceに事前登録されている主要テンプレートと、AIへの組み込みパラメータマッピング。

### 📁 1. AI Search (AI探索)
*   **一般検索**: `Google Search API`を用いた汎用クローリング・最新ソース要約。
*   **AI検索**: 生成AIが得意とする要約・コンセプト検索。
*   **最新情報検索**: 時間指定フィルタを活用した、直近24時間・1週間のニュース追跡。
*   **ファクトチェック**: 矛盾する複数Webソースを突き合わせ、情報の正確性と裏付け根拠（エビデンスURL）を提示。
*   **情報収集**: 指定ドメインやサイトからの効率的なテキスト・リスト抽出。

### 📁 2. Research (リサーチ＆競合分析)
*   **Deep Research (詳細調査・業界調査・市場調査)**: バックグラウンドで複数の自律ステップを実行し、対象業界のバリューチェーン、市場規模、主要トレンドを網羅した長文レポートを自動生成。
*   **Competitor Analysis (競合分析・ベンチマーク分析)**: 競合他社の製品、価格、機能、プロモーション手法を比較し、SWOTおよび3Cフレームワークの形式でマッピング。
*   **Customer Research (顧客分析・ペルソナ分析)**: ターゲット市場のニーズ、ペインポイント、検討プロセスを定義し、ユーザーペルソナシート（3パターン）を自律描画。

### 📁 3. Marketing Analysis (マーケティング施策分析)
*   **SEO Analysis**: 指定されたWebサイトURLとターゲットキーワードから、メタタグ診断、見出し構成、キーワードのトピッククラスター化、競合他社サイトとの比較、および「改善記事の骨子・構成案」を自動出力。
*   **AIO Analysis (AI検索最適化分析)**:
    *   *ChatGPT / Gemini / Perplexity 表示分析*: 各種主要対話AIのAPIに「自社サービス名」や「●●のおすすめ」などのプロンプトを投げ、どのようなコンテキストで自社が引用・言及されるかを定量評価。
    *   *AI引用分析 ＆ 改善提案*: AIに優先的にインデックスされ、好意的に紹介されるための「Web構造化マークアップ、推奨される解説テキスト定義、信頼性シグナル(E-E-A-T)強化施策」を出力。
*   **MEO Analysis (ローカル検索最適化分析)**:
    *   *入力項目*: `店舗名`, `GoogleビジネスプロフィールURL`, `業種`, `地域`
    *   *出力アセット*: 口コミ評価センチメント分析、キーワード最適化状況の「現状診断」、新宿・主要エリアでの競合店舗とのマップ検索「競合比較」、プロフィールの登録漏れ等の「課題」、投稿内容・写真・レビュー返信の「改善施策」を「優先順位付き実行ロードマップ」として一挙出力。
*   **SNS Analysis**: Instagram、TikTok、YouTube、X(旧Twitter)の人気投稿、アカウントのハッシュタグ・傾向分析と、エンゲージメントを高めるための「コンテンツカレンダー・台本構成」作成。
*   **Advertising Analysis**: Google広告、Meta(Facebook/Instagram)広告、TikTok広告の、ターゲットオーディエンス設定案、効果的なクリエイティブの切り口（コピー・画像構図指示）の自律作成。

### 📁 4. Business (ビジネス・戦略ドキュメント)
*   **営業提案書作成**: 顧客ターゲットの課題分析、自社のバリュープロポジション、サービス紹介、および導入ロードマップを網羅したプレゼンテーション構成案を自動生成。
*   **事業計画書作成** / **マーケティング戦略立案** / **新規事業分析(5フォース分析)**。

### 📁 5. Documents (実務ドキュメント & オフィスファイル)
*   **PowerPoint / Word / Excel / PDF 自動生成**: 記述されたドラフトを基に、表計算フィルタ付きExcel、スタイリッシュなスライドショー、およびエグゼクティブサマリーPDFの作成命令を実行レイヤー（Execution Layer）にディスパッチ。
*   **議事録作成** / **要件定義書作成**。

### 📁 6. Creation (プロトタイプ ＆ アプリ開発)
*   **Website Builder (LP・コーポレートサイト作成)**: 要件から、モダンなTailwind CSSおよびJavaScriptを用いたシングルページのHTMLコード（インタラクティブ、レスポンシブ）を生成。
*   **App Builder (Webアプリ・管理画面生成)**: データ操作可能なCRUDモックアップおよびSupabase SQL定義の出力。
*   **CRM Builder (顧客管理・営業管理)**: リード追加、ステータス変更、メモ記録がその場で動くプロトタイプの自動構成。
*   **AI Tool Builder (AIツール・SaaS生成)**: 独自のインプットを入力すると、裏側でGemini API等をコールする「マイAIツール」のフロントエンドコードを自動エクスポート。

### 📁 7. Agents (協調エージェントパッケージ)
*   **調査Agent** / **分析Agent** / **営業Agent** / **マーケティングAgent** / **開発Agent**:
    あらかじめ各担当者の役割定義、ゴール基準をセットした自律マルチエージェントをワンクリックでワークスペースにデプロイし、常駐型アドバイザーとして稼働させます。

## 4. 実行フロー設計 (Execution Flow Design)

ユーザーがテンプレートを選択してから、成果物が生成・保存されるまでのシーケンス定義。

```
[1. ユーザーが Marketplace からテンプレートを選択]
                      │
                      ▼
[2. dynamic_fields (JSON) を読み込み、入力フォームを動的描画]
                      │
                      ▼
[3. ユーザーがパラメータを入力し、[⚡ 実行] をクリック]
                      │
                      ▼
[4. 実行レコード (template_executions) を作成、ステータスを RUNNING に]
                      │
                      ▼
[5. AIルーティングエンジンが起動]
      ├─ テンプレート定義から recommended_model および system_instruction をロード
      ├─ ユーザーの入力値をシステム指示・プロンプトテンプレートにマッピング
      └─ 実行タスクを Execution Layer (実行レイヤー) の各ワーカーへ発行
                      │
                      ▼
[6. 各ワーカー自律実行（Web検索、ドキュメント/コードビルド等）]
                      │
                      ▼
[7. 生成成果物の検品 (QA Agent 監査)] ──► [不正・エラー検知] ──► [AIリテイク自動実行]
                      │ (パス)
                      ▼
[8. 成果物データ・アセットを saved_outputs にコミット]
                      │
                      ▼
[9. 実行レコードを SUCCESS に更新、ユーザーにリアルタイム完了通知]
                      │
                      ▼
[10. ユーザーが成果物をプレビュー確認、ZIP/Office/PDFでエクスポート]
```

## 5. 保存・複製 ＆ 共有設計 (Storage, Duplication & Sharing)

優れたテンプレートや実行パラメータを無駄にせず、繰り返し使えるようにするための保存メカニズム。

### ① ワンクリック複製 (Template Duplication)
*   Marketplaceに公開されている既存のプリセットテンプレートを、ユーザーが「マイテンプレート」として1クリックで複製・保存可能。
*   複製したテンプレートのプロンプトやシステム指示（System Instruction）を、自分の業務プロセスに合わせて自由に書き換え、カスタムテンプレートとして再登録できます。

### ② カスタムテンプレートの組織共有 (Enterprise Sharing System)
*   ユーザーが自作したカスタムテンプレートは、プライベート（自分専用）として使うことも、トグルスイッチ1つで「チーム/組織内（Organization-wide）」に共有することも可能。
*   組織内に共有されたテンプレートは、同僚のダッシュボードの「組織共有」エリアに即座に表示され、社内のベストプラクティス（例：「山田さんの作成した競合分析テンプレート」など）を瞬時に標準化・共有できます。

### ③ 実行パラメータの固定保存 (Execution Preset)
*   同じMEO分析テンプレートを「新宿店」「渋谷店」「池袋店」の複数店舗用に繰り返し使用する場合、一度入力した「GoogleビジネスプロフィールURL、業種、地域」のパラメータを「プリセット」として名前付きで保存可能。
*   次回から、店舗のパラメータをロードするだけで、ワンクリックで最新データに基づく再実行（分析の定期アップデート）を素早く行えます。

## 6. 将来拡張設計 (Future Extensibility)

ビジネスモデルや分析ドメインの拡張に伴い、マーケットプレイスをスピーディにグロースさせるためのモジュール設計。

### ① 今後の追加予定カテゴリ（拡張モジュール）のインテグレーション
*   **EC分析**: Amazon（露出・レビューセンチメント）、楽天市場（検索トレンド）、Shopify（ストア分析・SEO診断）を対象とした、EC特化型分析・商品紹介文作成テンプレート。
*   **不動産分析**: 指定エリアの競合賃貸、分譲価格相場、地域属性、および入居募集用アピールポイント作成。
*   **採用分析**: 競合他社の求人票（Job Description）比較、求職者のインサイト分析、スカウトメール作成。
*   **財務分析**: 決算書PDFや試算表データをセキュアに解析し、収益性・安全性の指標グラフを自動構成するテンプレート。

### ② サードパーティSDK・APIフックのオープン化
*   テンプレートの入力パラメータおよび出力フォーマットを汎用API（Webhooks）で公開。
*   将来的に外部のRPAツール（Zapier, Make）や、社内の基幹システムからマーケットプレイスのテンプレートをAPIコールで直接起動できるように設計します。これにより、「毎週月曜日に自動で競合AIOレポートを実行してPDFをSlackに投げる」といった超自律的なエンタープライズ統合を容易に実現可能にします。

# Project Bible - Chapter 02: Supreme Design System (至高のデザインシステム)
**Classification:** Universal Design Constitution / Core Visual Standards  
**Version:** 1.0.0  

---

## 1. 哲学 (Design Philosophy)

AI Workspaceにおけるデザインは、単なる「美しさ（Aesthetics）」に留まらない。デザインとは知的生産性を極限まで高めるための「認知工学（Cognitive Engineering）」である。本デザインシステムは、以下の6つのコア体験を同時に満たすことを絶対条件とする。

*   **「理解できる (Understandable)」**: 目にした瞬間に、情報の階層、関係性、現在の状況が脳内にマッピングされる。
*   **「迷わない (Unambiguous)」**: 操作の対象が自明であり、誤操作や認知的混乱を誘発する余計な視覚情報が存在しない。
*   **「早い (Efficient)」**: インタラクションの遅延、不必要な階層移動、余計なクリックを排除し、思考速度のまま作業が進行する。
*   **「高級 (Luxury)」**: 素材感、ピクセル単位の余白、フォント選定、滑らかな曲線が、最高峰のデジタルワークスペースを演出する。
*   **「未来 (Future)」**: 従来のOSのメタファーを脱却し、AIと人間の共生を象徴するインタラクティブなUIを展開する。
*   **「知的 (Intellectual)」**: 提示される情報の密度と整理方法が、知的興奮と創造的インスピレーションをユーザーに与え続ける。

## 2. 視覚の絶対規則 (Core Visual Rules)

### 2.1 3秒の第一印象則 (The 3-Second First Impression Rule)
ユーザーがアプリケーションを起動して、または新しいタブで成果物を見た瞬間、**「3秒以内」に「これは他のどのツールとも次元が違う、極上の知能空間である」と本能的に感じさせなければならない。**
*   **実装仕様**: ローディング開始時の静謐なインディゴフェードイン、一切のチープな境界線の排除、贅沢な「知的余白（Intellectual White Space）」の存在感、そして等幅フォントで刻まれるシステムの「思考パルス」により、圧倒的なプロフェッショナリズムを印象付ける。

### 2.2 自己説明的UI (The Zero Learning Rule)
説明書やツールチップの熟読、チュートリアルツアーを一切必要としない。初めて操作するユーザーでも、本能的に直感で動かすことができる。
*   **実装仕様**: 選択可能なカードはホバーで2px浮き上がり（Framer Motionイージング）、次に入力すべきエリアは呼吸するように淡くパープルに明滅。ユーザーの物理的な視線の移動を計算し、視覚誘導線（Visual Guideline）をレイアウト上に自然に構築する。

### 2.3 シングルスクリーン原則 (The One Screen Rule)
AI Workspaceの基本操作は、**ホーム画面（Dashboard Stage）の1枚で完結する。** 
*   **実装仕様**: 複雑な階層型メニューや設定画面の森をユーザーに探索させない。現在実行しているタスク、編集中の制作物、ユーザーに必要なアクションだけを、AIがその都度ホーム画面のステージ上に動的かつスマートに展開・収縮させる。

### 2.4 AIファースト原則 (The AI First Rule)
本システムにおいて、静的なナビゲーションメニュー、手動の設定項目、一般的な検索ボックスは脇役に過ぎない。
*   **実装仕様**: 常に「AI」がすべての主導権を握る。ユーザーが設定を探す前に、AIがユーザーの振る舞いから設定の変更を提案する。ユーザーが検索をかける前に、AIが関連情報をダッシュボード上にすでに配備している。インターフェースの中心には常にAIの「知能（Orchestration Portal）」が位置する。

## 3. 空間・情報・色調の美学 (Spatial & Visual Hierarchy)

### 3.1 知的余白の法則 (The White Space Rule)
情報を詰め込むことは知的ではない。理解の速度は「情報の少なさ（洗練度）」と「適切な余白（Negative Space）」に比例する。
*   **レイアウト仕様**: コンテンツ密度の高いデータやチャートを囲む外枠には、必ず `p-6` (24px) 以上の豊かなパディングを確保。情報のグループ間には `space-y-8` (32px) の空白を設け、脳が個別の「思考ブロック」をストレスなく処理できるように物理的距離を与える。

### 3.2 カード・ファーストの原則 (The Card First Rule)
人間は、フラットで長いプレーンテキストを読まされることを最も嫌う。情報は視覚的な「カード」としてカプセル化されなければならない。
*   **情報進化仕様**: 
    1.  **文章**を減らし、セマンティックな役割ごとに境界線で区切られた**「カード」**へ昇華。
    2.  カードの内部構造をさらに抽象化し、関係性やプロセスを示す**「図（Diagram）」**へ昇華。
    3.  図のインタラクションを通じて、ユーザーの脳に直感的な**「理解（Understanding）」**を瞬時にインストールする。

```
[プレーンな長文文章]  ──(カード化)──>  [整理された思考カード]  ──(ビジュアル化)──>  [自律SVG図解]  ──(納得)──>  【瞬時の理解】
```

## 4. 色彩とタイポグラフィの精神 (Color & Font Semiotics)

### 4.1 色彩哲学 (The Semantic Color Scale)
本システムにおける「色」は、装飾ではなく「意味とシステムの心理状態」を表現する絶対的なシグナルである。

| カラー名 | 意味・役割 | 代表クラス | 心理的効果 |
| :--- | :--- | :--- | :--- |
| **安心 (Trust / Slate Blue)** | システムの正常稼働、静的な信頼性、基本テキスト | `text-slate-900`<br>`bg-slate-50` | 思考を沈静化させ、システムへの確固たる安心感を醸成する。 |
| **成功 (Success / Emerald)** | タスク合格、ビルド成功、高スコア（95点以上） | `text-emerald-600`<br>`bg-emerald-50` | 達成感と安心を与え、次の生産的アクションへの動機を促す。 |
| **注意 (Attention / Amber)** | 改善の余地、矛盾の検出、一時的な保留、再検証中 | `text-amber-500`<br>`bg-amber-50` | 視野を狭めることなく、思考の微調整と確認を促す警告。 |
| **AI思考 (Cognitive / Purple)** | オーケストレーター議論中、推論、ディベート実行中 | `text-purple-600`<br>`bg-purple-50` | 「AIが裏側で知的かつ深遠に活動している」高揚感を演出。 |
| **最高品質 (Excellence / Gold)** | 100点満点評価、プレミアムなインサイト、特注成果物 | `text-yellow-600`<br>`bg-yellow-50` | 妥協のない最高峰の成果、ステータスの高さを表現する輝き。 |
| **プレミアム (Luxury / Deep Black)** | 最上位司令部のフレーム、最重要インジケータ | `bg-slate-950`<br>`text-white` | 重厚感、プロフェッショナル向けOSとしての威厳と先進性。 |

### 4.2 文字の読解から理解への転換 (Font Semiotics)
文字は「読ませるもの」ではない。形によって「直感的に意味を理解させる」記号である。
*   **Display Pairings**: 見出しには、幾何学的で未来感のある `Space Grotesk` や `Outfit` を採用。トラッキング（字間）をわずかに狭め（`tracking-tight`）、高い文字の密度でプロフェッショナルな知性をアピール。
*   **Data & System Pulse**: AIが推論した確率（Confidence Index）、実行ミリ秒、APIステータス、ログ出力などのシステムインジケータには、必ず等幅フォント（`JetBrains Mono` または `Fira Code`）を適用。無機質な美しさが客観的な正確性を担保する。

## 5. モーション＆ビジビリティ (Dynamic Reassurance)

### 5.1 理解を助けるアニメーション (Motion Philosophy)
モーションは「お飾り」ではない。UI要素の「登場理由」「移動」「接続関係」を人間に分からせるためのトランジションガイドである。
*   **実装基準**: `motion` (motion/react) を用いた `cubic-bezier(0.16, 1, 0.3, 1)` の流麗なイージングを徹底。
    -   **ポップアップ・ドロワー**: 拡大ではなく、わずかに（8px）下から浮き上がるフェードイン。
    -   **コネクタの接続**: 処理データが別のカードに流れる際、淡いインディゴの光波がライン上を自律的に移動（SVGダッシュエフェクト）。

### 5.2 知能の可視化 (AI Visibility Specification)
AIが「今、何を考えているか」「どの段階で、なぜ待たされているか」が常に美しく可視化されていなければならない。
*   **実装基準**:  
    AIの思考フェーズを「Ingestion ➔ Semantic Routing ➔ Debate Swarm ➔ Truth Grounding ➔ Score Validation」のタイムラインに分解し、現在の稼働中のフェーズをパープルのパルスインジケータで静かに実況。ブラックボックスによる不信感を、知的エンターテインメントと深い安心感へと昇華させる。

# AI Workspace 「目標達成・逆算プランニングエンジン（Goal Management Engine）」機能設計仕様書

本書は、AI Workspaceにおける経営目標、営業目標、開発スケジュール、学習計画、個人のライフスタイル目標などに至るあらゆる「ゴール（目標）」を設定し、AIが現在値と目標値のギャップから「逆算型のアクションプラン（行動計画）」を自動生成・アサイン、さらにリアルタイムの実行ログから「達成予測・リスク分析・改善提案」を自律的に繰り返す統合目標達成管制システム**「Goal Management Engine（ゴール・マネジメント・エンジン）」**のシステム構成、データスキーマ、進捗管理設計、および画面設計を定義した製品設計仕様書です。

単なるToDoリストや静的なKPI管理ツールとは異なり、目標を設定した瞬間、AIが「タスク・ワークフローの自動生成」「担当エージェントの自動配備」「Tool Connector Hubを介した外部データの自動KPI同期」を行い、毎日24時間Workspaceの稼働からリアルタイムに達成度を予測する、AI Workspace全体の**「戦略・ナビゲーションシステム（Strategic Navigation Brain）」**の役割を担います。

## 1. 業務フロー ＆ 逆算プランニングコンセプト (Overview & Concept)

Goal Management Engineは、目標（定量的、定性的）を設定すると、AIが自動的にマイルストーンやマクロステップに分解します。さらに各ステップを既存のAIエージェント、または自動ワークフローの「実行命令」としてマッピングし、実務の進捗から「達成予測とボトルネック（リスク）」を自律学習します。

```
                       [ 1. 目標設定 (Goal Settings) ]
        (売上・利益・案件数・集客・開発・学習・健康・自由設定などのKPI設定)
                                      │
                                      ▼
                   [ 2. AI逆算プランニング (AI Planning) ]
      ├─ 現在値と目標値のギャップ分析 ➔ 自動マイルストーン分割
      ├─ 優先順位（Priority）の動的決定 ➔ クリティカルパス（主動線）の選定
      └─ 必要リソース、担当エージェント、必要なツールの自動特定
                                      │
                                      ▼
                 [ 3. 実行 ＆ 自動KPI同期 (Auto-Tracking) ]
      ├─ エージェント・ワークフロー・オートメーションとタスクの紐付け
      ├─ Tool Connector Hub経由で外部SaaS（GA4、Sheets、MEO等）から実績自動ロード
      └─ 進捗（Progress）と目標達成率（Achievement Rate）のリアルタイム集計
                                      │
                                      ▼
                [ 4. 予測 ＆ ボトルネック検知 (AI Forecast) ]
      ├─ 過去の進捗ペースから「達成期限内の予測（Forecast）」を算出
      ├─ 遅延リスク・ボトルネック（エラー、アサイン不足、コスト不足）の検知
      └─ AIによる自動「改善提案（Recommendations）」およびプラン自動再編
                                      │ (フィードバックループ)
                                      └──────────────────────────┘
```

### テーブル定義詳細

#### ① `goals` (目標定義マスタ)
*   `id`: UUID (Primary Key)
*   `company_id`: UUID (Foreign Key - 組織スペースのID)
*   `title`: VARCHAR(255) (目標の名称。例: `新宿個室カフェ La Lounge オープン ＆ 初期集客`)
*   `category`: VARCHAR(100) (分類。`REVENUE`, `PROFIT`, `PROJECTS`, `MARKETING`, `DEVELOPMENT`, `LEARNING`, `HEALTH`, `CUSTOM`)
*   `description`: TEXT (目標の概要、定性的な成功定義)
*   `start_date` / `target_date`: TIMESTAMP (目標期間)
*   `current_progress_rate`: DECIMAL(5, 2) (マイルストーンの完了割合。0.00〜100.00 %)
*   `current_achievement_rate`: DECIMAL(5, 2) (KPI数値ベースの達成率。0.00〜100.00 %)
*   `forecast_status`: VARCHAR(50) (AIが算出した最新の達成予測ステータス。`ON_TRACK` (順調), `WARNING` (リスクあり), `BEHIND` (遅延傾向))
*   `created_at` / `updated_at`: TIMESTAMP

#### ② `goal_kpis` (定量KPI指標 ＆ 外部自動同期設定)
目標の達成度を測るための定量的な数値基準と、外部データ連携（Tool Hub経由）のマッピング。
*   `id`: UUID (Primary Key)
*   `goal_id`: UUID (Foreign Key)
*   `metric_name`: VARCHAR(255) (指標名。例: `Xフォロワー数`, `MEO検索順位`, `累計売上`)
*   `target_val`: DECIMAL(15, 4) (目標とする最終数値。例: `3.00` (MEO 3位以内))
*   `current_val`: DECIMAL(15, 4) (最新の実績値。例: `5.00` (現在5位))
*   `sync_source`: JSON (Tool Connector Hub経由の自動更新配線パラメータ。例: `{"provider": "GOOGLE_BUSINESS_PROFILE", "endpoint": "/v4/locations/...", "json_path": "$.local_rank"}`)
*   `updated_at`: TIMESTAMP (実績データの最終自動更新日時)

#### ③ `goal_action_plans` (逆算マイルストーン計画)
目標を逆算してAIが細分化した具体的な行動プラン・タスクリスト。
*   `id`: UUID (Primary Key)
*   `goal_id`: UUID (Foreign Key)
*   `step_order`: INT (プラン実行順序。1, 2, 3...)
*   `title`: VARCHAR(255) (マイルストーン名。例: `MEO対策の実行と口コミ返信の全自動化`)
*   `description`: TEXT (このマイルストーンの具体的なタスク要件、期待される成果物)
*   `status`: VARCHAR(50) (ステータス。`COMPLETED`, `IN_PROGRESS`, `PENDING`)
*   `start_date` / `due_date`: TIMESTAMP (マイルストーンの期日)
*   `created_at`: TIMESTAMP

#### ④ `goal_agent_assignments` (担当AIエージェント/チームマッピング)
各マイルストーンを実行、または全体の進行責任（Leader）を持つAI社員の配備マトリクス。
*   `id`: UUID (Primary Key)
*   `goal_id`: UUID (Foreign Key)
*   `agent_id`: UUID (Foreign Key - 割り当てられた `marketplace_agents` 等のID)
*   `role_desc`: VARCHAR(255) (チーム内でのこの目標に対する役割。例: `MEO対策担当`, `プロジェクト統括リーダー`)
*   `created_at`: TIMESTAMP

## 4. 進捗管理・予測・リスク分析ロジック (Progress & Forecasting Logic)

目標の達成予測を単なる「タスクの完了割合」ではなく、数理的かつ時間軸データに基づいてAIが自律評価するためのアルゴリズム設計。

```
[ KPI 過去時系列データ ] ➔ AI線形/非線形回帰 (Trend Projection)
                                 │
                                 ▼
                     +-----------------------+
                     |  AI Forecast Engine   |
                     +-----------------------+
                     | [分析 1] 現在ペースでの達成予想日 ➔ 目標期日と比較
                     | [分析 2] 外部要因（エラー、進捗バグ）の加味
                     | [分析 3] ギャップ量に対するリソース推奨算出
                     +-----------┬-----------+
                                 │ (予測出力)
                                 ▼
                       [ 達成ステータス判定 ]
       ├─ ON_TRACK : 予測達成日が 目標期日 以前
       ├─ WARNING  : 予測達成日が 期日後 1〜5日 の範囲 (軽微な遅延)
       └─ BEHIND   : 予測達成日が 期日後 5日以上 または 完全に停滞 (イエロー/レッドアラート)
```

### ① 予測達成可能性（Probability Score）の算定
*   **計算モデル**:
    過去の実績値更新履歴 $D = \{(t_1, v_1), (t_2, v_2), \dots, (t_n, v_n)\}$ から、時間 $t$ に対するKPI値 $v$ の改善率（スロープ）を動的に算出し、目標値 $V_{target}$ に到達する予想時期 $T_{forecast}$ を特定します。
    $$T_{forecast} = t_n + \frac{V_{target} - v_n}{\text{平均日次改善率 } \Delta v}$$
    $T_{forecast}$ が目標期限 $T_{target}$ より手前にある確率分布（信頼区間 95%）から、**「達成確度スコア（0〜100%）」**を毎日算出します。

### ② AI自律プランニング・リプランニング（Auto-Replanning Trigger）
*   **作動条件**:
    目標ステータスが `BEHIND` に転落した場合、または特定マイルストーンステップの進捗期日が超過した場合、バックグラウンドの「戦略コンサルタントAI（COO Agent）」が自動でトリガーされます。
*   **自己修正アクション**:
    1.  **ボトルネック解析**: 該当目標に紐づくナレッジグラフ、エージェント利用ログをスキャンし、遅延の真因（例：「NextJSコーダーのコンパイルエラーが10回連続発生している」「GBP APIのOAuthトークンが切れている」）を特定。
    2.  **改善アクションプラン生成**:
        - 担当エージェントの増員・スキル変更（Marketplaceから別エージェントを召喚）。
        - ルーティングLLMのアップグレード（Gemini Flashから、より賢いClaude 3.5 Sonnetへ一時的に引き上げ、難解なバグを一瞬で突破させる）。
        - スケジュール（Due Date）の現実的な再シミュレーションとマイルストーンの再構築。
    3.  **人間への通知・一括承認**: 改善されたリプラン構成案を「Mission Control Center」へ承認要請としてディスパッチ。人間が承認ボタンを押した瞬間に、チームの再配備とタスクが裏側で自動適用されます。

## 5. 他システム（OS、ツイン、ブレイン、ハブ、グラフ）との統合

Goal Management Engineは、AI Workspace全体の「戦略羅針盤」として、全モジュールの実行ベクトルを1つの目標方向に収束させる統合調整を行います。

1.  **AI Company OS / Personal AI OS 連携**
    *   AI Company OSの組織図内の「各部署のOKR（目標）」として、本Engineが統合配線されます。
    *   経営目標が決定すると、関連部署にアサインされたAIチーム（Development Team、MEO Team等）へ、必要マイルストーンが自動的にタスクとして分配（アサイン）されます。
2.  **Tool Connector Hub 連携 (KPIデータの全自動吸引)**
    *   目標に紐付けられた定量KPI（例: Google Analytics4のPV、WordPressの投稿数、Stripeの決済額）は、Tool Connector Hubのコネクタを介して、人手を一切介さずに毎日・毎時自動的にデータベース（`goal_kpis`）へ反映。常に「嘘のない、リアルタイムの客観的データ」で進捗が測定されます。
3.  **Knowledge Graph Engine 連携 (目標のコンテキスト共有)**
    *   目標、マイルストーン、KPI、担当エージェントは、すべてナレッジグラフ上のノードとしてマッピング。
    *   エージェントが「新宿カフェの集客ブログを書く」というタスク（ワークフロー）を実行する際、エージェントは自動的にナレッジグラフから「このタスクは『新宿店MEO 3位以内』という上位目標に結びついている」ことを理解。
    *   この理解（Grounding）により、文章のキーワード選定やトーンを「目標達成に最も有利な形」に自律調整し、ハルシネーションを極限まで排除した一貫性のある仕事を完遂します。
4.  **Private Agent Marketplace 連携 (スキルアセット調達)**
    *   目標達成プランの構築時に「YouTube動画による認知獲得」というマイルストーンが必要と判断された場合。
    *   AIは自動的にマーケットプレイスを検索し、動画構成・スクリプト生成に特化した「Video_Script_Creator_Agent」を一時的（スポット雇用）にアサインし、チームを動的にスケールさせます。

## 6. 将来拡張設計 (Future Extensibility Plan)

### ① Cross-Workspace Goal Alignment（企業間サプライチェーン・目標セキュア連動）
*   **提携企業、アライアンス先、または子会社のAI Workspace（複数のOS環境）の目標・進捗と自社の目標をセキュアにシンクロさせる仕組み**:
    *   自社が「新店舗オープン」という目標を設定した際、施工を担当するパートナー会社のWorkspaceが掲げる「内装工事完了期限」の目標ノードを暗号化（P2P）を介して連動。
    *   相手の社内情報（職人配置、コスト詳細など）の機密は保護しつつ、「期日の遅延予測アラート」だけを目標エンジン同士が自律通信。
    *   パートナーの工事遅延を事前に検知した自社AIが、「オープン集客広告の開始目標日」を自動的に5日後ろ倒しにプラン修正。サプライチェーン全体のロスを最小化する「マルチ・Workspace分散目標ガバナンス」へと拡張します。

### ② Swarm Goal Evolution (自律目的創造 ＆ AI自己モチベーション学習)
*   **目標管理AIが、自社の財務状況やビジネスチャンス（Knowledge Graph）を自律分析し、人間が指示していなくても「次に目指すべき新規目標」を自ら創造・提案する仕組み**:
    *   「新宿個室カフェ」の目標（売上・集客）がすべて大成功（100%達成）して安定期に入った際。
    *   目標エンジンの「CEO Brain」がナレッジグラフ、トレンドキーワード、財務余剰キャッシュデータを分析。
    *   人間が指示していないにもかかわらず、「次に狙うべきは『池袋エリアでの防音個室カフェ展開』または『深夜営業に特化したコワーキングスペースへの事業拡張』である。これを達成するための推奨目標、KPI、必要マイルストーン、チーム構成、および予算見積もりは以下の通りです」と、新規事業計画（Goal Proposal）を自律的にデリバリーする、完全自律自己組織化経営（Autonomous Strategic Self-Evolution）へと進化します。

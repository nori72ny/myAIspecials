# AI Workspace 「プレゼンテーション・ビルダー（Presentation Builder）」機能設計仕様書

本書は、AI Workspaceにおいて、調査・分析データ、提案内容、またはURLなどのインプット情報から、ビジネスに最適化された説得力のあるスライドストーリーを自律構築し、PowerPoint (pptx)、PDF、およびGoogleスライド形式の高品質なプレゼンテーション資料を自動生成する**「Presentation Builder（プレゼンテーション・ビルダー）」**の機能設計仕様書です。

単なる「箇条書きのテキストスライド」を作るのではなく、コンサルタントやトップセールスのストーリーライン（課題提起 ➔ 分析 ➔ 解決策 ➔ 費用対効果 ➔ ロードマップ）を模倣したスライド構成を自動設計し、図解・グラフ・比較表を美しくレイアウトした資料を数分で作成・出力します。

---

## 1. 業務フロー ＆ ストーリーライン構築コンセプト

Presentation Builderは、指定されたテーマや素材（SEO、MEO、競合分析レポート等）を自律的に構造化し、オーディエンス（クライアント、役員等）の意思決定を促す論理的なストーリーへと再構成（リライティング ＆ レイアウト）します。

```
[1. インプット情報の定義] (テーマ、案件名、会社名、URL、調査分析テキスト等の流し込み)
              │
              ▼
[2. AI 自律処理・プロット構築] (Storytelling Planner)
      ├─ ターゲット、目的、資料のトーン（コンサル風、モダン等）の特定
      ├─ 競合比較、SWOT、市場データ、KPI等の構造化
      └─ 全9章からなる「説得ストーリーライン」のドラフト構築
              │
              ▼
[3. スライド・トポグラフィー ＆ レイアウト設計] (Visual Arranger)
      ├─ 各スライドの役割に合わせた最適なレイアウト定義 (1カラム, 2カラム, 比較表, タイムライン)
      └─ グラフデータ（棒グラフ、折れ線、円グラフ、マトリクス）の自動抽出・定義
              │
              ▼
[4. 物理ファイル・レンダリング] (pptxgenjs / PDF Renderer)
      ├─ 指定されたデザインテンプレート（シンプル、モダン、高級、コンサル等）のカラーパレット適用
      └─ ベクター図、シェイプ、グラフ、テキスト、およびスライドマスターの統合ビルド
              │
              ▼
[5. クオリティ査読 ＆ 検品 (Presentation Agent QA)] ──► 文字あふれ、デザイン崩れの自己修正
              │
              ▼
[6. 成果物のコミット ＆ 保存] ──────────────────────► (ZIP・PPTXダウンロード / Projects & Knowledge へ自動コミット)
```

## 3. データベース・スキーマ設計 (Database Schema Design)

プレゼン全体のメタ設定、各スライドの論理構成（順序、テキスト、レイアウト種別）、グラフデータ、およびファイルエクスポート履歴を管理するデータベーススキーマ。

```
                  +---------------------------+
                  |  presentation_documents   |
                  +---------------------------+
                  | id (PK)                   |
                  | project_id (FK)           |
                  | doc_name                  |
                  | doc_description           |
                  | client_company_name       |
                  | design_preset_style       |
                  | color_palette_json (JSON) |
                  | current_version           |
                  | created_by (FK)           |
                  | created_at                |
                  | updated_at                |
                  +-------------+-------------+
                                | 1
                                |
                                ├─────────────────────────────┐
                                | 1..N                        | 1..N
                  +-------------v-------------+   +------------v------------+
                  |    presentation_slides    |   |   presentation_exports  |
                  +---------------------------+   +-------------------------+
                  | id (PK)                   |   | id (PK)                 |
                  | document_id (FK)          |   | document_id (FK)        |
                  | slide_order (INT)         |   | format (PPTX, PDF, etc) |
                  | layout_type               |   | version_number (INT)    |
                  | title                     |   | file_storage_url        |
                  | subtitle                  |   | byte_size               |
                  | body_markdown (TEXT)      |   | created_at              |
                  | background_image_url      |   +-------------------------+
                  | block_coordinates (JSON)  |
                  +-------------+-------------+
                                | 1
                                |
                                | 0..N
                  +-------------v-------------+
                  |    presentation_charts    |
                  +---------------------------+
                  | id (PK)                   |
                  | slide_id (FK)             |
                  | chart_type (BAR,LINE,etc) |
                  | chart_title               |
                  | chart_data_json (JSON)    |
                  | chart_style_config (JSON) |
                  +---------------------------+
```

### テーブル定義詳細

#### ① `presentation_documents` (プレゼンマスター台帳)
*   `id`: UUID (Primary Key)
*   `project_id`: UUID (Foreign Key - 所属するプロジェクトのID)
*   `doc_name`: VARCHAR(255) (スライド資料名。例: 「新宿個室カフェ La Lounge 出店計画・営業提案書」)
*   `doc_description`: TEXT (プレゼン資料の目的、想定する相手、ストーリーの狙い概要)
*   `client_company_name`: VARCHAR(255) (提案先企業・店舗・クライアント名)
*   `design_preset_style`: VARCHAR(100) (適用デザインプリセット。`CONSULTANT`, `MODERN`, `LUXURY`, `STARTUP`)
*   `color_palette_json`: JSON (使用するカラーパレット。プライマリー、セカンダリー、背景、テキスト色。例: `{"primary":"#0A2540", "accent":"#635BFF"}`)
*   `current_version`: INT (最新の世代バージョン番号)
*   `created_by`: VARCHAR(100) (作成ユーザーのID)
*   `created_at` / `updated_at`: TIMESTAMP

#### ② `presentation_slides` (各スライド構造定義テーブル)
プレゼンを構成する、個々のスライドの表示要素と順番。
*   `id`: UUID (Primary Key)
*   `document_id`: UUID (Foreign Key)
*   `slide_order`: INT (スライドの表示順序。例: `1`, `2`, `3`...)
*   `layout_type`: VARCHAR(100) (レイアウト種別。`COVER` (表紙), `TITLE_WITH_BULLETS` (1カラム箇条書き), `TWO_COLUMNS` (2カラム), `COMPARISON_TABLE` (比較表), `TIMELINE` (ロードマップ), `KPI_METRICS` (数値ハイライト), `SUMMARY` (まとめ))
*   `title`: VARCHAR(255) (スライドの見出しタイトル。例: 「競合比較: 新宿3丁目エリアにおける差別化要因」)
*   `subtitle`: VARCHAR(512) (スライドのサブタイトルまたはリード文)
*   `body_markdown`: TEXT (スライド内の箇条書き、説明本文Markdownテキスト)
*   `background_image_url`: VARCHAR(1024) (背景、またはスライド内挿入画像URL)
*   `block_coordinates`: JSON (PptxGenJS/PDF描画用の各ブロック配置比率・座標データ。例: `{"title":{"x":0.5, "y":0.5}, "body":{"x":0.5, "y":1.2}}`)

#### ③ `presentation_charts` (スライド内グラフ・図解定義)
スライドに埋め込まれるグラフやテーブルのデータソース。
*   `id`: UUID (Primary Key)
*   `slide_id`: UUID (Foreign Key)
*   `chart_type`: VARCHAR(50) (グラフ種別。`BAR` (棒グラフ), `LINE` (折れ線グラフ), `PIE` (円グラフ), `TABLE` (比較表), `KPI_BLOCKS` (主要KPI))
*   `chart_title`: VARCHAR(255) (グラフのタイトル)
*   `chart_data_json`: JSON (実際のグラフ描画用データ構造。例: `{"labels":["競合A","競合B","当店"],"values":[12,18,142]}`)
*   `chart_style_config`: JSON (配色、軸ラベル、凡例、最大最小値などのスタイル設定)

#### ④ `presentation_exports` (物理ファイル・出力履歴)
PowerPoint、PDF、Googleスライド用に実際にビルド・エクスポートされたファイルの履歴。
*   `id`: UUID (Primary Key)
*   `document_id`: UUID (Foreign Key)
*   `format`: VARCHAR(50) (出力ファイル形式。`PPTX`, `PDF`, `GOOGLE_SLIDES`)
*   `version_number`: INT (出力時の世代バージョン番号)
*   `file_storage_url`: VARCHAR(1024) (ストレージに保存された完成スライドファイルの絶対参照パス)
*   `byte_size`: BIGINT (ファイル容量)
*   `created_at`: TIMESTAMP

## 4. スライド自律生成・ストーリーテリングエンジンの仕組み

Presentation Builderが、プロフェッショナルなストーリー（説得シナリオ）を組み立て、オフィスファイルとして具現化するための内部アルゴリズム。

### ① AIストーリーテリング・プランナー (Storytelling Planner)
*   **コンサル型「起承転結」論理構築**:
    単に入力データを要約するのではなく、相手の意思決定を引き出すための「ピラミッド原則」を適用したスライド構成案を自動設計します。
    1.  **現状と機会 (Context / Opportunity)**: なぜ今この計画が必要なのか
    2.  **核心課題 (Complication / Challenge)**: 何が最大の壁となっているのか
    3.  **解決アプローチ (Solution / Design)**: どのような物理的・技術的手段で解決するか
    4.  **期待結果 (Impact / ROI)**: 投資に対してどの程度の回収（KPI）が見込めるか
    5.  **実行計画 (Roadmap)**: いつ、だれが、どのように進めるか

### ② スライドオートレイアウト・ビルダー (Auto-Layout Arranger)
*   **文字あふれ・重なり検知と自己修復**:
    AIがテキストを生成する際、スライド1枚の物理的なスペース制限を超えて文字があふれたり、グラフと重複したりする問題を自動検知します。
    *   *文字あふれ防止アルゴリズム*: スライド1枚あたりの最大文字数を検出し、制限を超過した場合は自動的に「箇条書きの要約リライティング」を実行するか、「スライドを論理的に2枚に分割」するリテイクをバックグラウンドで自己完結します。
*   **セマンティック図解化**:
    「AからBにデータが流れ、Cで保存される」というテキスト指示から、自動的に「3ステップ型矢印図解（Flow Shape）」を生成し、PptxGenJSのシェイプオブジェクトとして描画します。

## 5. 他システム（プロジェクト、ナレッジ、エージェント）との統合・連携

Presentation Builderは、AI Workspace全体とシームレスに同期し、業務の効率化とデータの資産化を自動化します。

1.  **プロジェクト連携 (Projects System)**
    *   ユーザーがプレゼン資料を作成すると、自動的に「新宿個室カフェプロジェクト」の情報を取得します。
    *   生成されたPowerPointファイル（.pptx）およびPDFは、プロジェクトの「Documents（資料）」ドメインへ「新宿カフェ出店計画_提案書_v1.pptx」として自動的にコミットされます。
    *   プロジェクトのタイムライン（Timeline）に、「Presentation Builderが、コンサル風テーマで全9枚のスライド提案書（PPTX）をDocumentsドメインへ保存しました」というアクティビティ履歴が自動記録されます。
2.  **ナレッジハブ連携 (Knowledge Hub)**
    *   プレゼン資料を作成するためにAIが自動構築した「ストーリーの論理構成（ストーリーボード全体のテキストMarkdown）」は、自動要約されてKnowledge Hubのプロジェクトフォルダへ「永続的な資料資産（アセット）」として同期保存されます。
    *   これにより、ユーザーはナレッジコパイロット（RAGチャット）を使い、「前回プレゼンビルダーで作成した提案書の中の、競合比較スライドの主な論点を教えて」といった自然言語での問い合わせが常時可能になります。
3.  **エージェント・ビルダー連携 (Agent Builder)**
    *   スライド作成の全工程は、以下の4名のエージェントチーム（Multi-Agent Team）がバケツリレー式に処理します。
        1.  **Research Agent**: 最新の市場データ・競合データの補填と検証
        2.  **Analysis Agent**: 3C/SWOTマトリクスやKPIの算出・データ構造化
        3.  **Proposal Agent**: クライアントの心理に沿った提案シナリオ・ストーリーテリング設計
        4.  **Presentation Agent**: PptxGenJSを駆動したレイアウトの組み立て、文字あふれの検修、PPTXのコンパイル実行
    *   エージェントチームの稼働実績、成功率、APIコストはすべて「Agent Dashboard」で定量トラッキングされます。

## 6. 将来拡張設計 (Future Extensibility Plan)

Presentation Builderが、クラウドスライド共有やリッチメディア対応を遂行するための将来拡張ロードマップ。

### ① Google Slides API 連携 ＆ 双方向リアルタイム同期
*   PowerPointファイルのダウンロード出力だけでなく、Google Slides APIとOAuth連携。
*   「Googleスライドで開く」ボタンを1クリックするだけで、ユーザーのGoogleドライブ上にプレゼンテーションが自動プロビジョニングされ、オンライン上で複数人と共同編集・ブラウザ表示ができるよう同期エンジンを構築します。

### ② AI自律スライドナレーション ＆ ピッチ動画自動生成
*   スライドを出力するだけでなく、AIが「スライド1枚あたり1分」のプレゼン用ピッチ原稿を自律執筆。
*   さらに、テキスト読み上げ音声合成API（TTS）や、バーチャルAIアバター生成エンジンと連携し、「作成したスライドをAI社員が自動で解説・プレゼンする3分間のピッチ紹介動画（MP4）」を自律生成。
*   営業メールやLPにそのピッチ動画を埋め込むことで、完全自動のオンライン自動接客・自動商談を可能にするアドオンモジュールを設計します。

# AI Workspace 「人間思考共創オペレーティングシステム・コグニティブOS（Cognitive OS）」機能設計仕様書

本書は、AI Workspaceを単なる「AIチャットUI（AI outputs）」から、人間の思考プロセス、着想、長期的な興味、課題解決の試行錯誤そのものをシステム全体で包摂・支援・共創するメタ認知OS**「Cognitive OS（コグニティブOS）」**へと進化させるためのシステム構成、データスキーマ、無限キャンバスレイアウト、アイデアインキュベータ論理、未来シミュレータアルゴリズム、および知識自己進化（Knowledge Evolution）ライフサイクルを定義した製品設計仕様書です。

本OSは、従来のアプリケーションが提供する「受動的なファイル管理」や「単発のプロンプト対話」の限界を完全に突破。ユーザーの脳内にある「意図記憶（Intent Memory）」「関心マップ」「脳内長期記憶（Brain Memory）」を24時間自動学習・マッピングし、無限の3次元思考空間（Thinking Canvas / Infinite Workspace）上で社内のすべてのプロジェクト、データレイク、会議、自律エージェントを自由自在に接続・インキュベート。さらに、未来予測（Future Simulator）を交えてアイデアの収益性・実現可能性を自動多重分析・自己進化（Knowledge Evolution）させ続ける、人類とAIの**「共生知能オペレーティング環境（Symbiotic Intelligence OS）」**を確立します。

## 1. システム概要 ＆ 認知OSコンセプト (Overview & Concept)

Cognitive OSは、下図のようにユーザーの断片的なアイデアや思考ログ、会話、資料を「脳内長期記憶（Brain Memory）」へ自律的にハイドレーション（抽出・格納）。それらを無限に広がる思考宇宙「Thinking Canvas」にノードとして配置し、AIがユーザーと対話しながらマインドマップや意思決定ツリーへと有機的に成長させます。
さらに、そのアイデアが市場で通用するかを「Idea Incubator」で瞬時に多角分析し、市場やAIの進化を踏まえた「Future Simulator」で10年先のシミュレーションを実行。最後に「Knowledge Evolution」が古い知識を自動更新し、常にWorkspace全体を最新の「超構造化知識脳」に自己最適化します。

```

     ===================================================================================
     ├─ ① 統合思考意図記憶（Intent Memory）

     ├─ ② 無限協調キャンバス（Thinking Canvas / Infinite Workspace）
     │    - 境界のない無限の2D/3D平面上に、Knowledge, Projects, Meeting, Agentを自由接続・配置
     │    - ユーザーとAIのリアルタイム共同マインドマップ展開、アイデア構造化

     │    - 過去のすべての試行錯誤、ボツ案、対話コンテキスト、成果物、ボイスログを長期分散格納

                         │
                     ┌───┴──────────────────────┐
                     ▼ (思考の視覚化 ✕ 物理実行)          ▼ (認知・意思決定の伝播)
```


### テーブル定義詳細

#### ① `cognitive_profiles` (コグニティブ・プロファイル)
ユーザーごとの思考の癖、AIとの思考アライン親和度、および脳内長期記憶の累積管理データを統括。
*   `id`: UUID (Primary Key)
*   `company_id`: UUID (Foreign Key - 組織スペースのID)
*   `user_email`: VARCHAR(255) (ユーザーのメールアドレス。例: `nori72ny@gmail.com`)
*   `overall_co_thinking_score`: DECIMAL(5, 2) (ユーザーとAIの共同思考同調率。98.50 %)
*   `brain_memory_bytes`: BIGINT (脳内記憶として構造化・インデックス化されたキャッシュの累積容量（バイト数）)
*   `created_at` / `updated_at`: TIMESTAMP

#### ② `cognitive_intent_memories` (意図・目的・興味記憶テーブル)
ユーザーの「暗黙の目的」や「現在熱狂している興味関心領域」を、Workspaceでの活動からAIが自動ラベリングして蓄積する、思考の重心マトリクス。
*   `id`: UUID (Primary Key)
*   `profile_id`: UUID (Foreign Key)
*   `category`: VARCHAR(50) (意図区分。`GOAL` (長期的目標), `INTEREST` (現在の関心), `PREFERENCE` (経営・思考の嗜好性))
*   `keyword`: VARCHAR(255) (キーワード。例: `「新宿深夜店舗活用」`, `「価格訴求の排除」`)
*   `confidence`: DECIMAL(5, 4) (その意図が正しいと判定されたAI確信度。0.0000〜1.0000)
*   `last_used_at` / `created_at`: TIMESTAMP

#### ③ `cognitive_thinking_nodes` (無限キャンバス上の浮遊ノード)
無限キャンバスにユーザーがドラッグ配置、またはAIが自律生成した思考エレメントの座標、種類、およびエレメント間の「セマンティックな接続線（エッジ）」の定義。
*   `id`: UUID (Primary Key)
*   `profile_id`: UUID (Foreign Key)
*   `node_type`: VARCHAR(50) (ノード種。`KNOW` (知識・実績), `AGENT` (エージェント起動トリガー), `PROJECT` (PM連携), `MEETING` (会議), `IDEA` (アイデア着想))
*   `payload_json`: JSON (ノード固有のデータスナップショット。`{"title": "深夜個室カフェLP", "url": "https://...", "stripe_revenue": 1250000}`)
*   `pos_x` / `pos_y`: DECIMAL(12, 4) (無限キャンバス上でのx座標、y座標)
*   `connections`: JSON (このノードから伸びている、他のノードへのセマンティック接続線配列。`[{"target_node_id": "node-405", "label": "売上因果相関", "strength": 0.92}]`)
*   `created_at`: TIMESTAMP

#### ④ `cognitive_brain_memories` (脳内長期記憶ストレージ)
過去のチャット、会議の文字起こし、提出した提案資料、失敗したボツ案、独り言のボイスメモなど、過去のすべての知的活動を「意味（セマンティック）」ごとに分解・チャンク化した不変の長期格納DB。
*   `id`: UUID (Primary Key)
*   `profile_id`: UUID (Foreign Key)
*   `memory_hash`: VARCHAR(64) (データの重複保存を防ぐ不変ハッシュ値)
*   `title`: VARCHAR(255) (記憶の要約。例: `「新宿店の過去2回の売上予算失敗の反省」`)
*   `content_chunk`: TEXT (意味分解されたテキスト本体)
*   `source_ref`: VARCHAR(255) (情報源。`meeting-204` (会議), `chat-44` (チャット), `doc-103` (資料))
*   `cognitive_decay_rate`: DECIMAL(5, 4) (エビングハウスの忘却曲線を模した、想起重要度の減衰レート。重要度に応じてAIが動的に再評価)
*   `created_at`: TIMESTAMP

#### ⑤ `cognitive_idea_incubators` (アイデア育成・シミュレーション履歴)
ユーザーの「カフェ深夜営業」のようなアイデアに対して、AIが自動並列実行した収益、競合、実現性、未来リスクの分析結果。
*   `id`: UUID (Primary Key)
*   `node_id`: UUID (Foreign Key - 該当するキャンバス上の `cognitive_thinking_nodes` のID)
*   `feasibility_score`: DECIMAL(5, 2) (技術的・物理的実現性。100点満点)
*   `market_size_json`: JSON (自動市場調査データ。`{"market_growth": "140%", "cagr": "8.5%", "target_users_count": 45000}`)
*   `revenue_model_json`: JSON (自動シミュレートされた収益予測モデル。`{"monthly_profit": 350000, "break_even_months": 4}`)
*   `competitor_analysis`: JSON (周辺競合店舗・サービスの自動特定と強み・弱み分析)
*   `simulations_json`: JSON (Future Simulatorによる1〜10年先の時間軸シナリオ予測の配列)

#### ⑥ `cognitive_knowledge_evolutions` (自己組織化知識更新・クリーンアップログ)
矛盾した知識の統合、古いデータの更新、重複マージなど、知識が自己進化（Evolution）した履歴を監査。
*   `id`: UUID (Primary Key)
*   `profile_id`: UUID (Foreign Key)
*   `action_type`: VARCHAR(50) (更新種別。`MERGE` (重複統合), `UPDATE` (古い情報の更新), `PRUNE` (忘却・刈り込み))
*   `target_nodes_json`: JSON (更新・統合対象となった記憶ノードのUUIDリスト)
*   `evolution_score`: DECIMAL(5, 2) (この最適化によって向上した、知識のファクト整合率・確信度)
*   `created_at`: TIMESTAMP

## 4. 思考・認知共創・未来シミュレーション論理 (Cognitive Logic)

Cognitive OSが、人間の思考の壁を取り払い、着想を1秒で現実の利益をもたらすプロダクトへ育成するための自律認知パイプライン。

```
     [ 1. 意図・関心・文脈の受動トラッキング (Ambient Harvesting) ]
       - ユーザーが話したボイス、プロジェクトのチャット、資料、 Stipeの売上実績を、
         Brain Memory（脳内長期記憶）へ 24時間サイレントに構造化・格納。
       - ユーザーの「新宿店の売上を伸ばしたい」「深夜の有効活用に関心がある」という熱狂ベクトル（興味・目的）を
         自動で `cognitive_intent_memories` へ重心プロファイリング。
                            │
                            ▼
     [ 2. 無限キャンバス上でのセマンティック共同連想 (Symbiotic Thinking) ]
       - ユーザーがキャンバスを開くと、AIが文脈に沿って「過去の新宿深夜営業の反省ノード」や
         「Stripe実績ノード」を自動的に『近くに浮遊』させる（想起の自律支援）。
       - ユーザーがアイデア（深夜個室カフェ）を置いた瞬間、AIは関連キーワードを自動マインドマップ展開。
       - 2つの無関係なノード（深夜営業 ✕ MEO）が結ばれた瞬間、その「セマンティックな因果接続」を検知。
                            │
                            ▼
     [ 3. アイデアインキュベート ✕ 未来シナリオシミュレーション (Incubate & Predict) ]
       - 結ばれたアイデアノードに対し、Idea Incubatorが自律起動。
         - ウェブ検索、Connector Hub、BI Centerを同時に並列探索。
         - 「新宿深夜カフェ市場の需要（140%成長）」「周辺競合3店舗の特定」「想定初期コストとブレイクイーブン」
```
